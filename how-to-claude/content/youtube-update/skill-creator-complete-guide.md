## 이게 뭔가요?

스킬 크리에이터(`/skill-creator`)는 Claude Code의 **스킬을 만들고, 테스트하고, 개선하는 전체 과정**을 도와주는 도구입니다.

비유하자면 이렇습니다:

> 스킬이 "직원에게 주는 업무 매뉴얼"이라면, 스킬 크리에이터는 **매뉴얼을 작성하고, 직원이 잘 따르는지 테스트하고, 문제가 있으면 매뉴얼을 수정하는 품질관리 팀**입니다.

별도 명령어를 외울 필요 없이, `/skill-creator` 뒤에 하고 싶은 걸 말하면 됩니다.

## 왜 알아야 하나요?

- 스킬을 직접 만들 수 있으면 **Claude Code를 내 업무에 완벽하게 맞출** 수 있습니다
- "만들었는데 잘 작동하나?" 확인을 자동으로 해주니 **시행착오가 줄어듭니다**
- 트리거 최적화까지 하면 **"불러야 할 때 안 불리는" 문제**가 사라집니다

## 어떻게 하나요?

스킬 크리에이터는 **9가지 기능**을 제공합니다. 하나씩 살펴봅니다.

---

### 기능 1: 새 스킬 만들기

처음부터 새로운 스킬을 만들고 싶을 때 사용합니다. 인터뷰 → 설계 → SKILL.md(스킬 설정 파일) 작성까지 한 번에 진행됩니다.

**사용 방법:**
```
/skill-creator 유튜브 영상을 요약해서 블로그 글로 만드는 스킬 만들어줘
/skill-creator 매주 월요일마다 회의록을 정리하는 스킬 만들어줘
```

**진행 순서:**
1. "이 스킬이 뭘 하게 하고 싶은지" 질문
2. "어떤 말에 반응해야 하는지" 트리거 설계
3. SKILL.md 파일 + 필요한 참조 파일 자동 생성

<div class="example-case">
<strong>예시</strong>

```
나: /skill-creator 이메일을 자동으로 답장하는 스킬 만들어줘

Claude: 몇 가지 질문드릴게요.
1. 어떤 종류의 이메일에 답장하나요? (고객 문의? 사내 메일?)
2. 답장 톤은 어떤 스타일인가요? (격식체? 친근체?)
3. 답장 전에 사용자 확인을 받을까요, 바로 보낼까요?
...
→ 인터뷰 완료 후 SKILL.md 자동 생성
```

</div>

### 기능 2: 기존 스킬 분석 및 개선

이미 만들어진 스킬의 문제점을 찾고 수정합니다.

**사용 방법:**
```
/skill-creator /push 스킬 분석하고 개선해줘
/skill-creator /updateyoutube 스킬의 문제점을 찾아줘
```

**분석 항목:**
- 트리거(description)가 충분한지
- 에러 처리가 빠져있진 않은지
- 로직이 모호하거나 비효율적이진 않은지

---

### 기능 3: 테스트 케이스 작성 및 실행

스킬이 실제로 잘 작동하는지 **자동 테스트**를 돌려봅니다.

**사용 방법:**
```
/skill-creator /push 스킬에 테스트 케이스 만들고 실행해줘
/skill-creator updateyoutube 스킬을 테스트해봐
```

**작동 원리:**
1. 실제 사용자가 입력할 법한 테스트 프롬프트(AI에게 보내는 요청 메시지) 2~3개 작성
2. "스킬 있는 버전"과 "스킬 없는 버전"을 동시에 실행
3. 결과를 비교해서 스킬이 실제로 차이를 만드는지 확인

<div class="example-case">
<strong>예시</strong>

`/push` 스킬을 테스트한다면:
- 테스트 1: "올려줘" → 스킬이 자동 커밋+푸시를 하는가?
- 테스트 2: ".env 파일이 있을 때" → 민감 파일을 제외하는가?
- 테스트 3: "변경사항이 없을 때" → 적절한 메시지를 보여주는가?

</div>

---

### 기능 4: 벤치마크 및 정량 평가

테스트 결과를 **숫자로** 평가합니다. "통과율 몇 %, 시간 얼마, 비용 얼마"를 계산합니다.

**사용 방법:**
```
/skill-creator /push 스킬 벤치마크 돌려줘
/skill-creator 이전 버전과 현재 버전 성능 비교해줘
```

**측정 항목:**

| 항목 | 설명 |
|------|------|
| 통과율 | 테스트 기준을 몇 % 통과했는지 |
| 토큰 사용량 | AI가 처리한 텍스트 양 (비용과 직결) |
| 실행 시간 | 작업 완료까지 걸린 시간 |

---

### 기능 5: 리뷰어 뷰어 (브라우저)

테스트 결과를 **브라우저에서 시각적으로 확인**하고 피드백을 남길 수 있는 화면입니다.

**사용 방법:**
```
/skill-creator 테스트 결과 리뷰어 열어줘
/skill-creator 결과를 브라우저에서 볼 수 있게 해줘
```

**화면 구성:**
- **Outputs 탭**: 테스트 케이스별 결과를 하나씩 보면서 피드백 작성
- **Benchmark 탭**: 통과율, 토큰(AI가 처리하는 텍스트 단위), 실행 시간 비교 대시보드
- 이전 버전 결과와 나란히 비교도 가능

피드백을 다 작성하면 "Submit All Reviews" 버튼을 누르면 됩니다.

---

### 기능 6: 반복 개선 루프

피드백을 반영하여 스킬을 수정하고, 다시 테스트하는 과정을 **만족할 때까지 반복**합니다.

**사용 방법:**
```
/skill-creator 피드백 반영해서 스킬 다시 개선해줘
/skill-creator 다음 iteration 진행해줘
```

**흐름:**
```
스킬 수정 → 테스트 재실행 → 리뷰어에서 확인 → 피드백 → 다시 수정 → ...
```

피드백이 모두 비어있거나 "됐어"라고 하면 종료됩니다.

---

### 기능 7: 블라인드 비교 (고급)

두 버전의 스킬 결과를 **독립적인 AI 심판**에게 보여주고, 어느 쪽이 더 좋은지 판단받습니다.

**사용 방법:**
```
/skill-creator 이전 버전이랑 새 버전 블라인드 비교해줘
/skill-creator A/B 비교 테스트 해줘
```

**핵심**: 심판 AI는 어떤 게 새 버전이고 어떤 게 구 버전인지 **모르는 상태**로 평가합니다. 그래서 편향 없는 결과를 얻을 수 있습니다.

---

### 기능 8: Description 트리거 최적화

스킬이 **언제 자동으로 불릴지** 결정하는 description(스킬 설명) 필드를 최적화합니다.

**사용 방법:**
```
/skill-creator /updateyoutube 스킬의 description 트리거 최적화해줘
/skill-creator 이 스킬이 잘 불리도록 description 개선해줘
```

**진행 순서:**
1. "이 스킬이 작동해야 하는 질문" 10개 + "작동하면 안 되는 질문" 10개 생성
2. 사용자가 검토하고 수정
3. 자동으로 최대 5회 반복하면서 최적의 description 탐색
4. 가장 높은 점수의 description을 적용

<div class="example-case">
<strong>예시</strong>

`/push` 스킬의 트리거 최적화:
- 작동해야 하는 질문: "올려줘", "커밋하고 푸시해", "깃허브에 올려"
- 작동하면 안 되는 질문: "push 알림 설정해줘", "git push의 원리가 뭐야?"

→ 최적화 후 description이 더 정확하게 반응

</div>

---

### 기능 9: 패키징 및 배포

완성된 스킬을 `.skill` 파일로 포장해서 다른 프로젝트에서도 쓸 수 있게 만듭니다.

**사용 방법:**
```
/skill-creator /push 스킬 패키징해줘
/skill-creator 이 스킬을 .skill 파일로 만들어줘
```

---

### 복합 요청

여러 기능을 한 번에 요청할 수도 있습니다:

```
/skill-creator /updateyoutube 스킬을 분석하고 테스트까지 돌려줘
/skill-creator 새 스킬 만들고 바로 벤치마크까지 해줘
/skill-creator /push 스킬 개선하고 description 최적화까지 해줘
```

## 실전 예시

<div class="example-case">
<strong>실전 케이스: /updateyoutube 스킬 품질 개선</strong>

1. `/skill-creator /updateyoutube 스킬을 분석해줘`로 시작
2. 문제점 발견: 전문 용어 괄호 설명 부족, Mermaid 품질 저하, 빈 데이터 방치

5. 기존 26개 문서 일괄 수정 (서브에이전트 3개 병렬 처리)
6. `/push`로 커밋

→ 결과: 전문 용어에 괄호 설명 추가, Mermaid 색상/노드 수 수정, 빈 표 셀 제거

</div>

## 주의할 점

- **테스트와 벤치마크는 토큰을 소모합니다** — 서브에이전트를 여러 개 돌리기 때문에, 간단한 스킬이면 "테스트 없이 바로 개선"을 요청하는 게 효율적일 수 있습니다
- **블라인드 비교도 비용이 큽니다** — 정말 두 버전 중 어느 게 나은지 판단이 안 설 때만 사용하세요
- **Description 최적화는 마지막에** — 스킬 본문이 확정된 후에 하는 게 좋습니다. 본문이 바뀌면 최적 description도 달라질 수 있기 때문입니다

## 정리

- `/skill-creator`는 스킬의 **생성 → 테스트 → 평가 → 개선 → 최적화 → 배포** 전체를 다루는 올인원 도구
- 별도 명령어 없이 **자연어로** 원하는 기능을 요청하면 됩니다
- 핵심 가치: "감으로 만들고 감으로 수정"에서 → "데이터 기반으로 만들고 데이터 기반으로 개선"으로
