## 이게 뭔가요?

2026년 초, Claude Code의 소스코드가 뜻밖의 경로로 인터넷에 공개됐습니다. Anthropic(앤트로픽)이 배포한 NPM(Node.js 패키지 관리자) 패키지에 소스맵(컴파일된 코드를 원본 코드와 연결하는 파일) 파일이 포함되어 있었고, 한 보안 연구자가 이를 추적해 TypeScript(타입스크립트) 원본 소스를 발견했습니다. 수 시간 만에 전체 코드베이스가 GitHub에 미러링(복사·공개)되어 확산됐습니다.

이 영상은 그 소스코드를 직접 분석해 실제 사용에 도움이 되는 8가지 핵심 인사이트를 정리한 내용입니다. 마치 자동차 설계도가 유출되어 "이 버튼이 실제로 이런 역할을 했구나"를 알게 되는 것처럼, 평소에는 보이지 않던 내부 구조가 드러났습니다.

> **참고:** 소스코드 저작권은 여전히 Anthropic 소유입니다. 이 문서는 사용법 인사이트만 다루며, 소스코드 다운로드나 배포는 권장하지 않습니다.

## 왜 알아야 하나요?

대부분의 사람들이 Claude Code를 "터미널에서 쓰는 챗봇" 정도로만 활용합니다. 하지만 소스코드가 밝혀준 실제 구조를 이해하면, 같은 도구로 훨씬 더 많은 일을 할 수 있습니다. 특히 숨겨진 커맨드와 퍼미션 최적화만 알아도 작업 속도와 비용이 크게 달라집니다.

## 8가지 핵심 인사이트

### 인사이트 1: Claude Code는 챗봇이 아닌 완전한 에이전트 런타임

소스코드가 밝혀준 가장 중요한 사실입니다. Claude Code는 단순히 "AI에 터미널 접근 권한을 준 것"이 아닙니다.

**기술 스택:** Bun(JavaScript 런타임 & 빌드 도구) + TypeScript(타입스크립트) + React + Ink(터미널 UI 프레임워크)로 구성된 완전한 애플리케이션입니다. 코드의 약 90%를 Claude Code 자신이 작성했다고 알려져 있습니다.

**실제 구성 요소:**
- **도구 시스템(Tool System)**: 파일 읽기·쓰기, 터미널 실행 등을 처리
- **커맨드 시스템(Command System)**: 슬래시 커맨드 처리
- **메모리 시스템(Memory System)**: 프로젝트/유저/세션 정보 저장
- **퍼미션 엔진(Permission Engine)**: 허용·차단 규칙 관리
- **멀티에이전트 코디네이터(Multi-Agent Coordinator)**: 여러 AI 에이전트 협업
- **MCP(Model Context Protocol, 외부 도구 연결 규격) 클라이언트 & 서버**: 외부 도구 연결

**실행 흐름:**
```
내 입력 → CLI 파서 → 쿼리 엔진 → LLM API 호출
         → 도구 실행 루프 → 터미널에 결과 표시
```

**왜 중요한가?** 챗봇처럼 질문만 하면 이 시스템의 10%만 쓰는 겁니다. 나머지 90%는 커맨드, 메모리, 퍼미션 설정에 있습니다.

### 인사이트 2: 대부분이 모르는 85개+ 슬래시 커맨드

소스코드에 따르면 Claude Code에는 수십 개의 슬래시 커맨드가 내장되어 있습니다. 대부분의 사용자는 `/help` 정도만 알고 있습니다.

**바로 써먹을 수 있는 핵심 커맨드:**

| 커맨드 | 기능 | 언제 쓰나 |
|--------|------|-----------|
| `/init` | CLAUDE.md 파일 자동 생성 | 새 프로젝트 시작할 때 |
| `/plan` | 실행 전 계획 먼저 수립 | 복잡한 작업, 파일 수정 전 검토 원할 때 |
| `/compact` | 대화 기록 압축 | 토큰 비용 줄이고 싶을 때 |
| `/review` | 코드 리뷰 자동 실행 | 작업 완료 후 품질 검사 |
| `/security-review` | 보안 취약점 검사 | 배포 전 보안 점검 |
| `/context` | 현재 로드된 파일 목록 확인 | 어떤 파일이 비용을 잡아먹는지 확인 |
| `/cost` | 현재 세션 비용 확인 | 토큰 사용량 모니터링 |
| `/resume` | 이전 세션 이어서 작업 | 다음 날 작업 재개할 때 |
| `/summary` | 현재 세션 요약 | 대화 내용 정리 |

대화가 길어지면 이전 내용이 모두 토큰을 소비합니다. `/compact`를 쓰면 중요한 정보는 유지하면서 불필요한 내용은 압축됩니다.

특정 정보를 보존하고 싶다면:
```
/compact 웹사이트 API 연동 관련 내용은 모두 보존해줘
```

### 인사이트 3: CLAUDE.md가 핵심 레버

소스코드는 CLAUDE.md 파일이 세 가지 레이어를 갖는다는 걸 보여줍니다:
- **유저 레벨 메모리**: 개인 설정과 선호 사항
- **추출 메모리(Extracted Memories)**: AI가 자동으로 학습한 내용
- **팀 메모리 동기화**: 팀원 간 설정 공유

이 파일은 매 세션마다 대화 시작 전에 자동으로 로드됩니다. 즉, CLAUDE.md에 적힌 내용은 매번 Claude에게 자동으로 전달되는 "업무 지침서"입니다.

**효과적인 CLAUDE.md 작성법:**

```markdown
# 프로젝트 규칙

## 기술 스택
- TypeScript strict mode 필수
- 패키지 관리자: pnpm (npm 사용 금지)
- 테스트 파일은 소스 파일 옆에 배치

## 절대 하지 말 것
- 데이터베이스 스키마를 마이그레이션 없이 수정하지 말 것
- 환경변수를 코드에 직접 하드코딩하지 말 것

## 코드 스타일
- 함수는 50줄 이내로 유지
- 컴포넌트 하나에 하나의 책임만
```

**나쁜 CLAUDE.md:** 프로젝트 역사, 회의록, 장황한 배경 설명
**좋은 CLAUDE.md:** 결정 규칙, 제약사항, 컨벤션(규칙)

### 인사이트 4: 퍼미션 설정이 속도를 결정한다

Claude Code가 매번 "이거 해도 될까요?", "이 파일 수정해도 될까요?" 하고 물어보는 게 답답했다면, 그건 퍼미션(Permission, 허용 권한) 설정이 최적화되지 않은 것입니다.

소스코드가 공개한 퍼미션 모드:
- **default**: 모든 작업마다 허가 요청
- **plan**: 먼저 계획 수립 후 실행
- **bypass**: 자동 실행 (위험한 작업 제외)

**와일드카드(wildcard, 포괄적 규칙) 퍼미션 설정:**

`settings.json` 또는 `settings.local.json`에 설정:

```json
{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Edit(src/**)",
      "Read(**)"
    ],
    "deny": [
      "Bash(rm -rf:*)"
    ]
  }
}
```

매일 반복하는 작업(git 커맨드, src 폴더 편집)은 한 번만 설정해두면 이후로는 매번 허가를 받지 않아도 됩니다.

### 인사이트 5: 멀티에이전트 아키텍처로 작업 분해

소스코드에는 **코디네이터 서브시스템(Coordinator Subsystem)**, **에이전트 도구**, **팀 도구**, **백그라운드 태스크(Background Task)** 시스템이 포함되어 있습니다.

이것이 의미하는 바: Claude Code는 하나의 요청을 여러 에이전트가 병렬로 처리하도록 설계되었습니다.

**분해적 프롬프팅(Decomposition Prompting) vs 일괄 프롬프팅:**

<div class="example-case">
<strong>나쁜 예: 일괄 프롬프팅</strong>

```
이 모듈 전체를 리팩터링하고, 테스트도 업데이트하고, 문서도 고쳐줘
```

→ 하나의 에이전트가 모든 걸 처리하려다 품질이 낮아집니다.

</div>

<div class="example-case">
<strong>좋은 예: 분해적 프롬프팅</strong>

```
1단계: 현재 모듈 구조를 분석하고 리팩터링 계획을 세워줘
2단계: 계획이 확인되면 리팩터링 실행
3단계: 변경된 코드에 맞춰 테스트 업데이트
4단계: 문서 반영
```

→ 각 단계가 명확하고, AI가 설계된 방식대로 작동합니다.

</div>

### 인사이트 6: MCP + 스킬이 진짜 확장 계층

소스코드가 확인해준 사실: Claude Code는 MCP(Model Context Protocol, 외부 도구 연결 규격) 클라이언트이자 서버입니다. 외부 도구에 연결할 수도 있고, 다른 시스템이 Claude Code에 연결할 수도 있습니다.

더 나아가 스킬(Skill)과 플러그인 계층도 존재합니다. 이것은 반복 작업을 자동화하고 프로젝트 고유의 기능을 쌓아갈 수 있는 확장 구조입니다.

**연결 가능한 것들:**
- 데이터베이스 (직접 쿼리 가능)
- 내부 업무 도구
- 문서 시스템
- 외부 API

Claude Code를 단순한 코딩 도구가 아닌 **업무 통합 레이어**로 만들 수 있습니다.

### 인사이트 7: 아직 공개되지 않은 내부 기능들

소스코드에서 `user_type: "ant"` (Anthropic 내부) 체크 코드가 발견됐습니다. 이는 특정 기능들이 내부 사용자에게만 활성화된다는 의미입니다.

**발견된 내부 기능 플래그들:**
- **음성 모드(Voice Mode)**: push-to-talk 방식으로 Claude Code 음성 조작
- **데몬 모드(Daemon Mode)**: Claude Code를 백그라운드 프로세스로 상시 실행
- **코디네이터 모드(Coordinator Mode)**: 여러 Claude 인스턴스를 병렬로 오케스트레이션(조율)
- **KAIROS**: 영구 메모리 + 자율 에이전트 데몬 시스템 (항시 작동 에이전트)

이 기능들이 언제 정식 출시될지는 소스코드만으로 알 수 없습니다. 하지만 Claude Code가 빠르게 기능을 출시하는 이유는 이미 내부적으로 개발·테스트가 완료된 것들을 순차 공개하기 때문입니다.

### 인사이트 8: 상위 1% 사용자의 7가지 습관

소스코드 분석이 말해주는 핵심: 탑 유저는 "더 좋은 프롬프트"가 아니라 **더 나은 운영 환경**을 설계합니다.

| # | 습관 | 핵심 |
|---|------|------|
| 1 | **CLAUDE.md 관리** | 짧고 명확하게, 정기 업데이트 |
| 2 | **핵심 커맨드 학습** | /plan, /compact, /context, /review, /cost, /resume |
| 3 | **퍼미션 최적화** | 반복 작업에 와일드카드 규칙 설정 |
| 4 | **작업 분해** | 검색 → 계획 → 실행 → 검증 단계 분리 |
| 5 | **컨텍스트 관리** | /compact 습관화, 불필요한 파일 제거 |
| 6 | **도구 연결** | MCP 서버, 플러그인, 스킬 추가 |
| 7 | **인프라처럼 설정** | 모델 라우팅, 백엔드 설정, 프라이버시 컨트롤 조정 |

## 주의할 점

- **컨텍스트는 곧 비용입니다.** 로드된 파일 수가 많을수록 토큰 비용이 올라갑니다. `/context`로 현재 로드 상태를 확인하고, 불필요한 파일은 정리하세요.
- **CLAUDE.md는 길수록 좋지 않습니다.** 핵심 규칙만 담고, 상세 내용은 별도 파일에 두고 링크로 연결하는 방식이 효과적입니다.
- **퍼미션 와일드카드는 신중하게 설정하세요.** 파일 삭제 등 되돌리기 어려운 작업은 항상 명시적으로 차단 규칙을 추가하는 것이 좋습니다.

## 정리

- Claude Code는 챗봇이 아닌 완전한 에이전트 런타임 — 커맨드·메모리·퍼미션 시스템을 활용해야 진짜 가치가 나온다
- /plan, /compact, /context, /cost 4개 커맨드만 습관화해도 작업 효율과 비용이 크게 달라진다
- CLAUDE.md를 "AI 업무 지침서"로 관리하고, 퍼미션을 와일드카드로 최적화하면 AI가 자율적으로 작동하는 환경이 완성된다
