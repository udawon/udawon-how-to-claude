## 이게 뭔가요?

Claude Code에서 `/plan` 명령으로 계획을 세울 때, 보통 터미널에서 수분씩 걸리고 결과가 텍스트로 쭉 나열됩니다. 이 방식을 완전히 바꾸는 기능이 **Ultra Plan**입니다.

Ultra Plan은 마치 혼자 레포트를 쓰는 대신 팀원 여러 명이 동시에 자료 조사를 하는 것과 같습니다. 터미널에서 시작만 하면 계획 작업이 Anthropic 클라우드로 넘어가고, Claude Opus 4.6 모델이 여러 에이전트(AI 작업 단위)를 병렬로 돌려 훨씬 빠르고 구조화된 계획을 만들어줍니다. 완성된 계획은 웹에서 문서처럼 검토하고 댓글을 달 수 있으며, 승인하면 터미널로 "텔레포트"해 바로 실행합니다.

## 왜 알아야 하나요?

- **속도**: 같은 작업을 로컬 플랜으로 4분+ 걸릴 때 Ultra Plan은 1분에 완료
- **실행 품질**: 계획이 명확할수록 에이전트가 더 빠르고 정확하게 구현 — 실제 테스트에서 Ultra Plan 버전이 로컬 플랜 버전보다 실행 시간도 2~3배 빨랐음
- **로컬 토큰 절약**: 대시보드 생성 테스트 기준, 로컬 플랜은 131,000 토큰 사용, Ultra Plan 로컬 실행은 82,000 토큰 사용 (클라우드 플래닝 토큰은 별도)
- **검토 인터페이스**: 터미널 텍스트 대신 문서 형태로 계획을 확인하고, 섹션별 댓글과 이모지 반응을 남길 수 있음

## 어떻게 하나요?

### 사전 조건

1. **Claude Pro, Max, Team, 또는 Enterprise 구독** 필요 (API 빌링 계정으로는 사용 불가)
2. **프로젝트에 Git 저장소 연결** 필요 — 클라우드에서 코드베이스를 읽으려면 Git/GitHub로 동기화되어 있어야 함
3. **CLI(커맨드라인 인터페이스) 권장** — VS Code 확장이나 데스크톱 앱에서도 지원되지만, `/ultraplan` 명령으로 시작하는 CLI 방식이 가장 안정적입니다

### 방법 1: `/ultra plan` 슬래시 명령으로 시작

```
/ultra plan {계획할 내용}
```

<div class="example-case">
<strong>예시</strong>

```
/ultra plan 사용자 인증 시스템을 구축해줘. JWT 토큰 기반, 소셜 로그인 포함
```

실행하면 "Do you want to do this in the cloud?" 질문이 나옵니다. Yes를 선택하면 클라우드 플래닝이 시작되고 웹 링크가 제공됩니다.

</div>

### 방법 2: 프롬프트에 "ultra plan" 키워드 포함

프롬프트 안에 "ultra plan"이라는 단어가 있으면 `ultra think`처럼 자동으로 감지되어 울트라 플랜 모드 실행 여부를 물어봅니다.

### 방법 3: 기존 로컬 계획을 클라우드에서 재작성

이미 로컬에서 계획을 작성했다면 "이 계획을 ultra plan으로 클라우드에서 다시 작성해줘"라고 요청하면 됩니다.

### Ultra Plan 워크플로우

```
CLI에서 /ultra plan 실행
    ↓
클라우드에서 계획 생성 (Opus 4.6, 멀티 에이전트)
    ↓
웹 링크로 계획 검토 (댓글 / 이모지 반응)
    ↓
"Approve & Teleport Back" 클릭
    ↓
터미널에서 계획 수신 → 바로 실행
```

## 실전 예시

<strong>실전 케이스: 대시보드 구축 비교 테스트</strong>

영상에서 동일한 대시보드 스펙을 로컬 플랜과 Ultra Plan 두 버전으로 동시에 실행한 결과입니다. Ultra Plan의 계획이 더 명확했기 때문에 에이전트가 구현 단계에서도 더 빠르게 움직인 것으로 보입니다.

</div>

| 항목 | 로컬 플랜 | Ultra Plan |
|------|-----------|-----------|
| 계획 완료 시간 | 15분 이상 | 약 2분 |
| 전체 구현 완료 | 45분+ | 10~15분 |
| 로컬 토큰 사용 | 131,000 토큰 | 82,000 토큰 |
| 결과물 품질 | 기능 완성 | 유사하게 완성 |

<div class="example-case">
<strong>실전 케이스: 스킬 활용이 필요한 경우</strong>

Ultra Plan이 클라우드에서 프로젝트 전체를 읽더라도, 특정 스킬(Claude Code에 추가하는 기능 모듈)을 자동으로 인식하지 못하는 경우가 있습니다. 예를 들어 시각화 스킬이 있는데 Ultra Plan이 대신 Mermaid 다이어그램을 사용했다면, 웹 검토 화면에서 댓글로 "이미 visualizations 스킬이 있으니 그걸 사용해"라고 지정하면 됩니다.

</div>

## Ultra Plan의 내부 구조

영상 제작자가 에이전트에게 소스코드·커뮤니티 포럼·공식 문서를 분석시킨 결과:

| 항목 | 로컬 플랜 | Ultra Plan |
|------|-----------|-----------|
| 실행 환경 | 내 터미널 | Anthropic 클라우드 컨테이너 |
| 사용 모델 | 현재 세션 모델 | Claude Opus 4.6 고정 |
| 플래닝 방식 | 단일 에이전트, 순차 처리 | 멀티 에이전트 (병렬 탐색 3개 + 비평 1개) |
| 최대 실행 시간 | 세션에 따라 다름 | 클라우드 30분 상한 |
| 터미널 차단 여부 | 플랜 중 터미널 블로킹 | 웹에서 기다리는 동안 터미널 계속 사용 가능 |
| 검토 인터페이스 | 터미널 텍스트 | 웹 문서 형태, 섹션별 댓글 가능 |

> 에이전트 수(3개 병렬 + 1개 비평)는 영상 제작자의 소스코드·포럼 분석 기준이며, Anthropic 공식 문서에는 내부 구성이 공개되어 있지 않습니다.

## 주의할 점

- **CLI 권장**: 공식적으로는 VS Code 확장과 데스크톱 앱도 지원하지만, 영상 제작 시점 테스트에서 일부 환경은 동작하지 않았습니다. 가장 안정적인 방법은 터미널 CLI입니다
- **Git 필수**: 프로젝트가 Git으로 연결되어 있지 않으면 "웹에서 사용할 수 없다"는 오류가 납니다. `git init && git push`로 간단히 해결됩니다
- **토큰 비용 주의**: Ultra Plan은 `ultra think`처럼 토큰을 더 많이 씁니다. 클라우드 플래닝 비용은 현재 별도 표시가 안 되어 정확한 수치 확인이 어렵습니다
- **Research Preview 단계**: 아직 연구 미리보기 단계로, 가끔 인증 오류가 발생할 수 있습니다. 재실행하면 정상 동작합니다
- **스킬 명시**: 특정 스킬을 활용하길 원한다면 처음부터 프롬프트에 스킬 이름을 명시하는 것이 좋습니다

## 정리

- Ultra Plan은 `/ultra plan` 또는 프롬프트에 "ultra plan" 키워드로 실행하며, 계획 작업을 Anthropic 클라우드로 오프로드해 로컬보다 훨씬 빠르게 구조화된 계획을 생성한다
- 웹에서 문서 형태로 계획을 검토하고 댓글을 달 수 있으며, 승인하면 터미널로 돌아와 바로 실행된다
- CLI + Pro/Max 구독 + Git 연결이 필수 조건이며, Research Preview 단계이므로 기능이 계속 개선될 예정이다
