# 비용 관리: Claude Code 사용 비용을 줄이는 방법

Claude Code는 대화할 때마다 "토큰"이라는 단위를 소비합니다. 토큰은 텍스트를 처리하는 단위로, 많이 쓸수록 비용이 늘어납니다. 이 문서는 비용을 파악하고 줄이는 방법을 설명합니다.

---

## 평균 비용은 얼마인가요?

| 구분 | 금액 |
|---|---|
| 개발자 1인 평균 일비용 | 약 $6 (약 8,500원) |
| 90% 사용자 일비용 상한 | $12 이하 |
| Sonnet 4.6 기준 개발자 1인 월비용 | 약 $100~200 |

> 동시에 여러 인스턴스를 실행하거나 자동화에 활용하면 비용이 올라갈 수 있습니다.

Pro, Max 구독자는 구독에 사용량이 포함되어 있으므로 토큰 비용이 별도 청구되지 않습니다.

---

## 현재 사용량 확인하기

### `/cost` 명령 (API 사용자)

```
Total cost:            $0.55
Total duration (API):  6m 19.7s
Total duration (wall): 6h 33m 10.2s
Total code changes:    0 lines added, 0 lines removed
```

이 명령은 현재 세션의 토큰 소비량을 보여줍니다. Pro/Max 구독자는 `/stats`로 사용 패턴을 확인하세요.

## 팀 비용 관리

### API 사용 시 지출 한도 설정

Claude Console에서 워크스페이스 단위로 지출 한도를 설정할 수 있습니다. 관리자는 사용량과 비용 보고서를 확인할 수 있습니다.

> Claude Code 첫 인증 시 "Claude Code"라는 워크스페이스가 자동 생성됩니다. 이 워크스페이스는 비용 추적 전용이며, API 키를 직접 생성할 수 없습니다.

### 팀 규모별 요청 한도 권장값

팀이 커질수록 동시 사용 비율이 낮아지므로, 인원당 할당량을 줄여 전체 한도를 효율적으로 운영할 수 있습니다.

| 팀 규모 | 인당 TPM (분당 토큰) | 인당 RPM (분당 요청) |
|---|---|---|
| 1~5명 | 200k~300k | 5~7 |
| 5~20명 | 100k~150k | 2.5~3.5 |
| 20~50명 | 50k~75k | 1.25~1.75 |
| 50~100명 | 25k~35k | 0.62~0.87 |
| 100~500명 | 15k~20k | 0.37~0.47 |
| 500명 이상 | 10k~15k | 0.25~0.35 |

예: 200명 팀이라면 인당 20k TPM × 200 = 총 400만 TPM을 요청하면 됩니다.

## 토큰 사용량을 줄이는 방법

토큰 비용은 처리하는 문맥(context)이 클수록 올라갑니다. 아래 방법으로 문맥을 작게 유지하세요.

### 1. 작업 사이에 대화 초기화

`/clear` 명령으로 이전 대화를 지우고 새 작업을 시작하세요. 이전 대화 내용이 불필요하게 남아있으면 매 요청마다 그 내용을 처리해야 해서 비용이 증가합니다.

> 이전 대화를 나중에 다시 보고 싶다면, `/clear` 전에 `/rename`으로 세션 이름을 붙여두세요.

### 2. 맞는 모델 선택하기

| 모델 | 적합한 상황 |
|---|---|
| Haiku | 간단한 하위 작업, 빠른 처리 |
| Sonnet | 대부분의 코딩 작업 (권장) |
| Opus | 복잡한 아키텍처 결정, 다단계 추론 |

`/model` 명령으로 세션 도중 모델을 바꿀 수 있습니다.

### 3. MCP 서버 수 줄이기

MCP 서버는 연결만 해도 도구 정의가 문맥에 포함됩니다. 사용하지 않는 서버는 `/mcp`에서 비활성화하세요.

CLI 도구(`gh`, `aws`, `gcloud`)를 사용할 수 있다면 MCP 서버보다 훨씬 효율적입니다.

### 4. CLAUDE.md를 간결하게 유지

CLAUDE.md 파일은 세션 시작 시 항상 문맥에 포함됩니다. 특정 작업에만 필요한 상세 지침은 [스킬(Skills)](./skills.md)로 분리하세요. 스킬은 호출될 때만 문맥에 로드됩니다. 목표: CLAUDE.md는 500줄 이하로 유지.

### 5. 구체적인 프롬프트 작성

- 나쁜 예: "이 코드베이스를 개선해줘" (넓은 파일 검색 유발)
- 좋은 예: "auth.ts의 login 함수에 입력값 검증을 추가해줘"

### 6. 복잡한 작업은 계획 먼저

`Shift+Tab`으로 계획 모드를 활성화하면 Claude가 먼저 접근 방법을 제안하고 승인을 구합니다. 잘못된 방향으로 오래 진행되어 토큰을 낭비하는 것을 막습니다.

### 7. 하위 에이전트에 분산 처리

테스트 실행, 로그 파일 분석 등 출력이 방대한 작업은 [하위 에이전트(Sub-agents)](./sub-agents.md)에게 위임하세요. 상세한 결과는 하위 에이전트 문맥에 남고, 요약만 메인 대화로 돌아옵니다.

### 8. 확장 사고(Extended Thinking) 조정

Opus 모델의 확장 사고는 기본으로 활성화되며 최대 31,999 토큰을 씁니다. 단순한 작업에는 사고 깊이를 줄이면 비용을 절약할 수 있습니다.

- `/model`에서 노력 수준 조정 (낮음/보통/높음)
- `MAX_THINKING_TOKENS=8000` 환경변수로 예산 제한

## 에이전트 팀 비용

에이전트 팀은 여러 Claude 인스턴스를 동시에 실행합니다. 팀원 수에 비례해 비용이 올라갑니다.

**에이전트 팀 비용 절약 팁:**

- 팀원에는 Sonnet 모델 사용
- 팀 크기를 최소로 유지
- 작업 완료 후 팀 즉시 해산 (대기 중에도 비용 발생)
- 팀원 시작 프롬프트를 간결하게 작성

---

## 백그라운드 토큰 사용

Claude Code는 아무것도 하지 않을 때도 약간의 토큰을 씁니다.

- 이전 대화 요약 (claude --resume 기능용)
- `/cost` 같은 상태 확인 명령

이 백그라운드 사용은 세션당 **$0.04 미만**입니다.

## 예시 케이스

### 상황 1: API를 쓰는 스타트업 팀 (10명)
인당 100k~150k TPM을 설정하고, 팀 전체 한도를 Console에서 지정해두세요. 팀원들이 작업 전환 시 `/clear`를 습관화하고 CLAUDE.md를 500줄 이하로 유지하면 월 비용을 30~40% 줄일 수 있습니다.

### 상황 2: 혼자서 여러 프로젝트를 관리하는 개발자
Pro 구독자라면 토큰 비용 걱정 없이 사용할 수 있습니다. 단, 응답 속도나 한도에 걸릴 경우 작업 규모가 큰 것부터 Opus → Sonnet 순서로 모델을 낮춰보세요.

### 상황 3: 자동화 파이프라인에 Claude Code를 사용하는 경우
`--max-budget-usd 5.00` 옵션으로 단일 실행 당 최대 비용을 제한하고, 하위 에이전트로 로그 분석 등 반복 작업을 분리하면 비용을 예측 가능하게 제어할 수 있습니다.
