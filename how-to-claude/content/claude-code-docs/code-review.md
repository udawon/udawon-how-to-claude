# 자동 코드 리뷰: PR을 자동으로 검토하기

> **참고**: 이 기능은 Teams 및 Enterprise 구독 전용입니다. Zero Data Retention(데이터 보존 없음) 정책을 사용하는 조직에서는 사용할 수 없습니다.

## 이게 뭔가요?

코드 리뷰(Code Review) 기능은 GitHub의 PR(Pull Request, 코드 변경 요청)이 열리거나 업데이트될 때 Claude가 자동으로 코드를 분석하고 문제를 찾아 댓글로 알려주는 서비스입니다.

마치 경험 많은 시니어 개발자 여러 명이 자동으로 코드를 검토해 주는 것과 같습니다. 인간 리뷰어가 미처 놓칠 수 있는 논리 오류, 보안 취약점, 숨겨진 버그 등을 찾아냅니다.

**중요한 점**: Claude의 리뷰는 PR을 승인하거나 차단하지 않습니다. 기존 리뷰 프로세스는 그대로 유지되면서 추가적인 검토 의견을 제공하는 방식입니다.

## 어떻게 작동하나요?

PR이 열리거나 업데이트되면 다음 과정이 진행됩니다.

1. **여러 전문 에이전트가 병렬 분석**: 각각 다른 종류의 문제를 담당하는 여러 Claude 에이전트가 동시에 코드를 검토합니다.
2. **전체 코드베이스 맥락 파악**: 변경된 코드만이 아니라 전체 코드와의 관계를 함께 분석합니다.
3. **거짓 양성 검증**: 발견된 문제가 실제 문제인지 한 번 더 검증하여 잘못된 경고를 줄입니다.
4. **결과 정리 및 게시**: 중복 제거 후 심각도 순으로 정렬해 해당 코드 라인에 직접 댓글로 게시합니다.

리뷰는 평균 20분 내에 완료됩니다. 문제가 발견되지 않으면 PR에 간단한 확인 댓글을 남깁니다.

## 심각도 분류

| 표시 | 심각도 | 의미 |
|:---:|---|---|
| 🔴 | 일반(Normal) | 병합 전에 반드시 수정해야 할 버그 |
| 🟡 | 사소함(Nit) | 작은 문제로 수정하면 좋지만 차단 수준은 아님 |
| 🟣 | 기존 문제(Pre-existing) | 이번 PR이 만든 것은 아니지만 코드베이스에 존재하는 버그 |

각 발견 항목에는 **접을 수 있는 상세 설명**이 포함되어 있어, Claude가 왜 이 문제를 발견했는지와 어떻게 검증했는지 확인할 수 있습니다.

## 기본적으로 무엇을 검사하나요?

기본 설정에서는 **코드의 정확성**에 집중합니다. 서비스를 중단시킬 수 있는 실제 버그를 찾는 것이 우선이며, 코드 형식이나 테스트 커버리지 등은 기본적으로 검사하지 않습니다.

추가로 검사할 항목은 `CLAUDE.md` 또는 `REVIEW.md` 파일로 커스터마이징할 수 있습니다.

## 설정 방법

### 1단계: Claude Code 관리자 설정 열기

[claude.ai/admin-settings/claude-code](https://claude.ai/admin-settings/claude-code)로 이동해 Code Review 섹션을 찾습니다. Claude 조직의 관리자 권한과 GitHub 조직에 앱을 설치할 권한이 필요합니다.

### 2단계: 설정 시작

**Setup** 버튼을 클릭합니다. GitHub App 설치 과정이 시작됩니다.

### 3단계: Claude GitHub App 설치

안내에 따라 Claude GitHub App을 GitHub 조직에 설치합니다. 아래 권한을 요청합니다.

| 권한 | 수준 |
|---|---|
| Contents (코드 내용) | 읽기 및 쓰기 |
| Issues (이슈) | 읽기 및 쓰기 |
| Pull requests (PR) | 읽기 및 쓰기 |

### 4단계: 저장소 선택

Code Review를 활성화할 저장소를 선택합니다. 목록에 저장소가 보이지 않으면 GitHub App 설치 시 해당 저장소 접근 권한을 부여했는지 확인하세요.

### 5단계: 리뷰 트리거 설정

각 저장소별로 언제 리뷰를 실행할지 설정합니다.

| 설정 | 설명 | 권장 상황 |
|---|---|---|
| PR 생성 시에만 | PR을 열거나 초안 상태에서 준비 완료로 변경할 때 한 번만 실행 | 비용을 아끼고 싶을 때 |
| 모든 푸시마다 | PR 브랜치에 커밋이 추가될 때마다 실행, 수정된 문제는 자동으로 해결 표시 | 지속적인 피드백이 필요할 때 |

처음 사용 시에는 "PR 생성 시에만"으로 시작하고, 필요하면 "모든 푸시마다"로 전환하는 것을 권장합니다.

## 예시 케이스

### 케이스 1: 출시 전 마지막 안전망

스타트업 개발팀이 새 결제 기능을 출시하기 직전입니다. 팀원들이 인간 리뷰를 했지만 혹시 놓친 게 있을까 걱정됩니다. Code Review가 자동으로 🔴 심각도 버그를 발견합니다: "특정 조건에서 금액 계산이 음수가 될 수 있습니다." 개발팀은 출시 전에 이 버그를 수정할 수 있습니다.

### 케이스 2: 보안 취약점 조기 발견

개발자가 사용자 입력을 데이터베이스 쿼리에 직접 넣는 코드를 작성했습니다. 이를 검토하는 인간 리뷰어가 미처 발견하지 못했지만, Code Review가 🔴 심각도로 "SQL 인젝션 취약점이 발생할 수 있습니다"라고 경고합니다.

### 케이스 3: 기존 버그 발견

이번 PR과 무관하지만 같은 파일에 오래된 버그가 있습니다. Code Review가 🟣 심각도로 표시해 팀이 별도 작업으로 수정할 수 있도록 알려줍니다.

## 리뷰 내용 커스터마이징

### CLAUDE.md 사용

저장소 루트에 `CLAUDE.md` 파일이 있으면, Code Review는 이 파일의 규칙을 기준으로 새로 위반된 항목을 사소함(Nit) 수준으로 표시합니다.

반대로 PR이 `CLAUDE.md` 내용을 구식으로 만들면, Claude가 문서 업데이트도 제안합니다.

폴더별로 다른 규칙을 적용하고 싶다면, 하위 폴더에도 `CLAUDE.md`를 만들면 해당 경로의 파일에만 적용됩니다.

### REVIEW.md 사용

리뷰 전용 규칙은 `REVIEW.md` 파일로 관리하세요. `CLAUDE.md`에 넣고 싶지 않은 리뷰 특화 지침을 여기에 작성합니다.

```markdown
# 코드 리뷰 가이드라인

## 항상 확인할 것
- 새 API 엔드포인트에는 통합 테스트가 있어야 함
- 데이터베이스 마이그레이션은 하위 호환성을 유지해야 함
- 에러 메시지에 내부 정보가 노출되지 않아야 함

## 코드 스타일
- 중첩 조건문보다 조기 반환(early return)을 선호
- 로그는 f-string이 아닌 구조화된 로깅 사용

## 건너뛸 것
- src/gen/ 아래의 자동 생성 파일
- .lock 파일의 형식 변경
```

`REVIEW.md`는 저장소 루트에 두면 Claude가 자동으로 인식합니다. 별도 설정이 필요없습니다.

## 사용량 확인

[claude.ai/analytics/code-review](https://claude.ai/analytics/code-review)에서 조직 전체의 Code Review 활동을 확인할 수 있습니다.

| 항목 | 내용 |
|---|---|
| 리뷰된 PR 수 | 기간별 리뷰 횟수 |
| 주간 비용 | Code Review 주간 지출 |
| 피드백 | 개발자가 문제를 수정해 자동으로 해결된 댓글 수 |
| 저장소별 현황 | 저장소별 리뷰 횟수 및 해결된 댓글 수 |

## 비용 안내

Code Review는 토큰 사용량 기준으로 청구됩니다. 평균 비용은 리뷰당 약 $15~25 수준이며, PR 크기와 코드베이스 복잡도에 따라 달라집니다.

월별 지출 한도를 설정하려면 [claude.ai/admin-settings/usage](https://claude.ai/admin-settings/usage)에서 Claude Code Review 서비스의 한도를 설정하세요.

**비용에 영향을 주는 요소:**
- PR이 크고 복잡할수록 비용이 증가합니다.
- "모든 푸시마다" 설정은 커밋 수만큼 비용이 배가됩니다.
- 문제 검증 단계에서 추가 토큰이 사용됩니다.
