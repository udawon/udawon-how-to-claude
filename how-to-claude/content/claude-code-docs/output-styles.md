# 출력 스타일 가이드: Claude의 대답 방식 바꾸기

출력 스타일(Output Styles)은 Claude가 대답하는 방식과 톤을 바꾸는 기능입니다. 같은 질문을 해도, 선택한 스타일에 따라 Claude가 전혀 다른 방식으로 대답합니다. 마치 같은 선생님이 상황에 따라 강의식으로, 토론식으로, 또는 실습식으로 가르치는 것처럼요.

---

## 기본 제공 스타일 3가지

| 스타일 | 설명 | 누구에게 적합? |
|--------|------|-------------|
| **Default** (기본) | 효율적인 소프트웨어 엔지니어링 도우미 | 일반 개발 작업 |
| **Explanatory** (설명형) | 코드 사이사이에 "Insights(인사이트)" 추가 | 코드베이스를 이해하고 싶을 때 |
| **Learning** (학습형) | 설명 + 직접 코딩 과제 부여 | 배우면서 성장하고 싶을 때 |

### Default: 기본 모드

빠르고 효율적으로 소프트웨어 개발을 돕는 모드입니다. 군더더기 없이 핵심만 답변합니다.

### Explanatory: 설명형 모드

코드 작업을 도와주면서, 왜 이렇게 했는지 "인사이트"를 함께 제공합니다.

예: 버튼 컴포넌트를 만들면서 → "이 컴포넌트가 왜 이렇게 설계됐는지", "이 패턴의 장단점은 무엇인지"를 함께 설명

### Learning: 학습형 모드

코드의 일부를 `TODO(human)` 마커로 남겨두고, 직접 채워보도록 유도합니다. 단순히 답을 주는 것이 아니라, 함께 배워나갑니다.

---

## 스타일 변경하기

### 방법 1: 명령어로 변경

```
/output-style explanatory
/output-style learning
/output-style default
```

### 방법 2: 메뉴에서 선택

```
/output-style
```

또는 `/config` 메뉴에서도 접근 가능합니다.

변경 사항은 로컬 프로젝트 설정(`.claude/settings.local.json`)에 저장됩니다.

## 커스텀 출력 스타일 만들기

나만의 스타일을 만들 수도 있습니다. 마크다운 파일에 지침을 작성하면 됩니다.

### 파일 형식

```markdown
---
name: 내 커스텀 스타일
description: 이 스타일이 무엇을 하는지 설명
keep-coding-instructions: false
---

# 커스텀 스타일 지침

여기에 Claude가 어떻게 대답해야 하는지 설명합니다.

## 구체적인 동작

- 항상 이렇게 해주세요
- 저런 것은 하지 마세요
```

### 저장 위치

| 범위 | 경로 |
|------|------|
| 개인 전체 | `~/.claude/output-styles/내스타일.md` |
| 특정 프로젝트 | `.claude/output-styles/내스타일.md` |

### frontmatter 옵션

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `name` | 스타일 이름 | 파일 이름 |
| `description` | 설명 (UI에 표시됨) | 없음 |
| `keep-coding-instructions` | 코딩 관련 기본 지침 유지 여부 | `false` |

---

## 출력 스타일 vs 다른 기능 비교

헷갈릴 수 있는 유사 기능들과의 차이:

| 기능 | 어떻게 다른가? |
|------|-------------|
| **출력 스타일** | Claude의 시스템 프롬프트 자체를 변경. 대화 내내 적용됨 |
| **CLAUDE.md** | 시스템 프롬프트에 추가 지침을 덧붙임. 기본 동작은 유지 |
| **`--append-system-prompt`** | 시스템 프롬프트 끝에 내용 추가. 기본 동작 유지 |
| **서브에이전트(Sub-agents)** | 특정 작업에만 투입되는 전문 에이전트 |
| **스킬(Skills)** | `/스킬이름`으로 호출하는 재사용 가능한 작업 단위 |

## 예시 케이스

**상황 1: 낯선 코드베이스를 처음 접하는 개발자**

> 새로운 회사에 입사한 이신입 씨는 복잡한 레거시 코드를 이해해야 합니다. **Explanatory** 모드로 전환하고 코드를 탐색했습니다. Claude가 코드를 수정하면서 "이 패턴은 ORM의 N+1 문제를 피하기 위해 이렇게 작성되었습니다"처럼 설계 의도를 함께 설명해줘서 빠르게 코드베이스를 이해했습니다.

**상황 2: 프로그래밍을 배우는 비개발자**

> 마케터 김마케 씨는 간단한 데이터 분석 스크립트를 직접 작성하고 싶습니다. **Learning** 모드로 Claude와 함께 공부합니다. Claude가 코드의 일부를 비워두고 `TODO(human): 여기서 필터링 조건을 추가해보세요`라고 남겨줍니다. 직접 코드를 채워넣으면서 배워갑니다.

**상황 3: 마케팅 콘텐츠 작성 에이전트로 활용**

> 콘텐츠 매니저 박콘텐츠 씨는 Claude Code를 코딩이 아닌 콘텐츠 작업에 사용하고 싶습니다. 커스텀 출력 스타일을 만들었습니다:
> ```markdown
> ---
> name: 콘텐츠 작성 도우미
> description: 마케팅 콘텐츠 작성에 최적화된 스타일
> keep-coding-instructions: false
> ---
> 당신은 마케팅 콘텐츠 전문가입니다.
> 항상 독자 친화적인 언어를 사용하고,
> 핵심 메시지를 먼저 제시한 후 상세 내용을 설명하세요.
> ```
> 이제 Claude Code가 마케팅 도우미로 동작합니다.

**상황 4: 코드 리뷰어로 활용**

> 팀장 최팀장 씨는 주니어 개발자들의 코드를 리뷰할 때 단순히 문제를 찾는 것이 아니라 성장을 돕고 싶습니다. **Explanatory** 모드를 사용해서 코드 수정과 함께 "왜 이렇게 바꿔야 하는지", "이 패턴의 장단점은 무엇인지" 설명이 담긴 리뷰를 자동으로 생성합니다.
