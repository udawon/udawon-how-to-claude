# Slack에서 Claude Code 사용하기

팀이 사용하는 Slack 채널에서 `@Claude`를 멘션하면 코딩 작업을 자동으로 감지해서 Claude Code 웹 세션을 생성합니다. 개발 도구로 이동하지 않고 팀 대화 공간에서 바로 코드 작업을 시작할 수 있습니다. 마치 개발팀 동료를 채널에 태그하면 그 사람이 알아서 작업하고 결과를 가져오는 것과 같습니다.

## 예시 케이스

> 고객 지원팀 정수아 씨가 버그를 발견하고 개발 채널에 보고합니다. 대화 스레드에서 `@Claude 이 에러 로그를 보고 로그인 실패 원인을 찾아서 고쳐줘`라고 입력합니다. Claude가 스레드의 모든 대화를 읽고 관련 저장소를 찾아 코드를 분석한 뒤, 수정 작업을 완료하면 `@정수아 수정 완료했습니다. PR 링크를 확인해주세요`라고 응답합니다.

---

## 사용 사례

| 사용 사례 | 설명 |
|-----------|------|
| **버그 조사 및 수정** | Slack에 보고된 버그를 즉시 수정 작업으로 전환 |
| **빠른 코드 리뷰** | 팀 피드백을 바탕으로 소규모 기능 구현 또는 리팩토링 |
| **협업 디버깅** | 슬랙 대화의 맥락(오류 재현, 사용자 리포트)을 활용한 디버깅 |
| **병렬 작업** | 여러 코딩 작업을 동시에 시작하고 완료 알림 수신 |

## 사전 준비사항

| 요구사항 | 세부 내용 |
|----------|-----------|
| Claude 플랜 | Pro, Max, Teams, Enterprise (프리미엄 시트) |
| Claude Code 웹 접근 | Claude Code on the web 활성화 필요 |
| GitHub 계정 | Claude Code on the web에 연결된 저장소 필요 |
| Slack 인증 | Slack 계정이 Claude 계정과 연결되어 있어야 함 |

## 설정 방법

### 1단계: Slack에 Claude 앱 설치

워크스페이스 관리자가 [Slack 앱 마켓플레이스](https://slack.com/marketplace/A08SF47R6P4)에서 Claude 앱을 설치합니다.

### 2단계: Claude 계정 연결

1. Slack의 앱 목록에서 **Claude** 클릭
2. **앱 홈(App Home)** 탭으로 이동
3. **연결(Connect)** 버튼 클릭
4. 브라우저에서 인증 완료

### 3단계: Claude Code on the web 설정

[claude.ai/code](https://claude.ai/code)에 접속해서:
- 동일한 계정으로 로그인
- GitHub 계정 연결
- 작업할 저장소 인증

### 4단계: 라우팅 모드 선택

Claude 앱 홈에서 **라우팅 모드**를 설정합니다:

| 모드 | 동작 방식 | 추천 상황 |
|------|-----------|-----------|
| **Code만** | 모든 @멘션을 Claude Code 세션으로 라우팅 | 개발 전용 채널 |
| **Code + Chat** | 메시지 내용을 분석해서 코딩 작업은 Claude Code, 일반 질문은 Claude Chat으로 자동 분류 | 개발 + 일반 질문 혼합 채널 |

> Code + Chat 모드에서 잘못 분류된 경우 "Code로 재시도" 또는 "Chat으로 재시도" 버튼이 제공됩니다.

### 5단계: 채널에 Claude 추가

Claude는 자동으로 채널에 추가되지 않습니다. 사용하고 싶은 채널에서:
```
/invite @Claude
```

Claude는 초대된 채널에서만 @멘션에 응답합니다.

## 작동 원리

### 자동 감지

`@Claude`를 멘션하면 Claude가 메시지를 분석해서 코딩 작업인지 판단합니다. 코딩 의도가 감지되면 자동으로 Claude Code on the web 세션을 생성합니다.

> Claude Code in Slack은 **채널**에서만 작동합니다. 다이렉트 메시지(DM)에서는 작동하지 않습니다.

### 맥락 수집

- **스레드에서 멘션**: 스레드의 모든 메시지를 맥락으로 활용
- **채널에서 멘션**: 최근 채널 메시지를 맥락으로 활용

이를 통해 Claude가 문제를 이해하고, 적합한 저장소를 선택하며, 작업 방향을 결정합니다.

> Claude는 대화 맥락에 있는 다른 메시지의 지시도 따를 수 있습니다. 신뢰할 수 있는 채널에서만 Claude를 사용하세요.

### 세션 흐름

```
1. @Claude 멘션 → 코딩 의도 감지
2. claude.ai/code에 새 세션 생성
3. 작업 진행 상황을 Slack 스레드에 업데이트
4. 완료 시 @멘션으로 요약 + 액션 버튼 제공
5. "세션 보기" 또는 "PR 생성" 클릭
```

## 사용자 인터페이스

### 앱 홈

Claude 계정 연결 상태 확인 및 연결/해제 관리

### 메시지 액션 버튼

| 버튼 | 기능 |
|------|------|
| **세션 보기** | 브라우저에서 전체 Claude Code 세션 열기 |
| **PR 생성** | 변경사항으로 바로 풀 리퀘스트 생성 |
| **Code로 재시도** | Chat으로 분류된 응답을 Code 세션으로 재시도 |
| **저장소 변경** | Claude가 잘못 선택한 저장소 변경 |

## 접근 권한 관리

### 사용자 수준

| 항목 | 내용 |
|------|------|
| 세션 실행 | 각 사용자의 Claude 계정으로 실행 |
| 사용량 제한 | 개인 플랜 사용량에 포함 |
| 저장소 접근 | 본인이 연결한 저장소만 접근 가능 |
| 세션 기록 | claude.ai/code에서 확인 가능 |

### 워크스페이스 수준

| 항목 | 내용 |
|------|------|
| 앱 설치 | 워크스페이스 관리자가 결정 |
| Enterprise Grid | 조직 관리자가 워크스페이스별 접근 제어 |
| 앱 제거 | 즉시 모든 사용자 접근 차단 |

### 채널 기반 접근 제어

- Claude는 초대된 채널에서만 응답합니다.
- 관리자는 특정 채널에만 Claude를 초대해서 사용 범위를 제한할 수 있습니다.
- 퍼블릭 및 프라이빗 채널 모두 지원합니다.

## 효과적인 요청 작성 방법

| 팁 | 예시 |
|----|------|
| **구체적으로** | "버그 고쳐줘" → "src/auth/login.ts 132번째 줄의 NullPointerException 수정해줘" |
| **맥락 제공** | 저장소나 프로젝트 이름 언급 |
| **완료 기준 명시** | "테스트도 작성해줘", "문서도 업데이트해줘" |
| **스레드 활용** | 버그 논의 스레드에서 직접 Claude를 멘션 |

## Slack vs. 웹 직접 사용

| Slack 사용 | 웹 직접 사용 |
|------------|--------------|
| 이미 Slack에서 논의 중인 내용 | 파일 업로드가 필요할 때 |
| 비동기로 작업을 시작하고 싶을 때 | 실시간으로 개발하면서 대화할 때 |
| 팀원들이 진행 상황을 볼 수 있어야 할 때 | 복잡하고 긴 작업을 진행할 때 |

---

## 문제 해결

| 증상 | 해결 방법 |
|------|-----------|
| 세션이 시작되지 않음 | 앱 홈에서 Claude 계정 연결 확인, Claude Code on the web 접근 권한 확인 |
| 저장소가 표시되지 않음 | claude.ai/code에서 저장소 연결, GitHub 권한 확인, GitHub 계정 재연결 |
| 잘못된 저장소 선택 | "저장소 변경" 버튼 클릭, 요청에 저장소 이름 직접 명시 |
| 인증 오류 | 앱 홈에서 Claude 계정 재연결, 올바른 Claude 계정인지 확인 |

## 현재 제한사항

| 제한사항 | 내용 |
|----------|------|
| 지원 플랫폼 | GitHub만 지원 |
| PR | 세션당 하나의 PR만 생성 가능 |
| 사용량 제한 | 개인 Claude 플랜의 사용량 제한 적용 |
| 웹 접근 필요 | Claude Code on the web 접근 권한이 없는 사용자는 일반 Claude Chat 응답만 받음 |
| DM 미지원 | 다이렉트 메시지에서는 작동하지 않음 |
