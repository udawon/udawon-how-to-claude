Claude Code는 내부적으로 여러 도구를 사용합니다.
어떤 도구가 있는지 알면, 원하는 도구를 직접 요청해서 더 정확한 결과를 얻을 수 있습니다.

---

## AskUserQuestion — 모호할 때 먼저 물어보게 하기

Claude가 판단하기 애매한 상황에서 추측으로 진행하는 대신 사용자에게 먼저 질문하도록 유도합니다.

**예시 케이스:**
```
(X) "버튼 디자인 바꿔줘"
-> Claude가 임의로 색상/크기/모양을 결정해서 원하지 않는 결과가 나올 수 있음

(O) "버튼 디자인을 바꾸고 싶어. 시작하기 전에 먼저 어떤 버튼인지,
     어떻게 바꿀지 나한테 물어봐줘"
-> Claude가 AskUserQuestion으로 구체적인 의도를 확인하고 작업 시작

비유: 미용사가 "어떻게 잘라드릴까요?"를 먼저 묻는 것
     물어보지 않고 알아서 자르면 결과가 달라질 수 있음
```

---

## Read — 기억 대신 직접 확인하게 하기

Claude가 기억에 의존해 답하는 대신 파일을 직접 열어 확인하게 합니다.
할루시네이션 방지에도 효과적입니다.

**예시 케이스:**
```
(X) "현재 메인 페이지 코드가 어때?"
-> Claude가 이전 대화 기억으로 답변. 실제 파일과 다를 수 있음

(O) "메인 페이지 파일을 직접 읽고 나서 어떤 구조인지 설명해줘"
-> Claude가 파일을 실제로 열어 현재 상태 기준으로 설명

비유: "지난번 계약서 내용이 뭐였지?"보다
     "계약서를 꺼내서 직접 읽어줘"가 정확한 것과 같음
```

---

## Grep — 프로젝트 전체에서 단어 찾기

파일 수백 개 중에서 특정 단어나 코드가 어디에 있는지 빠르게 찾습니다.

**예시 케이스:**
```
"이 프로젝트에서 '로그인'이라는 단어가 들어간 파일을 모두 찾아줘"
-> 전체 파일을 뒤져서 해당 단어가 있는 위치를 모두 알려줌

"useState를 사용하는 컴포넌트가 어디어디 있어?"
-> 모든 파일에서 useState를 검색해서 목록 제공

비유: 책 뒤의 색인(Index)에서 특정 단어가 나오는 페이지를 모두 찾는 것
```

---

## Glob — 파일 이름 패턴으로 찾기

"이미지 파일만", "TypeScript 파일만" 처럼 이름 패턴으로 파일을 모아서 찾습니다.

**예시 케이스:**
```
"이 프로젝트의 모든 페이지 파일이 어디에 있는지 알려줘"
-> page.tsx 파일들의 위치를 모두 찾아서 구조를 파악

"설정 파일들이 뭐가 있어?"
-> .config, .json 파일들을 찾아서 목록화

비유: 파일탐색기에서 "*.jpg"로 검색하면 사진 파일만 나오는 것
```

## Bash — 직접 명령어 실행

터미널에 입력해야 하는 명령어를 Claude가 대신 실행해줍니다.
패키지 설치, 서버 실행, git 명령어 등에 사용됩니다.

**예시 케이스:**
```
"현재 설치된 패키지 목록을 보여줘"
-> npm list 명령어를 실행해서 결과를 보여줌

"git에 커밋하고 푸시해줘"
-> git add, commit, push를 순서대로 실행

"빌드해보고 오류가 있으면 알려줘"
-> npm run build를 실행하고 결과를 분석

비유: 비서가 대신 전화를 걸거나 서류를 팩스로 보내주는 것
```

## WebSearch — 최신 정보 검색

Claude의 학습 데이터에 없는 최신 정보나 공식 문서를 검색합니다.

**예시 케이스:**
```
"Next.js 16의 새로운 기능이 뭐야?"
-> 웹을 검색해서 최신 공식 문서 기반으로 답변
   (기억에만 의존하면 구버전 정보를 말할 수 있음)

"이 라이브러리 최신 버전이 뭐야?"
-> 실시간으로 검색해서 정확한 버전 정보 제공

비유: 오래된 교과서로 공부하는 것 vs. 최신 뉴스를 직접 검색하는 것
```

---

## WebFetch — 특정 URL 내용 가져오기

특정 웹페이지나 공식 문서의 내용을 직접 읽어와서 분석합니다.

**예시 케이스:**
```
"https://docs.example.com/api 를 읽고 이 API 사용법을 알려줘"
-> 해당 페이지 내용을 가져와서 요약하고 설명

"이 에러 메시지를 Stack Overflow에서 찾아봐줘"
-> 관련 페이지를 열어서 해결책을 찾아줌

비유: "이 책 사이트에 가서 목차를 읽어와줘"라고 심부름을 시키는 것
```

## Agent — 복잡한 탐색 작업 위임

대규모 파일 탐색이나 복잡한 분석 작업을 별도 에이전트에게 맡깁니다.
메인 대화 흐름을 방해하지 않고 독립적으로 처리됩니다.

**예시 케이스:**
```
"이 프로젝트 전체 구조를 파악하고 어떤 파일이 어떤 역할인지 정리해줘"
-> 에이전트가 모든 파일을 탐색하고 정리해서 요약만 전달

"버그가 어디서 발생하는지 전체 코드를 뒤져서 찾아줘"
-> 에이전트가 광범위하게 코드를 조사하고 원인 후보를 리포트

비유: "이 빌딩 전체를 돌면서 비상구가 몇 개인지 세어서 알려줘"처럼
     사람을 보내서 보고를 받는 것
```

## TodoWrite — 작업 목록 관리

Claude가 여러 단계의 작업을 처리할 때 할 일 목록을 만들고 추적하게 합니다.
긴 작업에서 어디까지 했는지 잃어버리지 않습니다.

**예시 케이스:**
```
"이번 작업에서 뭘 해야 하는지 할 일 목록을 먼저 만들고,
 하나씩 체크하면서 진행해줘"

-> Claude가 작업 목록을 만들고 완료할 때마다 체크
   작업이 길어져도 현재 어느 단계인지 파악 가능

비유: 이삿짐 작업에서 체크리스트를 만들고
     하나씩 완료할 때마다 체크하는 것
```

## 도구를 직접 지정하는 방법

Claude는 보통 상황에 맞게 알아서 도구를 선택하지만, 원하는 도구를 명시적으로 요청할 수 있습니다.

```
"파일을 직접 읽어서 확인해줘" → Read 사용 유도
"전체 프로젝트에서 검색해줘" → Grep/Glob 사용 유도
"웹에서 찾아봐줘" → WebSearch 사용 유도
"실행해서 결과를 보여줘" → Bash 사용 유도
"먼저 물어봐줘" → AskUserQuestion 사용 유도
```

> 도구 이름을 직접 말하지 않아도 됩니다.
> "확인해줘", "검색해줘", "실행해줘" 처럼 행동을 설명하면
> Claude가 알맞은 도구를 선택합니다.
