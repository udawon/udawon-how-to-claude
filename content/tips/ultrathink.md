## ultrathink란?

프롬프트에 `ultrathink`라는 단어를 포함하면 Claude가 해당 요청에 대해 **최고 수준의 추론 모드**로 동작합니다. 설정 변경 없이 단어 하나만 추가하면 됩니다.

**예시 케이스:**

<div class="example-case">

(X) "이 인증 시스템 어떻게 리팩토링할까?"
→ 빠른 답변은 나오지만 깊이가 부족할 수 있음

(O) "<span class="keyword-highlight">ultrathink</span>를 활용해서 이 인증 시스템 리팩토링 방법을 설계해줘"
→ Claude가 트레이드오프, 엣지케이스, 대안들을 깊이 분석한 뒤 답변

비유: 급하게 아무 레스토랑 vs 충분히 고민해서 오늘 기분에 딱 맞는 곳
     시간을 더 쓰는 만큼 결과 품질이 다름
</div>

## 언제 사용하면 효과적인가?

### 1. 복잡한 아키텍처 결정

"<span class="keyword-highlight">ultrathink</span> - 우리 서비스가 트래픽이 10배 증가했을 때
 현재 구조의 어디가 먼저 무너질지 분석해줘.
 그리고 지금 당장 대비할 수 있는 방법을 알려줘."

→ 단순히 "캐시를 쓰세요"가 아니라
   병목 지점, 우선순위, 비용 대비 효과까지 종합 분석

### 2. 원인 불명의 버그

"<span class="keyword-highlight">ultrathink</span> - 로그인은 되는데 새로고침하면 로그아웃되는 버그야.
 세션 저장 방식, 토큰 만료, 쿠키 설정 등 가능한 원인을
 체계적으로 검토해줘."

→ 여러 가설을 세우고 각각 검증하는 체계적 접근

### 3. 여러 구현 방식 비교

"<span class="keyword-highlight">ultrathink</span> - 실시간 알림 기능을 WebSocket, SSE, Polling 중 어떤 방식으로
 구현할지 결정해야 해. 우리 서비스 특성(접속자 500명, 모바일 중심)에
 맞는 방식을 골라줘."

→ 각 방식의 장단점을 우리 상황에 맞게 비교 분석

### 4. 보안 검토

"<span class="keyword-highlight">ultrathink</span> - 이 API 엔드포인트의 보안 취약점을 최대한 꼼꼼하게
 검토해줘. 악의적인 사용자가 어떤 방식으로 공격할 수 있을지
 시나리오별로 분석해줘."

## 확장 사고 모드 토글

프롬프트에 키워드를 넣는 것 외에, 단축키로 확장 사고 모드를 켜고 끌 수 있습니다.

| 단축키 | 설명 |
|--------|------|
| <kbd>Alt</kbd>+<kbd>T</kbd> (Windows/Linux) | 확장 사고 모드 켜기/끄기 |
| <kbd>Option</kbd>+<kbd>T</kbd> (macOS) | 확장 사고 모드 켜기/끄기 |

활성화하면 모든 요청에 대해 Claude가 깊이 사고합니다.
특정 요청에서만 깊은 사고가 필요하면 프롬프트에 키워드를 넣는 방식이 더 효율적입니다.

## 사고 깊이 키워드

`ultrathink` 외에도 단계별 키워드가 있습니다:

| 키워드 | 사고 깊이 |
|--------|----------|
| `think` | 기본 확장 사고 |
| `think hard` | 더 깊은 사고 |
| `think harder` | 매우 깊은 사고 |
| `ultrathink` | 최대 수준 사고 |

## 도구 사용 과정 보기 (<kbd>Ctrl</kbd>+<kbd>O</kbd>)

<kbd>Ctrl</kbd>+<kbd>O</kbd>는 **상세 출력 토글**입니다. Claude가 어떤 도구를 어떻게 사용하는지 과정을 볼 수 있습니다.

> 주의: <kbd>Ctrl</kbd>+<kbd>O</kbd>는 사고 과정이 아닌 **도구 사용 과정**을 보여줍니다. 확장 사고 모드는 <kbd>Alt</kbd>+<kbd>T</kbd>로 토글합니다.

## ultrathink vs 일반 요청 비교

| 상황 | 일반 요청 | ultrathink |
|------|-----------|------------|
| 아키텍처 설계 | 일반적인 패턴 제안 | 서비스 특성에 맞춘 심층 분석 |
| 버그 디버깅 | 명백한 원인 찾기 | 다양한 가설 체계적 검증 |
| 기술 선택 | 대세 기술 추천 | 상황별 트레이드오프 비교 |
| 코드 리뷰 | 명백한 문제 지적 | 숨겨진 엣지케이스까지 발굴 |

## 주의사항

- 응답 시간이 더 길어질 수 있음 (깊이 있는 사고 = 더 많은 토큰 사용)
- 간단한 작업에는 필요 없음
  예: 변수명 바꾸기, 주석 추가, 오타 수정
- 적절한 사용 예:
  복잡한 기능 설계, 어려운 버그, 중요한 기술 결정
- 불필요한 사용 예:
  "버튼 색깔 바꿔줘", "오타 고쳐줘"
