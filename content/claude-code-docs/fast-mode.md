# 빠른 모드 가이드: Opus를 2.5배 빠르게 사용하기

> **현재 연구 미리보기(Research Preview) 단계입니다.** 기능, 가격, 이용 가능 여부가 변경될 수 있습니다.

빠른 모드(Fast Mode)는 Claude Opus 4.6을 **2.5배 빠르게** 실행하는 특별 설정입니다. 마치 고속도로 요금소에서 하이패스를 쓰는 것처럼, 추가 비용을 내는 대신 더 빠른 응답을 받을 수 있습니다.

> 빠른 모드는 **다른 모델이 아닙니다.** 같은 Opus 4.6을, 빠른 설정으로 실행하는 것입니다. 품질과 기능은 동일합니다.

---

## 빠른 모드 켜고 끄기

터미널에서:
```
/fast
```

Tab 키를 누르면 켜짐/꺼짐이 전환됩니다.

설정 파일로 기본 활성화:
```json
{
  "fastMode": true
}
```

켜지면 프롬프트 옆에 `↯` 아이콘이 표시됩니다.

---

## 비용 비교

| 모드 | 입력 (백만 토큰당) | 출력 (백만 토큰당) |
|------|-----------------|-----------------|
| 일반 Opus 4.6 (200K 이하) | $15 | $75 |
| **빠른 모드 Opus 4.6 (200K 이하)** | **$30** | **$150** |
| 빠른 모드 Opus 4.6 (200K 초과) | $60 | $225 |

> **팁:** 대화 중간에 빠른 모드를 켜면 이전 전체 대화 내용에 대해서도 빠른 모드 요금이 청구됩니다. 처음부터 켜는 것이 경제적입니다.

---

## 빠른 모드 vs 노력 수준 비교

두 설정 모두 응답 속도에 영향을 주지만, 방식이 다릅니다.

| 설정 | 효과 | 품질 영향 |
|------|------|---------|
| **빠른 모드** | 같은 품질, 낮은 지연시간, 높은 비용 | 품질 동일 |
| **낮은 노력 수준** | 적게 생각, 빠른 응답, 비용 절약 | 복잡한 작업에서 품질 저하 가능 |

두 가지를 함께 사용할 수도 있습니다: 빠른 모드 + 낮은 노력 수준 = 최대 속도

---

## 사용 조건

빠른 모드를 사용하려면 다음 조건이 모두 충족되어야 합니다.

| 조건 | 설명 |
|------|------|
| 구독 | Pro, Max, Team, Enterprise, 또는 Console |
| 추가 사용량(Extra Usage) | 계정에서 활성화되어 있어야 함 |
| 클라우드 제공자 | Amazon Bedrock, Google Vertex AI, Microsoft Azure에서는 **사용 불가** |
| 팀/기업 계정 | 관리자가 먼저 활성화해야 함 |

> **참고:** 빠른 모드는 기본 구독 사용량이 아닌, 추가 사용량으로 별도 청구됩니다.

---

## 팀/기업 관리자 설정

**빠른 모드 활성화:**
- API 고객: Claude Code 기본 설정에서 활성화
- Teams/Enterprise: 관리자 설정 > Claude Code에서 활성화

**세션마다 다시 켜도록 강제하기:**

직원들이 실수로 빠른 모드를 계속 켜두는 것을 방지하려면, 관리 설정에 추가:

```json
{
  "fastModePerSessionOptIn": true
}
```

이렇게 하면 매 세션 시작 시 빠른 모드가 꺼진 상태로 시작됩니다. 필요할 때만 `/fast`로 켤 수 있습니다.

---

## 속도 제한에 걸렸을 때

빠른 모드 속도 제한에 걸리면:
1. 자동으로 일반 Opus 4.6으로 전환됩니다
2. `↯` 아이콘이 회색으로 변합니다
3. 제한이 풀리면 자동으로 빠른 모드로 돌아옵니다

직접 끄려면 다시 `/fast`를 실행하세요.

---

## 예시 케이스

**상황 1: 실시간 디버깅 세션**

> 서비스 장애가 발생했습니다! 운영자 오운영 씨는 빠르게 원인을 찾아야 합니다. `/fast`를 입력해 빠른 모드를 켰습니다. Opus가 2.5배 빠르게 응답해주면서 로그를 분석하고 원인을 빠르게 파악했습니다. 장애 상황이 끝나자 `/fast`로 다시 껐습니다.

**상황 2: 빠른 반복 개발 (Rapid Iteration)**

> UI 개발자 유UI 씨는 디자인 변경 사항을 빠르게 적용하면서 결과를 바로바로 확인해야 합니다. 빠른 모드를 켜두면 Claude의 응답을 기다리는 시간이 크게 줄어들어 작업 흐름이 훨씬 좋아졌습니다.

**상황 3: 장시간 자동화 작업에서는 끄기**

> 자동화 스크립트 담당자 자자동 씨는 밤새 CI/CD 파이프라인을 돌립니다. 이런 장시간 자동화 작업은 속도보다 비용이 중요합니다. 빠른 모드를 끄고 일반 모드로 실행하여 비용을 절약합니다.

**상황 4: 팀에서 비용 관리**

> 팀장 최팀장은 팀원들이 빠른 모드를 항상 켜두어 비용이 급증하는 것을 발견했습니다. `fastModePerSessionOptIn: true`를 관리 설정에 적용하여, 이제 각 세션이 시작할 때마다 빠른 모드가 꺼져 있도록 했습니다. 팀원들은 정말 필요할 때만 의식적으로 켜게 됩니다.
