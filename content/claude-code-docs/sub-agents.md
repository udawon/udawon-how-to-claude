# 서브 에이전트 (Sub-agents)

> 서브 에이전트를 비유하자면, 회사에서 특정 업무를 전담하는 팀원에게 일을 위임하는 것과 같습니다. "코드 리뷰는 코드 리뷰 담당에게, 데이터 분석은 데이터 분석 담당에게" 역할을 분리합니다.

---

## 서브 에이전트란?

서브 에이전트는 특정 유형의 작업을 처리하도록 설계된 전문화된 AI 조수입니다. 각 서브 에이전트는:

- 자체 대화 창(컨텍스트 윈도우)에서 독립적으로 실행
- 맞춤화된 시스템 프롬프트로 동작
- 지정된 도구만 사용 가능
- 독립적인 권한 체계 보유

Claude가 서브 에이전트의 설명과 일치하는 작업을 만나면 자동으로 해당 에이전트에게 위임합니다.

---

## 서브 에이전트를 사용하면 좋은 점

| 장점 | 설명 |
|------|------|
| **컨텍스트 보존** | 탐색과 구현을 별도 창에서 실행해 메인 대화가 깔끔하게 유지됨 |
| **제약 강화** | 특정 에이전트가 사용할 수 있는 도구를 제한 |
| **재사용성** | 사용자 레벨 에이전트로 모든 프로젝트에서 활용 |
| **전문화** | 특정 도메인에 집중된 시스템 프롬프트 |
| **비용 절감** | 빠르고 저렴한 모델(Haiku 등)로 간단한 작업 라우팅 |

---

## 기본 내장 서브 에이전트

Claude Code에는 Claude가 자동으로 사용하는 기본 에이전트들이 있습니다.

| 에이전트 | 모델 | 도구 | 목적 |
|---------|------|------|------|
| **Explore** | Haiku (빠름) | 읽기 전용 | 코드베이스 탐색 및 분석 |
| **Plan** | 메인과 동일 | 읽기 전용 | 계획 모드 시 컨텍스트 수집 |
| **General-purpose** | 메인과 동일 | 모든 도구 | 복잡한 다단계 작업 |

---

## 예시 케이스

> **상황**: 풀스택 개발자 태양씨는 Claude와 대화하다가 "테스트 스위트를 실행하고 실패한 테스트를 고쳐줘"라고 요청했습니다. 테스트 실행 결과가 수천 줄에 달해 메인 대화창이 꽉 찼고, Claude가 이전 맥락을 잊어버리는 현상이 발생했습니다. 서브 에이전트를 사용하면 테스트 실행의 상세 로그는 에이전트의 별도 창에서 처리하고, 메인 대화에는 "실패한 테스트 3개, 원인은 X, 수정 완료"라는 요약만 돌아옵니다.

## 첫 번째 서브 에이전트 만들기

### `/agents` 명령어로 만들기 (권장)

1. Claude Code에서 실행:
   ```
   /agents
   ```

2. **Create new agent** → **User-level** 선택 (모든 프로젝트에서 사용)

3. **Generate with Claude** 선택 후 원하는 에이전트 설명:
   ```
   코드를 스캔하고 가독성, 성능, 모범 사례 개선점을 제안하는 코드 개선 에이전트.
   각 문제를 설명하고, 현재 코드를 보여주고, 개선된 버전을 제공해야 함.
   ```

4. 도구 선택: 읽기 전용 검토 에이전트라면 **Read-only tools**만 선택

5. 모델 선택: Sonnet (코드 패턴 분석에 균형 잡힌 선택)

6. 색상 선택 후 저장

7. 바로 사용 가능:
   ```
   code-improver 에이전트를 사용해 이 프로젝트의 개선점을 제안해줘
   ```

### 파일로 직접 만들기

`~/.claude/agents/` (모든 프로젝트용) 또는 `.claude/agents/` (현재 프로젝트용) 폴더에 마크다운 파일을 만듭니다.

**파일 형식**:

```markdown
---
name: code-reviewer
description: 코드 품질과 모범 사례를 검토합니다
tools: Read, Glob, Grep
model: sonnet
---

당신은 코드 품질과 보안의 높은 기준을 유지하는 시니어 코드 리뷰어입니다.
```

## 에이전트 설정 항목 전체 정리

| 항목 | 필수 | 설명 |
|------|------|------|
| `name` | 필수 | 고유 식별자 (소문자, 하이픈 사용) |
| `description` | 필수 | Claude가 언제 이 에이전트를 사용할지 판단하는 기준 |
| `tools` | 선택 | 사용 가능한 도구 목록. 생략 시 모든 도구 상속 |
| `disallowedTools` | 선택 | 사용 금지 도구 목록 |
| `model` | 선택 | `sonnet`, `opus`, `haiku`, `inherit` (기본값: `inherit`) |
| `permissionMode` | 선택 | 권한 처리 방식 |
| `maxTurns` | 선택 | 최대 대화 턴 수 |
| `skills` | 선택 | 시작 시 컨텍스트에 주입할 스킬 목록 |
| `mcpServers` | 선택 | 이 에이전트에서 사용할 MCP 서버 |
| `hooks` | 선택 | 에이전트 생명주기 훅 |
| `memory` | 선택 | 대화 간 지속 기억: `user`, `project`, `local` |
| `background` | 선택 | `true` 설정 시 항상 백그라운드로 실행 |
| `isolation` | 선택 | `worktree` 설정 시 격리된 git worktree에서 실행 |

## 에이전트 저장 위치와 우선순위

| 위치 | 적용 범위 | 우선순위 |
|------|---------|---------|
| `--agents` CLI 플래그 | 현재 세션 | 1 (가장 높음) |
| `.claude/agents/` | 현재 프로젝트 | 2 |
| `~/.claude/agents/` | 모든 프로젝트 | 3 |
| 플러그인 `agents/` 폴더 | 플러그인 활성화된 곳 | 4 (가장 낮음) |

## 도구 제어 방법

### 허용 도구 목록 (allowlist)

```yaml
---
name: safe-researcher
description: 제한된 기능으로 조사하는 에이전트
tools: Read, Grep, Glob, Bash
---
```

### 금지 도구 목록 (denylist)

```yaml
---
name: read-only-agent
tools: Read, Grep, Glob, Bash
disallowedTools: Write, Edit
---
```

### 권한 모드

| 모드 | 동작 |
|------|------|
| `default` | 표준 권한 확인 및 승인 요청 |
| `acceptEdits` | 파일 편집 자동 승인 |
| `dontAsk` | 권한 요청 자동 거부 |
| `bypassPermissions` | 모든 권한 확인 생략 |
| `plan` | 계획 모드 (읽기 전용 탐색) |

## 지속 기억 (Persistent Memory)

에이전트가 여러 대화에 걸쳐 학습한 내용을 기억하게 할 수 있습니다.

```yaml
---
name: code-reviewer
description: 코드 품질과 모범 사례를 검토합니다
memory: user
---

코드를 검토할 때 발견하는 패턴, 관례, 반복되는 문제를 에이전트 메모리에 업데이트하세요.
```

| 범위 | 저장 위치 | 언제 사용? |
|------|---------|---------|
| `user` | `~/.claude/agent-memory/<name>/` | 모든 프로젝트에서 학습 내용 공유 |
| `project` | `.claude/agent-memory/<name>/` | 프로젝트별 지식 (버전 관리 포함) |
| `local` | `.claude/agent-memory-local/<name>/` | 프로젝트별 지식 (버전 관리 제외) |

**활용 팁**:
- 작업 전: "이전 메모리를 확인하고 시작해줘"
- 작업 후: "학습한 내용을 메모리에 저장해줘"
- 에이전트 파일에 메모리 관리 지시를 포함시키면 자동으로 관리됨

## 서브 에이전트와 메인 대화 구분

| 상황 | 추천 방식 |
|------|---------|
| 빈번한 주고받기, 반복적 수정이 필요한 작업 | 메인 대화 |
| 대용량 출력이 예상되는 작업 | 서브 에이전트 |
| 특정 도구만 사용해야 하는 작업 | 서브 에이전트 |
| 독립적으로 완결되는 작업 | 서브 에이전트 |
| 빠른 한 번의 변경 | 메인 대화 |

## 실용적인 에이전트 예시

### 코드 리뷰어 (읽기 전용)

```markdown
---
name: code-reviewer
description: 전문 코드 리뷰어. 코드 품질, 보안, 유지보수성을 검토합니다. 코드 작성 또는 수정 직후 즉시 사용하세요.
tools: Read, Grep, Glob, Bash
model: inherit
---

당신은 코드 품질과 보안의 높은 기준을 유지하는 시니어 코드 리뷰어입니다.

호출 시:
1. git diff로 최근 변경사항 확인
2. 수정된 파일에 집중
3. 즉시 리뷰 시작

리뷰 체크리스트:
- 코드가 명확하고 읽기 쉬운지
- 함수명과 변수명이 적절한지
- 코드 중복이 없는지
- 오류 처리가 적절한지
- 보안 취약점이 없는지

우선순위별 피드백:
- 중요 이슈 (반드시 수정)
- 경고 (수정 권장)
- 제안 (개선 고려)
```

### 디버거

```markdown
---
name: debugger
description: 오류, 테스트 실패, 예상치 못한 동작의 디버깅 전문가. 문제 발생 시 즉시 사용하세요.
tools: Read, Edit, Bash, Grep, Glob
---

당신은 근본 원인 분석을 전문으로 하는 디버깅 전문가입니다.

호출 시:
1. 오류 메시지와 스택 트레이스 캡처
2. 재현 단계 확인
3. 실패 위치 격리
4. 최소한의 수정 구현
5. 해결책 검증

각 이슈에 대해:
- 근본 원인 설명
- 진단을 뒷받침하는 증거
- 구체적인 코드 수정
- 테스트 방법
- 예방 권장사항
```

### 데이터 과학자

```markdown
---
name: data-scientist
description: SQL 쿼리, BigQuery 작업, 데이터 인사이트를 위한 데이터 분석 전문가. 데이터 분석 작업 시 사용하세요.
tools: Bash, Read, Write
model: sonnet
---

당신은 SQL과 BigQuery 분석을 전문으로 하는 데이터 과학자입니다.

각 분석에 대해:
- 쿼리 접근 방법 설명
- 가정 사항 문서화
- 주요 발견 강조
- 데이터 기반 다음 단계 제안
```

## 에이전트 비활성화

특정 에이전트를 사용하지 못하도록 설정에서 차단할 수 있습니다:

```json
{
  "permissions": {
    "deny": ["Agent(Explore)", "Agent(my-custom-agent)"]
  }
}
```

또는 CLI 플래그로:

```bash
claude --disallowedTools "Agent(Explore)"
```
