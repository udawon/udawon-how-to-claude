# 서버 관리 설정 — 조직 전체 Claude Code 정책 중앙 관리

서버 관리 설정은 IT 부서나 관리자가 모든 팀원의 Claude Code 동작을 중앙에서 제어할 수 있게 해주는 기능입니다. 마치 회사 그룹웨어의 관리자 설정처럼, 한 곳에서 바꾸면 모든 직원의 Claude Code에 자동으로 적용됩니다.

이 기능은 MDM(모바일 기기 관리) 같은 별도 기기 관리 인프라 없이도 정책을 배포할 수 있어, 개인 노트북 등 관리되지 않는 기기에서도 작동합니다.

> **현재 상태**: 공개 베타 — Claude for Teams 및 Claude for Enterprise 고객 대상

---

## 이런 상황에 필요합니다

- MDM 없이도 전 직원의 Claude Code 설정을 통일하고 싶을 때
- 특정 명령어 실행을 조직 차원에서 금지해야 할 때
- 보안 정책을 개발자가 임의로 바꾸지 못하게 해야 할 때
- 관리되지 않는 개인 기기에도 정책을 적용해야 할 때

---

## 사용 요건

| 항목 | 요건 |
|------|------|
| 플랜 | Claude for Teams 또는 Claude for Enterprise |
| Claude Code 버전 | Teams: 2.1.38 이상 / Enterprise: 2.1.30 이상 |
| 네트워크 | `api.anthropic.com` 접근 가능 |

---

## 서버 관리 vs 엔드포인트 관리 설정 비교

| 구분 | 서버 관리 설정 | 엔드포인트 관리 설정 |
|------|--------------|-------------------|
| **적합한 상황** | MDM 없는 조직, 관리되지 않는 기기 | MDM이 있는 조직, 기기 등록 환경 |
| **설정 방식** | Claude.ai 웹 인터페이스에서 JSON 편집 | MDM 프로파일, Windows 레지스트리, 관리형 파일 |
| **보안 강도** | 중간 — 서버에서 전달, 사용자가 바이너리 수정 가능 | 높음 — OS 수준에서 파일 보호 가능 |
| **적용 시점** | 인증 시 및 매시간 자동 갱신 | 기기에 배포된 즉시 |

> 기기가 MDM에 등록되어 있다면 엔드포인트 관리 설정이 더 강한 보안을 제공합니다.

---

## 설정 방법

### 1단계: 관리자 콘솔 열기

[Claude.ai](https://claude.ai)에서 **Admin Settings > Claude Code > Managed settings**로 이동합니다.

### 2단계: 설정 JSON 작성

JSON 형식으로 정책을 작성합니다. `settings.json`에서 사용 가능한 모든 설정을 여기서도 사용할 수 있습니다.

**예시: 위험 명령 차단 및 권한 우회 방지**
```json
{
  "permissions": {
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  },
  "disableBypassPermissionsMode": "disable"
}
```

위 설정의 의미:
- `Bash(curl *)`: 외부 URL로 데이터를 전송할 수 있는 curl 명령 차단
- `Read(./.env)`, `Read(./.env.*)`: 환경변수 파일(비밀번호 등 저장) 읽기 차단
- `Read(./secrets/**)`: 비밀 정보 폴더 읽기 차단
- `disableBypassPermissionsMode`: 사용자가 권한 설정을 우회하지 못하게 차단

### 3단계: 저장 및 배포

변경사항을 저장합니다. Claude Code 클라이언트는 다음 시작 시 또는 매시간 자동으로 새 설정을 받습니다.

---

## 설정 적용 확인 방법

사용자가 Claude Code를 재시작하게 합니다. 설정에 보안 승인이 필요한 항목이 포함되어 있다면, 시작 시 관리형 설정을 설명하는 안내 창이 표시됩니다.

사용자가 `/permissions` 명령을 실행하면 현재 적용된 권한 규칙을 확인할 수 있습니다.

---

## 접근 권한

다음 역할만 서버 관리 설정을 변경할 수 있습니다:
- **Primary Owner** (최상위 소유자)
- **Owner** (소유자)

> 설정 변경은 조직 내 모든 사용자에게 즉시 적용되므로, 신뢰할 수 있는 인원에게만 접근을 허용하세요.

---

## 설정 우선순위

서버 관리 설정과 엔드포인트 관리 설정은 Claude Code의 설정 계층에서 **가장 높은 우선순위**를 가집니다. 명령줄 인수를 포함한 다른 어떤 설정도 이를 덮어쓸 수 없습니다.

두 방식이 모두 설정된 경우, **서버 관리 설정이 우선** 적용되고 엔드포인트 관리 설정은 사용되지 않습니다.

---

## 설정 전달 방식 상세

### 가져오기 및 캐싱 동작

| 상황 | 동작 |
|------|------|
| 캐시 없는 첫 실행 | 비동기적으로 설정 가져옴. 실패 시 관리형 설정 없이 계속 진행 |
| 캐시 있는 재실행 | 즉시 캐시된 설정 적용 후 백그라운드에서 최신 설정 가져옴 |
| 네트워크 장애 | 캐시된 설정 적용 (없으면 관리형 설정 미적용 상태로 동작) |

> 대부분의 설정은 재시작 없이 자동 적용됩니다. OpenTelemetry 설정 등 일부 고급 설정은 완전한 재시작이 필요합니다.

### 보안 승인 대화상자

아래 유형의 설정은 사용자 명시적 승인이 필요합니다:
- **셸 명령 실행 설정**: 셸 명령을 실행하는 설정
- **커스텀 환경변수**: 알려진 안전 목록에 없는 변수
- **훅 설정**: 모든 훅 정의

사용자가 거부하면 Claude Code가 종료됩니다.

> `-p` 플래그를 사용한 비대화형 모드에서는 승인 대화상자를 건너뛰고 설정이 자동 적용됩니다.

---

## 사용할 수 없는 경우

서버 관리 설정은 `api.anthropic.com`에 직접 연결이 필요하며, 아래 환경에서는 사용할 수 없습니다:

- Amazon Bedrock
- Google Vertex AI
- Microsoft Foundry
- `ANTHROPIC_BASE_URL`로 설정된 커스텀 API 엔드포인트
- LLM 게이트웨이

---

## 감사 로깅

설정 변경 감사 로그는 컴플라이언스 API 또는 감사 로그 내보내기를 통해 확인할 수 있습니다. 접근 방법은 Anthropic 계정팀에 문의하세요.

감사 이벤트에 포함된 정보:
- 수행된 작업 유형
- 작업을 수행한 계정과 기기
- 변경 전/후 값 참조

---

## 보안 고려사항

서버 관리 설정은 중앙 집중식 정책 적용을 제공하지만, 클라이언트 측 제어입니다. 관리되지 않는 기기에서 관리자/sudo 권한을 가진 사용자는 Claude Code 바이너리, 파일시스템, 또는 네트워크 설정을 수정할 수 있습니다.

| 상황 | 동작 |
|------|------|
| 사용자가 캐시 파일 수정 | 시작 시 변조된 파일 적용, 다음 서버 가져오기 시 올바른 설정으로 복원 |
| 사용자가 캐시 파일 삭제 | 첫 실행 동작 발생: 짧은 미적용 구간 후 설정 복원 |
| API 사용 불가 | 캐시 설정 적용 (없으면 다음 성공 시까지 관리형 설정 미적용) |
| 다른 조직으로 인증 | 관리 조직 외 계정에는 설정 미전달 |

> 더 강한 보안이 필요하다면 MDM 솔루션의 엔드포인트 관리 설정을 사용하세요.

## 예시 케이스

**상황**: 스타트업이 빠르게 성장하면서 개발자가 15명에서 50명으로 늘었습니다. MDM 도구가 없고 개발자들은 개인 맥북을 씁니다. 보안팀이 Claude Code가 `.env` 파일이나 비밀 폴더에 접근하지 못하게 하고 싶지만, 개발자마다 설정을 직접 변경하는 것은 불가능합니다.

**해결책**:
1. Claude.ai 관리자 콘솔의 Managed Settings에서 아래 JSON을 설정합니다:
   ```json
   {
     "permissions": {
       "deny": [
         "Read(./.env)",
         "Read(./.env.*)",
         "Read(./secrets/**)",
         "Bash(cat .env*)"
       ]
     },
     "disableBypassPermissionsMode": "disable"
   }
   ```
2. 저장 후 모든 직원에게 Claude Code를 재시작하도록 안내합니다.

결과: 50명 개발자 모두의 Claude Code에 동일한 보안 정책이 자동 적용됩니다. 개발자가 직접 해당 설정을 바꿀 수 없습니다.
