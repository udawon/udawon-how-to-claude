# 체크포인트: 실수해도 되감기가 가능합니다

## 이게 뭔가요?

체크포인트(Checkpointing)는 Claude Code가 파일을 수정하기 전에 자동으로 현재 상태를 저장해두는 기능입니다. 잘못된 방향으로 작업이 진행되더라도 언제든지 이전 상태로 되돌아갈 수 있습니다.

비디오 게임의 **세이브 포인트**와 똑같은 개념입니다. 어려운 구간을 도전하기 전에 저장해두면, 실패해도 그 지점부터 다시 시작할 수 있습니다. Claude Code는 매 작업마다 자동으로 세이브 포인트를 만들어줍니다.

## 어떻게 작동하나요?

### 자동 저장

- 여러분이 Claude에게 메시지를 보낼 때마다 새 체크포인트가 자동으로 생성됩니다.
- 체크포인트는 세션이 종료되어도 유지됩니다. 나중에 대화를 재개해도 이전 체크포인트에 접근할 수 있습니다.
- 30일 후 자동으로 정리됩니다 (설정으로 변경 가능).

### 되감기 방법

`Esc` 키를 두 번 누르거나 `/rewind` 명령어를 입력하면 되감기 메뉴가 열립니다. 세션의 각 메시지 목록이 표시됩니다.

원하는 지점을 선택한 후, 다음 중 하나를 선택합니다.

| 옵션 | 설명 |
|---|---|
| 코드와 대화 모두 복원 | 파일과 대화 내역을 모두 선택한 지점으로 되돌립니다 |
| 대화만 복원 | 파일은 현재 상태 유지, 대화 내역만 되돌립니다 |
| 코드만 복원 | 대화는 현재 상태 유지, 파일만 되돌립니다 |
| 여기서부터 요약 | 선택한 지점 이후의 대화를 압축합니다 (파일은 변경 없음) |
| 취소 | 아무것도 하지 않고 메시지 목록으로 돌아갑니다 |

되감기 후에는 선택한 지점의 원래 메시지가 입력창에 자동으로 복원되어, 바로 다시 보내거나 수정할 수 있습니다.

## 복원 vs 요약: 차이점

### 복원 옵션들

세 가지 복원 옵션은 모두 **상태를 되돌립니다**. 코드 변경사항, 대화 내역, 또는 두 가지 모두를 이전 상태로 복구합니다.

### 여기서부터 요약

이 옵션은 **복원이 아닙니다**. 다음과 같이 작동합니다.

- 선택한 메시지 이전의 내용은 그대로 보존됩니다.
- 선택한 메시지와 이후 내용이 AI가 생성한 짧은 요약으로 교체됩니다.
- 디스크의 실제 파일은 변경되지 않습니다.
- 원본 메시지들은 세션 기록에 보존되어 Claude가 필요시 참조할 수 있습니다.

이것은 `/compact` 명령어와 비슷하지만, 대화 전체가 아닌 **특정 지점 이후만 압축**한다는 차이가 있습니다. 초반의 중요한 지시사항은 그대로 유지하면서 나중에 쌓인 긴 디버깅 내역만 압축할 때 유용합니다.

> **분기(Fork)와의 차이**: 요약은 현재 세션 내에서 컨텍스트를 압축합니다. 완전히 다른 접근 방식을 시도하면서 원본 세션을 그대로 보존하고 싶다면, `claude --continue --fork-session` 명령어로 세션을 분기하세요.

## 예시 케이스

### 케이스 1: 기능 구현 방향 바꾸기

개발자가 새로운 검색 기능을 구현하기 시작했습니다. Claude가 파일 3개를 수정하고 나서야 "이 방식은 성능에 문제가 있겠다"는 것을 깨달았습니다. `Esc`를 두 번 누르고 "코드와 대화 모두 복원"을 선택해 시작 시점으로 돌아가서 다른 방식으로 다시 구현합니다.

### 케이스 2: 실수로 중요 파일 삭제

Claude에게 파일 정리를 맡겼는데 중요한 설정 파일이 삭제됐습니다. 되감기 메뉴에서 삭제 직전 체크포인트를 찾아 "코드만 복원"을 선택하면 파일을 복구할 수 있습니다. 대화 내역은 그대로 유지됩니다.

### 케이스 3: 긴 디버깅 세션 후 컨텍스트 정리

2시간 동안 버그를 디버깅하며 대화 창이 매우 길어졌습니다. 버그는 수정됐지만 컨텍스트 한도가 가득 차고 있습니다. 버그 수정이 완료된 지점을 선택하고 "여기서부터 요약"을 선택합니다. 초반의 기능 명세와 설계 논의는 그대로 유지하면서, 긴 디버깅 과정만 짧은 요약으로 압축합니다.

### 케이스 4: 다른 구현 방식 비교

Claude가 A 방식으로 로그인 기능을 구현했습니다. "B 방식으로도 해보고 싶다"는 생각이 들었을 때, A 방식 구현 전 체크포인트로 되돌아가 B 방식을 시도해볼 수 있습니다.

## 제한사항

### bash 명령어로 변경된 파일은 추적되지 않습니다

체크포인트는 Claude의 파일 편집 도구를 통해 변경된 파일만 추적합니다. Claude가 bash 명령어로 직접 파일을 조작한 경우에는 되감기로 복원할 수 없습니다.

```bash
# 이런 명령어들은 체크포인트로 복원 불가
rm file.txt
mv old.txt new.txt
cp source.txt dest.txt
```

### 외부에서 변경된 파일은 추적되지 않습니다

Claude Code 외부에서 여러분이 직접 수정한 파일이나, 다른 세션에서 수정된 파일은 현재 세션의 체크포인트에 포함되지 않습니다. 단, 현재 세션에서도 수정된 파일이라면 추적됩니다.

### Git을 대체하지 않습니다

체크포인트는 **빠른 세션 단위 복구**를 위한 도구입니다. 영구적인 버전 관리는 Git을 계속 사용해야 합니다.

| 기능 | 체크포인트 | Git |
|---|---|---|
| 목적 | 빠른 임시 복구 | 영구적인 버전 관리 |
| 유지 기간 | 30일 | 영구 |
| 협업 | 불가 | 가능 |
| 비유 | 게임 세이브 포인트 | 공식 저장소 |

체크포인트는 "로컬 되돌리기", Git은 "영구 기록"으로 생각하면 됩니다. 두 가지를 함께 사용하는 것이 가장 안전합니다.
