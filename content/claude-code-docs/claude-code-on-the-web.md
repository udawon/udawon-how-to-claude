# 웹에서 Claude Code 사용하기

Claude Code를 내 컴퓨터에 설치하지 않고도 웹 브라우저에서 바로 사용할 수 있습니다. Anthropic의 안전한 클라우드 서버에서 코드 작업이 진행되며, 앱을 닫아도 작업이 계속됩니다. 마치 외부 개발 업체에 일을 맡기면 내가 자리를 비워도 작업이 진행되는 것과 같습니다.

> 현재 리서치 프리뷰 단계입니다.

## 예시 케이스

> 기획자 강민서 씨는 GitHub에 올라온 회사 프로젝트의 API 문서를 자동으로 업데이트해야 합니다. claude.ai/code에 접속해서 GitHub 계정을 연결하고, 저장소를 선택한 뒤 "모든 API 엔드포인트를 분석하고 README에 문서를 추가해줘"라고 입력합니다. 이후 회의를 마치고 돌아왔을 때 작업이 완료되어 있고, 변경사항을 확인한 뒤 바로 PR을 생성합니다.

---

## 이런 상황에 적합합니다

| 상황 | 설명 |
|------|------|
| **코드 질문** | 코드 구조나 기능 구현 방법 질문 |
| **버그 수정** | 명확히 정의된 버그를 수정하는 작업 |
| **병렬 작업** | 여러 버그 수정 작업을 동시에 진행 |
| **로컬에 없는 저장소** | 내 컴퓨터에 체크아웃하지 않은 코드 작업 |
| **백엔드 변경** | 테스트 작성 후 코드 구현하는 TDD 방식 작업 |

---

## 사용 가능한 플랜

| 플랜 | 사용 가능 여부 |
|------|----------------|
| Pro | 가능 |
| Max | 가능 |
| Team | 가능 |
| Enterprise (프리미엄 시트) | 가능 |

---

## 시작 방법

1. [claude.ai/code](https://claude.ai/code) 접속
2. GitHub 계정 연결
3. Claude GitHub 앱을 저장소에 설치
4. 기본 환경 선택
5. 코딩 작업 입력
6. 변경사항을 diff로 검토하고 반복 수정 후 PR 생성

---

## 작동 원리

작업을 시작하면 내부에서 이런 일이 일어납니다:

1. **저장소 복제**: Anthropic 관리 VM에 저장소가 복제됩니다.
2. **환경 준비**: 코드와 함께 안전한 클라우드 환경이 구성됩니다. 설정 스크립트가 있으면 실행됩니다.
3. **네트워크 설정**: 보안 설정에 따라 인터넷 접근이 구성됩니다.
4. **작업 실행**: Claude가 코드 분석, 수정, 테스트를 진행합니다.
5. **완료**: 브랜치에 변경사항이 푸시되고 PR 생성이 가능해집니다.

---

## 변경사항 비교(Diff) 검토

Claude가 파일을 수정하면 `+12 -1` 같은 표시가 나타납니다. 클릭하면:
- 왼쪽: 수정된 파일 목록
- 오른쪽: 각 파일의 구체적인 변경 내용

특정 줄에 댓글을 달면 Claude가 수정 사항을 반영합니다. PR을 생성하지 않고도 여러 번 반복해서 수정할 수 있습니다.

## 터미널과 웹 간 세션 이동

> 세션 이동은 **웹 → 터미널** 방향만 가능합니다. 기존 터미널 세션을 웹으로 보낼 수는 없으며, `--remote` 플래그는 새 웹 세션을 생성합니다.

### 터미널에서 웹 세션 시작하기

```bash
claude --remote "src/auth/login.ts의 인증 버그를 수정해줘"
```

이 명령으로 claude.ai에 새 웹 세션이 생성되고 클라우드에서 실행됩니다. `/tasks`로 진행 상황을 확인하거나, claude.ai 또는 Claude 모바일 앱에서 직접 대화할 수 있습니다.

**여러 작업 병렬 실행:**
```bash
claude --remote "auth.spec.ts의 불안정한 테스트 수정"
claude --remote "API 문서 업데이트"
claude --remote "logger를 구조화된 출력으로 리팩토링"
```

### 웹 세션을 터미널로 가져오기

| 방법 | 명령 |
|------|------|
| 현재 세션에서 | `/teleport` 또는 `/tp` 입력 |
| 명령줄에서 | `claude --teleport` |
| 작업 목록에서 | `/tasks` → `t` 키 |
| 웹 인터페이스 | "CLI에서 열기" 클릭 |

**텔레포트 요구사항:**

| 조건 | 설명 |
|------|------|
| 깨끗한 Git 상태 | 커밋되지 않은 변경사항 없어야 함 (없으면 stash 처리 안내) |
| 올바른 저장소 | 동일 저장소에서 실행해야 함 |
| 브랜치 존재 | 웹 세션의 브랜치가 원격에 푸시되어 있어야 함 |
| 동일 계정 | 웹 세션과 동일한 Claude.ai 계정으로 인증 |

## 클라우드 환경 설정

### 기본 제공 도구

환경에 미리 설치된 도구를 확인하려면 Claude에게 `check-tools` 실행을 요청합니다.

| 언어/도구 | 포함 내용 |
|-----------|-----------|
| Python | Python 3.x, pip, poetry, 과학 라이브러리 |
| Node.js | 최신 LTS, npm, yarn, pnpm, bun |
| Ruby | 3.1.6~3.3.6 버전, gem, bundler, rbenv |
| PHP | 8.4.14 버전 |
| Java | OpenJDK, Maven, Gradle |
| Go | 최신 안정 버전, 모듈 지원 |
| Rust | cargo 포함 |
| C++ | GCC, Clang |
| PostgreSQL | 16 버전 |
| Redis | 7.0 버전 |

### 환경 추가/수정하기

- **새 환경 추가**: 환경 선택 드롭다운 → "환경 추가" → 이름, 네트워크 접근, 환경 변수, 설정 스크립트 입력
- **기존 환경 수정**: 환경 이름 옆 설정 버튼 클릭

---

## 설정 스크립트

새 클라우드 세션이 시작될 때 Claude Code가 실행되기 전에 실행되는 Bash 스크립트입니다. 기본 이미지에 없는 도구를 설치할 때 사용합니다.

예시: `gh` CLI 설치
```bash
#!/bin/bash
apt update && apt install -y gh
```

> 설정 스크립트는 새 세션을 시작할 때만 실행되며, 기존 세션을 재개할 때는 실행되지 않습니다.

---

## 네트워크 접근 설정

기본값은 **제한된 접근**으로, npm, PyPI, RubyGems, crates.io 등 일반적인 패키지 레지스트리만 허용됩니다. 환경 설정에서 다음 중 선택 가능합니다:

- **인터넷 없음**: 완전히 격리된 환경
- **제한된 접근** (기본): 주요 패키지 레지스트리만 허용
- **전체 접근**: 모든 인터넷 접근 허용

## 세션 공유

### Teams/Enterprise 계정
**Private** (나만 보기) 또는 **Team** (조직 멤버 공개) 중 선택 가능합니다. GitHub 계정 기반으로 저장소 접근 권한이 검증됩니다.

### Max/Pro 계정
**Private** (나만 보기) 또는 **Public** (로그인한 모든 사용자에게 공개) 중 선택 가능합니다. 공유 전에 민감한 정보(인증 키, 비밀번호 등)가 포함되어 있는지 확인하세요.

---

## 세션 관리

- **보관**: 세션 위에 마우스를 올려 보관 아이콘 클릭 (목록에서 숨겨지지만 삭제되지 않음)
- **삭제**: 보관된 세션 필터링 후 삭제 아이콘 클릭, 또는 세션 제목 드롭다운에서 **삭제** 선택

> 삭제는 영구적이며 되돌릴 수 없습니다.

## 현재 제한사항

| 제한사항 | 내용 |
|----------|------|
| 지원 플랫폼 | GitHub만 지원 (GitLab, Bitbucket 등 미지원) |
| 계정 인증 | 웹과 로컬 이동 시 동일 계정 필요 |

---

## 보안

- **격리된 VM**: 각 세션이 독립된 VM에서 실행
- **네트워크 접근 제한**: 기본적으로 제한된 접근만 허용
- **인증 정보 보호**: git 인증 정보 등 민감한 정보는 프록시를 통해 안전하게 처리

> 네트워크 접근이 비활성화된 경우에도 Anthropic API와의 통신은 허용되므로 데이터가 완전히 격리되지는 않습니다.
