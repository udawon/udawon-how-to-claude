# 터미널 구성 가이드: Claude Code가 더 편리해지는 터미널 설정

터미널이 잘 설정되어 있으면 Claude Code를 훨씬 편하게 사용할 수 있습니다. 마치 작업실을 자신에게 맞게 정리하는 것처럼요. 이 가이드에서는 줄바꿈, 알림, 큰 텍스트 처리 등 실용적인 설정을 알려드립니다.

---

## 테마(색상) 설정

Claude Code의 색상 테마는 터미널 앱에서 설정합니다. Claude Code 자체에서는 `/config` 명령어로 Claude Code 인터페이스 테마를 맞출 수 있습니다.

---

## 줄바꿈 입력 방법

Claude Code에서 메시지를 입력할 때 새 줄을 추가하는 방법은 여러 가지입니다.

| 방법 | 설명 | 지원 터미널 |
|------|------|-----------|
| `\` + Enter | 백슬래시 후 엔터 | 모든 터미널 |
| Shift+Enter | 그냥 Shift를 누른 채 Enter | iTerm2, WezTerm, Ghostty, Kitty |
| 단축키 설정 | 아래 방법으로 설정 가능 | 기타 터미널 |

### 다른 터미널에서 Shift+Enter 설정하기

Claude Code 안에서 아래 명령어를 실행하면 자동으로 설정해줍니다:

```
/terminal-setup
```

VS Code, Alacritty, Zed, Warp 터미널을 지원합니다.

> **참고:** iTerm2, WezTerm, Ghostty, Kitty를 사용 중이라면 이미 Shift+Enter가 되므로 `/terminal-setup` 명령어가 보이지 않습니다.

### Option+Enter 설정 (macOS)

**Mac Terminal.app:**
1. 설정 → 프로파일 → 키보드
2. "Option을 Meta 키로 사용" 체크

**iTerm2, VS Code 터미널:**
1. 설정 → 프로파일 → 키
2. 왼쪽/오른쪽 Option 키를 "Esc+"로 설정

## 알림 설정

Claude가 작업을 마치고 내 입력을 기다릴 때 알림을 받을 수 있습니다.

### 터미널별 알림 지원

| 터미널 | 알림 지원 | 설정 필요? |
|-------|---------|---------|
| Kitty | 예 | 없음 |
| Ghostty | 예 | 없음 |
| iTerm 2 | 예 | 아래 설정 필요 |
| 기본 macOS Terminal | 아니오 | Hooks 사용 |
| 기타 터미널 | 아니오 | Hooks 사용 |

**iTerm 2 알림 설정:**
1. iTerm 2 설정 → 프로파일 → 터미널
2. "Notification Center Alerts" 활성화
3. "Filter Alerts" 클릭 → "Send escape sequence-generated alerts" 체크

알림이 안 보이면 OS 설정에서 터미널 앱의 알림 권한을 확인하세요.

### 알림 Hooks (모든 터미널에서 가능)

터미널 알림 대신 또는 추가로, 소리 재생이나 슬랙 메시지 발송 같은 커스텀 동작을 설정할 수 있습니다. 훅(Hook)은 알림을 대체하는 것이 아니라 함께 실행됩니다.

## 긴 텍스트 처리 팁

매우 긴 코드나 지시사항을 다룰 때:

| 상황 | 권장 방법 |
|------|---------|
| 긴 코드 붙여넣기 | 직접 붙여넣기 대신 파일로 저장 후 읽게 하기 |
| VS Code 터미널 사용 | 특히 긴 내용 붙여넣기 시 잘릴 수 있음 주의 |

**추천 방법:** "이 파일을 읽어줘"라고 말하면서 파일 경로를 알려주는 것이 가장 안정적입니다.

---

## Vim 모드

키보드로 텍스트를 빠르게 편집하는 Vim 방식을 선호한다면:

```
/vim
```

또는 `/config`에서 설정 가능합니다.

**지원 기능:**

| 종류 | 기능 |
|------|------|
| 모드 전환 | `Esc` (일반 모드), `i`, `a`, `o` (입력 모드) |
| 이동 | `h`, `j`, `k`, `l` (방향), `w`, `b` (단어 이동) |
| 편집 | `x` (글자 삭제), `dd` (줄 삭제), `yy` (줄 복사) |
| 텍스트 객체 | `iw` (단어 안), `i"` (따옴표 안) 등 |

## 예시 케이스

**상황 1: 처음 Claude Code를 쓰는 개발자, 줄바꿈이 안 돼서 당황**

> 개발자 신입 박신입 씨는 메시지를 입력하다가 Enter를 눌렀더니 바로 전송되어버렸습니다. 여러 줄로 질문하고 싶었는데요. Alacritty를 사용하고 있어서 `/terminal-setup`을 실행해 Shift+Enter를 설정했습니다. 이제 Shift+Enter로 줄을 바꾸고, 완성된 메시지를 Enter로 전송합니다.

**상황 2: Claude가 언제 끝났는지 모르겠는 상황**

> 프리랜서 최자유 씨는 Claude에게 작업을 시키고 다른 일을 합니다. Claude가 언제 끝나는지 계속 터미널을 확인해야 해서 불편했습니다. Ghostty 터미널을 사용하고 있어서 알림이 자동으로 설정됩니다. 이제 Claude가 끝나면 데스크탑 알림이 와서 다른 일에 집중할 수 있게 됐습니다.

**상황 3: 대용량 파일 분석 요청**

> 데이터 분석가 데이터 씨는 5000줄짜리 CSV 파일 내용을 Claude에게 붙여넣기 했더니 터미널이 느려지고 내용이 잘렸습니다. 이후에는 파일을 직접 붙여넣지 않고, "~/data/sales.csv 파일을 읽고 분석해줘"라고 말했습니다. Claude가 직접 파일을 열어서 분석해줬습니다.

**상황 4: Vim 사용자의 메시지 편집**

> 오랫동안 Vim을 사용해온 시니어 개발자 김시니어 씨는 Claude에게 보낼 긴 메시지를 작성할 때 마우스를 쓰기 싫습니다. `/vim`으로 Vim 모드를 켜서 익숙한 키 조합으로 메시지를 빠르게 편집합니다.
