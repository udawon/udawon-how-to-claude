## 이게 뭔가요?

Claude Code의 내장 명령어는 **슬래시(/) 하나를 누르면 나타나는 기능 버튼 목록**입니다.

TV 리모컨을 생각해보세요. 채널 변경, 볼륨 조절, 입력 전환 — 각 버튼마다 하나의 기능이 있습니다. Claude Code도 마찬가지로, `/clear`는 화면 초기화, `/diff`는 변경 내용 보기, `/voice`는 음성 입력처럼 **슬래시 뒤에 명령어를 붙이면 해당 기능이 바로 실행**됩니다.

터미널(키보드로 명령어를 입력하는 화면)에서 `/`를 입력하면 사용 가능한 명령어 목록이 자동으로 나타나고, 화살표 키로 선택하거나 이름을 직접 입력할 수 있습니다.

## 왜 알아야 하나요?

**알면 좋은 점:**
- Claude에게 긴 문장으로 요청하지 않아도 **한 단어로 기능 실행** 가능
- 세션 관리, 코드 리뷰, 설정 변경 등을 빠르게 처리
- 어떤 기능이 있는지 알아야 Claude Code를 100% 활용 가능

**모르면 불편한 점:**
- "이전 대화로 돌아가고 싶은데 방법을 모르겠다" → `/resume`을 몰라서 새 세션을 시작
- "변경한 코드를 확인하고 싶다" → `/diff`를 몰라서 일일이 파일을 열어봄
- 존재하는 기능을 모르고 돌아가는 비효율

## 어떻게 하나요?

### 세션 관리 명령어

대화(세션)를 시작하고, 저장하고, 되돌리는 명령어들입니다.

| 명령어 | 기능 | 설명 |
|--------|------|------|
| `/clear` | 대화 초기화 | 현재 대화 내용을 모두 지우고 새로 시작 |
| `/resume` | 이전 세션 이어서 | 이전에 하던 대화를 다시 불러옴 |
| `/rename` | 세션 이름 변경 | 현재 세션에 알아보기 쉬운 이름 붙이기 |
| `/fork` | 대화 분기 | 현재 대화를 복제해서 새 방향으로 진행 |
| `/rewind` | 되감기 | 대화를 특정 시점으로 되돌림 (실수했을 때 유용) |

<div class="example-case">
<strong>예시: 실수한 작업 되돌리기</strong>

Claude에게 파일 수정을 시켰는데 결과가 마음에 들지 않습니다. `/rewind`를 입력하면 수정하기 전 시점으로 대화가 되돌아갑니다. 다시 다른 방향으로 지시를 내릴 수 있습니다. 만약 두 가지 방향을 모두 시도해보고 싶다면 `/fork`로 대화를 복제한 뒤 각각 다른 지시를 내릴 수도 있습니다.

</div>

### Git & 코드 관련 명령어

코드 변경 확인, 리뷰, 보안 검사와 관련된 명령어들입니다.

| 명령어 | 기능 | 설명 |
|--------|------|------|
| `/diff` | 변경 내용 보기 | Claude가 수정한 파일들의 변경 사항 확인 |
| `/pr-comments` | PR 댓글 보기 | GitHub PR(코드 변경 요청)에 달린 댓글 불러오기 |
| `/security-review` | 보안 검사 | 현재 코드의 보안 취약점 점검 |

### 설정 관련 명령어

Claude Code의 동작 방식을 조정하는 명령어들입니다.

| 명령어 | 기능 | 설명 |
|--------|------|------|
| `/config` | 설정 열기 | Claude Code 설정 파일 편집 |
| `/permissions` | 권한 관리 | 파일 읽기/쓰기, 명령 실행 등의 권한 설정 |
| `/hooks` | 훅 관리 | Hook(특정 시점에 자동 실행되는 기능) 설정 확인/편집 |
| `/memory` | 메모리 편집 | CLAUDE.md(프로젝트 규칙 파일)를 직접 편집 |
| `/model` | 모델 변경 | 사용할 AI 모델을 변경 (Opus, Sonnet 등) |

### 도구 & 확장 명령어

외부 도구 연결, 스킬, 플러그인 관련 명령어들입니다.

| 명령어 | 기능 | 설명 |
|--------|------|------|
| `/mcp` | MCP 서버 관리 | MCP(외부 도구 연결 기능) 서버 상태 확인/관리 |
| `/skills` | 스킬 목록 | 사용 가능한 스킬(전문 작업 매뉴얼) 목록 보기 |
| `/agents` | 에이전트 관리 | 서브 에이전트(보조 AI) 설정 확인 |
| `/plugin` | 플러그인 관리 | Plugin(확장 기능) 설치/제거/목록 확인 |

### 유틸리티 명령어

기타 편의 기능 명령어들입니다.

| 명령어 | 기능 | 설명 |
|--------|------|------|
| `/voice` | 음성 입력 | 음성 받아쓰기 모드 활성화 (Space 키로 녹음) |
| `/effort` | 사고 수준 조절 | Claude의 사고 깊이 조정 (빠른 응답 vs 깊은 분석) |
| `/fast` | 빠른 모드 | 경량 모델로 전환하여 빠르게 답변 |
| `/context` | 컨텍스트 확인 | 현재 대화에서 Claude가 인식하고 있는 정보 확인 |
| `/cost` | 비용 확인 | 현재 세션에서 사용한 토큰(AI 사용량 단위) 비용 확인 |
| `/export` | 대화 내보내기 | 현재 대화 내용을 파일로 저장 |
| `/btw` | 별도 요청 | 현재 작업 흐름을 끊지 않고 별도 질문/요청 |
| `/plan` | 계획 세우기 | 작업 계획을 먼저 세운 뒤 실행 |
| `/copy` | 복사 | Claude의 마지막 응답을 클립보드(복사 저장소)에 복사 |
| `/doctor` | 진단 | Claude Code 설치 상태 및 문제 진단 |

<div class="example-case">
<strong>예시: 자주 쓰는 명령어 조합</strong>

프리랜서 개발자 이지은씨의 일상적인 작업 흐름입니다:

1. 아침에 `/resume`으로 어제 하던 작업 이어서 시작
2. 코드 수정이 끝나면 `/diff`로 변경 내용 확인
3. `/security-review`로 보안 문제 없는지 점검
4. 작업 중 급한 질문이 생기면 `/btw`로 현재 흐름을 끊지 않고 질문
5. 퇴근 전 `/cost`로 오늘 사용량 확인

</div>

## 실전 예시

<div class="example-case">
<strong>실전 케이스: 새 프로젝트에서 명령어 활용하기</strong>

신입 개발자 최준호씨가 새 프로젝트를 시작합니다. 먼저 `/doctor`로 Claude Code가 정상 동작하는지 확인합니다. `/model`로 복잡한 설계 작업에 적합한 Opus 모델로 변경합니다. `/plan`으로 "회원가입 기능 만들기" 계획을 세운 뒤 실행합니다. 중간에 방향이 잘못된 것 같아 `/rewind`로 되돌리고 다시 시작합니다. 최종 코드가 완성되면 `/diff`로 전체 변경 사항을 리뷰하고, `/security-review`로 보안 점검까지 마칩니다.

## 주의할 점

- **슬래시(/)는 반드시 입력 맨 앞에** — 문장 중간에 `/`를 쓰면 명령어로 인식되지 않습니다.
- **대소문자 구분 없음** — `/Clear`나 `/CLEAR`도 동작하지만, 소문자 `/clear`가 표준입니다.
- **버전에 따라 명령어가 다를 수 있음** — Claude Code 업데이트 시 새 명령어가 추가되거나 기존 명령어가 변경될 수 있습니다. `/`를 눌러 전체 목록을 확인하세요.
- **커스텀 슬래시 명령어** — `.claude/commands/` 폴더에 마크다운 파일을 넣으면 나만의 명령어를 만들 수도 있습니다.

## 정리

- Claude Code의 내장 명령어는 **`/` + 단어**로 기능을 빠르게 실행하는 단축키입니다.
- 세션 관리, 코드 리뷰, 설정, 도구 연결, 유틸리티 등 **6가지 카테고리**로 분류됩니다.
- `/`를 입력하면 전체 목록이 나타나니, 모르는 명령어가 있으면 먼저 목록을 확인하세요.
