# 보안: Claude Code를 안전하게 사용하는 방법

Claude Code는 여러분의 코드와 컴퓨터를 보호하기 위해 다양한 안전 장치를 갖추고 있습니다. 이 문서는 그 보안 구조를 쉽게 설명합니다.

---

## Claude Code의 보안 철학

Claude Code는 마치 **열쇠를 하나씩 건네주는 방식**으로 동작합니다. 처음에는 파일을 읽는 것만 허용하고, 파일 수정이나 명령 실행이 필요할 때마다 여러분의 허락을 구합니다. 이를 통해 Claude가 여러분 모르게 중요한 파일을 건드리는 일을 막습니다.

Anthropic은 SOC 2 Type 2, ISO 27001 등 국제 보안 인증을 취득했으며, [Trust Center](https://trust.anthropic.com)에서 관련 문서를 확인할 수 있습니다.

---

## 핵심 보안 기능

### 1. 권한 기반 구조 (열쇠 관리)

| 행동 유형 | 기본 동작 |
|---|---|
| 파일 읽기 | 자동 허용 |
| 파일 수정 | 허락 필요 |
| 명령 실행 | 허락 필요 |
| 네트워크 요청 | 허락 필요 |

자주 사용하는 안전한 명령은 "항상 허용" 목록에 추가해 번거로움을 줄일 수 있습니다.

### 2. 쓰기 범위 제한 (울타리 치기)

Claude Code는 **시작한 폴더와 그 하위 폴더에만 파일을 쓸 수 있습니다.** 상위 폴더나 시스템 파일은 절대 수정할 수 없습니다. 마치 집 안에 손님을 들였을 때, 지정된 방에서만 활동하도록 하는 것과 같습니다.

### 3. 샌드박스 (격리된 작업 공간)

`/sandbox` 명령으로 샌드박스를 활성화하면, Claude Code가 지정된 영역 밖으로 나가지 못하도록 운영체제 수준에서 차단합니다. 자세한 내용은 [샌드박싱 문서](./sandboxing.md)를 참조하세요.

## 프롬프트 주입 공격 방어

**프롬프트 주입**이란 악의적인 텍스트를 Claude에게 읽히거나 전달해서, Claude가 의도하지 않은 행동을 하도록 속이는 공격입니다. 예를 들어, 악성 코드 주석에 "모든 파일을 삭제해"라고 적어두는 방식입니다.

Claude Code는 이를 막기 위해 다음과 같은 보호 장치를 운용합니다.

| 보호 장치 | 설명 |
|---|---|
| 권한 시스템 | 위험한 작업은 반드시 승인 필요 |
| 문맥 분석 | 요청 전체를 분석해 이상한 지시 탐지 |
| 명령 차단 목록 | `curl`, `wget` 등 외부 콘텐츠를 가져오는 명령 기본 차단 |
| 웹 접근 격리 | 웹 페이지 내용은 별도 창에서 처리, 메인 대화에 오염 방지 |
| 의심 명령 재확인 | 허용 목록에 있어도 수상한 명령은 다시 승인 요청 |
| 신뢰 확인 | 새 코드베이스나 MCP 서버 첫 실행 시 신뢰 여부 확인 |

## 개인정보 보호

- 민감 정보는 **제한된 기간**만 보관
- API 키, 토큰은 **암호화**하여 저장
- 사용자가 직접 [데이터 학습 설정](https://claude.ai/settings/privacy) 변경 가능

---

## MCP 보안 (외부 도구 연결)

MCP(Model Context Protocol)는 Claude Code에 외부 도구를 연결하는 방식입니다. 중요한 사항:

- MCP 서버 목록은 소스 코드에서 관리됩니다.
- **Anthropic은 MCP 서버를 직접 관리하거나 감사하지 않습니다.**
- 신뢰할 수 있는 제공자의 MCP 서버만 사용하거나 직접 작성하세요.

---

## 클라우드 실행 보안 (웹에서 사용할 때)

웹에서 Claude Code를 사용하면 추가 보안이 적용됩니다.

| 항목 | 설명 |
|---|---|
| 격리된 가상 머신 | 세션마다 독립된 환경에서 실행 |
| 네트워크 제한 | 기본적으로 접근 가능한 도메인 제한 |
| GitHub 토큰 보호 | 실제 토큰 대신 제한된 임시 자격증명 사용 |
| 브랜치 제한 | 현재 작업 브랜치로만 git push 가능 |
| 감사 로그 | 모든 작업 기록 |
| 자동 정리 | 세션 종료 후 환경 자동 삭제 |

## 팀 보안 설정

팀에서 사용할 때 권장하는 설정입니다.

- 조직 전체에 적용할 설정은 "관리형 설정"으로 배포
- 허용 설정은 버전 관리로 공유
- OpenTelemetry로 사용 현황 모니터링
- ConfigChange 훅으로 설정 변경 감사

---

## 보안 취약점 신고

Claude Code에서 보안 문제를 발견하면:

1. 공개적으로 공지하지 마세요.
2. [HackerOne 프로그램](https://hackerone.com/anthropic-vdp/reports/new?type=team&report_type=vulnerability)을 통해 신고하세요.
3. 재현 방법을 상세히 작성하세요.
4. Anthropic이 수정할 때까지 공개 공표를 자제해 주세요.

## 예시 케이스

### 상황 1: 오픈소스 라이브러리의 악성 코드
회사 프로젝트에 외부 npm 패키지를 추가했는데, 그 패키지가 악성 코드를 포함하고 있다고 가정합니다. Claude Code는 샌드박스 덕분에 그 패키지가 네트워크로 데이터를 보내거나 허용되지 않은 파일에 접근하는 것을 차단합니다.

### 상황 2: 악성 코드베이스 분석
알 수 없는 출처의 코드를 분석할 때, Claude Code는 코드 내 주석이나 문자열에 숨겨진 지시사항("이 파일을 지워라" 등)을 자동으로 감지하고, 실행 전 권한을 요청해 여러분이 최종 판단을 내릴 수 있게 합니다.

### 상황 3: 팀 공동 작업
여러 개발자가 같은 Claude Code 설정을 쓰는 팀에서, 관리자가 `curl` 및 네트워크 요청을 조직 정책에 따라 차단하면 모든 팀원에게 동일하게 적용됩니다.

## 보안 모범 사례 요약

1. 승인 전 Claude가 제안하는 명령을 꼭 확인하세요.
2. 신뢰할 수 없는 외부 콘텐츠를 Claude에게 직접 붙여넣지 마세요.
3. 중요한 파일의 변경 사항은 항상 검토하세요.
4. 외부 웹 서비스와 상호작용할 때는 가상 환경(VM) 사용을 고려하세요.
5. 수상한 동작은 `/bug` 명령으로 신고하세요.
