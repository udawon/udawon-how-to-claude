# 예약 작업: 정해진 시간에 자동으로 실행하기

## 이게 뭔가요?

예약 작업(Scheduled Tasks)은 Claude Code에서 특정 명령을 일정한 간격으로 또는 정해진 시간에 자동으로 반복 실행하도록 설정하는 기능입니다.

예를 들어 "5분마다 배포가 완료됐는지 확인해줘"처럼 요청하면 Claude가 5분마다 자동으로 배포 상태를 확인하고 결과를 알려줍니다.

이것은 마치 **알림 타이머**와 같습니다. "매 시간마다 이 파일을 확인해줘"라고 설정해두면 Claude가 알아서 반복 확인하는 방식입니다.

**중요한 제한사항**: 예약 작업은 현재 Claude Code 세션이 실행 중일 때만 동작합니다. 터미널을 닫으면 모든 예약 작업이 사라집니다. 세션이 종료되어도 유지되는 예약 작업이 필요하다면 [Desktop 예약 작업](/en/desktop#schedule-recurring-tasks) 또는 [GitHub Actions](/en/github-actions)를 사용하세요.

## /loop 명령어: 반복 실행 설정

`/loop` 명령어는 가장 간단하게 반복 작업을 설정하는 방법입니다.

```
/loop 5m 배포가 완료됐는지 확인하고 결과를 알려줘
```

Claude가 5분마다 해당 작업을 자동으로 실행합니다.

### 간격 설정 방법

| 형식 | 예시 | 의미 |
|---|---|---|
| 앞에 간격 명시 | `/loop 30m 빌드 확인` | 30분마다 실행 |
| 뒤에 `every` 구문 | `/loop 빌드 확인 every 2 hours` | 2시간마다 실행 |
| 간격 없음 | `/loop 빌드 확인` | 기본값 10분마다 실행 |

**지원되는 시간 단위:**
- `s` = 초 (초는 내부적으로 분으로 반올림됩니다)
- `m` = 분
- `h` = 시간
- `d` = 일

> **참고**: 7분, 90분처럼 깔끔하게 나누어지지 않는 간격은 가장 가까운 값으로 자동 조정됩니다. Claude가 선택한 값을 알려줍니다.

### 다른 명령어 반복 실행

반복 실행할 내용으로 다른 Claude 명령어나 스킬을 지정할 수도 있습니다.

```
/loop 20m /review-pr 1234
```

20분마다 PR #1234를 자동으로 리뷰합니다.

## 일회성 리마인더 설정

반복이 아닌 특정 시간에 한 번만 실행할 작업은 자연어로 설명하면 됩니다.

```
오후 3시에 릴리즈 브랜치 푸시하는 거 알려줘
```

```
45분 후에 통합 테스트 통과했는지 확인해줘
```

Claude가 지정한 시간에 정확히 한 번 실행한 뒤 자동으로 해당 작업을 삭제합니다.

## 예약 작업 관리

### 현재 예약된 작업 확인

```
예약된 작업이 뭐가 있어?
```

### 특정 작업 취소

```
배포 확인 작업 취소해줘
```

### 내부 도구 직접 사용

| 도구 | 용도 |
|---|---|
| `CronCreate` | 새 예약 작업 생성 (5자리 cron 표현식, 실행할 프롬프트, 반복 여부 지정) |
| `CronList` | 현재 예약된 모든 작업 목록 조회 |
| `CronDelete` | 작업 ID로 특정 작업 삭제 |

각 작업에는 8자리 고유 ID가 부여됩니다. 한 세션에서 최대 50개의 예약 작업을 만들 수 있습니다.

## 예시 케이스

### 케이스 1: 배포 상태 자동 모니터링

개발자가 프로덕션 배포를 시작하고 다른 작업을 하고 싶을 때:

```
/loop 5m 배포 상태 확인하고 완료되면 알려줘
```

5분마다 Claude가 배포 상태를 확인합니다. 배포가 완료되면 알림을 받고, `/loop` 작업을 취소하면 됩니다.

### 케이스 2: 장시간 빌드 감시

CI 빌드가 1시간 이상 걸릴 때:

```
in 1 hour check if the build passed and summarize any failures
```

1시간 후에 Claude가 빌드 결과를 확인하고 실패 항목이 있으면 요약해 줍니다.

### 케이스 3: 정기 PR 리뷰

팀에서 특정 PR이 진행 중일 때, 새로운 커밋이 추가됐는지 주기적으로 확인하고 싶을 때:

```
/loop 30m /review-pr 456
```

30분마다 PR #456을 자동으로 리뷰합니다.

### 케이스 4: 업무 마감 알림

오늘 중요한 작업이 있을 때:

```
오후 5시에 오늘 완료하지 못한 이슈 목록 알려줘
```

오후 5시에 한 번만 실행되고 자동 삭제됩니다.

## 작동 방식 상세

### 실행 우선순위

예약 작업은 낮은 우선순위로 실행됩니다. Claude가 다른 응답을 처리 중이라면, 예약 작업은 현재 작업이 끝날 때까지 대기합니다.

### 시간대

모든 시간은 Claude Code가 실행 중인 컴퓨터의 **로컬 시간대**를 기준으로 합니다. `0 9 * * *`은 UTC가 아닌 여러분의 현지 오전 9시를 의미합니다.

### 약간의 지연(Jitter)

많은 사용자가 동시에 API를 호출하는 것을 방지하기 위해 약간의 랜덤 지연이 추가됩니다.

| 작업 유형 | 지연 범위 |
|---|---|
| 반복 작업 | 설정 간격의 최대 10% (최대 15분) |
| 일회성 작업 (정각/30분) | 최대 90초 일찍 실행 |

같은 작업은 항상 같은 지연값을 가집니다.

### 3일 자동 만료

반복 작업은 생성 후 3일이 지나면 자동으로 삭제됩니다. 마지막 실행 후 사라집니다. 3일 이상 필요한 경우 만료 전에 취소하고 새로 만들거나, [Desktop 예약 작업](/en/desktop#schedule-recurring-tasks)을 사용하세요.

## cron 표현식 참고

`CronCreate`는 5자리 표준 cron 표현식을 사용합니다: `분 시 일 월 요일`

| 예시 | 의미 |
|---|---|
| `*/5 * * * *` | 5분마다 |
| `0 * * * *` | 매시 정각 |
| `0 9 * * *` | 매일 오전 9시 |
| `0 9 * * 1-5` | 평일 오전 9시 |
| `30 14 15 3 *` | 3월 15일 오후 2시 30분 |

요일은 `0` 또는 `7`이 일요일이고 `6`이 토요일입니다.

## 예약 작업 비활성화

예약 작업 기능 자체를 끄고 싶다면 환경 변수를 설정합니다.

```
CLAUDE_CODE_DISABLE_CRON=1
```

이 변수를 설정하면 `/loop` 명령어와 cron 도구들이 비활성화되고, 기존에 예약된 작업도 실행되지 않습니다.

## 제한사항 정리

| 제한 | 내용 |
|---|---|
| 세션 의존성 | Claude Code가 실행 중일 때만 작동 |
| 미실행 보충 없음 | 작업 시간에 Claude가 바쁘면 한 번만 실행, 누적 실행 없음 |
| 재시작 시 초기화 | Claude Code 재시작 시 모든 예약 작업 삭제 |
| 최대 50개 | 한 세션에서 동시에 최대 50개 작업 |
| 3일 만료 | 반복 작업은 3일 후 자동 삭제 |
