# Claude Code 데스크탑 앱 완전 가이드

Claude Code 데스크탑 앱의 **Code 탭**은 터미널 없이도 Claude Code의 모든 기능을 그래픽 인터페이스로 사용할 수 있게 해줍니다. 마치 AI 비서가 내 컴퓨터 화면을 함께 보면서 코드를 작업하는 것과 같습니다.

## 예시 케이스

> 스타트업 창업자 박서연 씨는 개발자는 아니지만 자신의 웹사이트 코드를 직접 수정하고 싶습니다. 데스크탑 앱을 설치하고 Code 탭을 열어 프로젝트 폴더를 선택합니다. "회원가입 버튼을 클릭했을 때 환영 이메일이 발송되지 않는 문제가 있어"라고 입력하면, Claude가 코드를 분석하고 어떤 파일을 어떻게 수정할지 계획을 먼저 보여준 뒤, 승인하면 수정을 진행합니다.

---

## 데스크탑 앱만의 특별한 기능

| 기능 | 설명 |
|------|------|
| **시각적 변경사항 비교** | 수정 전후를 나란히 비교, 줄별 댓글 추가 가능 |
| **앱 미리보기** | 개발 서버를 내장 브라우저로 실시간 확인 |
| **GitHub PR 모니터링** | 풀 리퀘스트의 CI 상태 자동 확인 및 자동 수정 |
| **병렬 세션** | 여러 작업을 동시에 독립적으로 진행 (Git 격리) |
| **예약 작업** | Claude를 정해진 일정에 자동으로 실행 |
| **커넥터** | GitHub, Slack, Linear 등 외부 도구 연결 |
| **원격 실행** | Anthropic 클라우드나 SSH로 원격 환경에서 작업 |

---

## 세션 시작하기

세션을 시작하기 전 4가지를 설정합니다:

1. **환경**: Local(내 컴퓨터), Remote(Anthropic 클라우드), SSH(원격 서버) 중 선택
2. **프로젝트 폴더**: Claude가 작업할 폴더 선택
3. **모델**: 사용할 AI 모델 선택 (세션 시작 후 변경 불가)
4. **권한 모드**: Claude에게 얼마나 자율권을 줄지 결정

---

## 권한 모드 비교

| 모드 | 동작 방식 | 추천 상황 |
|------|-----------|-----------|
| **승인 요청 모드** | 파일 수정 또는 명령 실행 전 매번 승인 요청 | 처음 사용할 때 |
| **편집 자동 승인 모드** | 파일 수정은 자동, 명령 실행은 승인 요청 | 파일 변경을 신뢰할 때 |
| **계획 모드** | 코드 분석 후 계획만 작성, 실제 수정 안 함 | 복잡한 작업 전 검토 시 |
| **권한 무시 모드** | 모든 승인 없이 자동 실행 | 격리된 환경에서만 사용 |

> 복잡한 작업은 **계획 모드**로 시작해서 방향을 잡은 뒤, **편집 자동 승인 모드**로 전환하는 것을 권장합니다.

## 앱 미리보기 기능

Claude가 개발 서버를 자동으로 시작하고 내장 브라우저에서 확인합니다. 파일을 수정할 때마다 자동으로 검증합니다.

미리보기 패널에서 할 수 있는 것들:
- 실행 중인 앱을 직접 조작
- Claude가 스크린샷 찍고, 버튼 클릭하고, 폼 입력하며 자동 검증
- 서버 시작/중지
- 세션 데이터 유지 (재시작해도 로그인 상태 유지)

---

## 변경사항 비교(Diff) 기능

Claude가 파일을 수정하면 `+12 -1` 같은 변경 통계가 나타납니다. 이를 클릭하면:

- 왼쪽: 수정된 파일 목록
- 오른쪽: 각 파일의 변경 내용

특정 줄을 클릭하면 댓글을 달 수 있습니다. 댓글을 여러 개 달고 `Cmd+Enter` (Mac) / `Ctrl+Enter` (Windows)를 누르면 Claude가 모든 댓글을 읽고 수정합니다.

## GitHub PR 모니터링

풀 리퀘스트를 생성하면 세션에 CI 상태 표시줄이 나타납니다.

| 기능 | 설명 |
|------|------|
| **자동 수정** | CI 테스트 실패 시 Claude가 자동으로 원인 분석 후 수정 |
| **자동 병합** | 모든 체크가 통과되면 자동으로 PR 병합 |

> PR 모니터링을 사용하려면 GitHub CLI(`gh`)가 설치되어 있어야 합니다.

---

## 병렬 세션으로 동시 작업

사이드바에서 **+ 새 세션**을 클릭하면 여러 작업을 동시에 진행할 수 있습니다. Git 저장소에서는 각 세션이 독립적인 작업 공간(worktree)을 사용하므로 서로 간섭하지 않습니다.

세션 종료 후 사이드바에서 마우스를 올리면 보관 아이콘이 나타나 세션을 정리할 수 있습니다.

---

## 클라우드에서 장시간 작업 실행

대규모 리팩토링, 테스트 실행, 마이그레이션 같은 오래 걸리는 작업은 **Remote** 환경을 선택하면 앱을 닫아도 계속 실행됩니다. claude.ai/code나 Claude 모바일 앱에서 진행 상황을 확인할 수 있습니다.

---

## 예약 작업 설정하기

사이드바에서 **일정**을 클릭하고 **+ 새 작업**을 선택합니다.

| 설정 항목 | 설명 |
|-----------|------|
| 이름 | 작업 식별자 |
| 설명 | 작업 목록에 표시될 간단한 설명 |
| 프롬프트 | 실행할 작업 지시사항 |
| 빈도 | 수동/시간별/매일/평일/매주 중 선택 |

예약 작업 예시:
- 매일 오전 9시: "오늘 커밋된 코드를 검토하고 잠재적 버그를 보고해줘"
- 매주 월요일: "의존성 패키지의 보안 업데이트 확인해줘"

> 예약 작업은 앱이 열려 있고 컴퓨터가 켜져 있을 때만 실행됩니다. 자는 동안 건너뛴 작업은 앱을 열면 한 번 실행됩니다.

---

## 외부 도구 연결 (커넥터)

입력창 옆 **+** 버튼 → **커넥터**를 클릭하면 다양한 서비스를 연결할 수 있습니다:
- Google 캘린더
- Slack
- GitHub
- Linear
- Notion

연결 후 Claude에게 "오늘 일정 확인해줘", "이슈 생성해줘" 같은 요청을 할 수 있습니다.

---

## SSH를 통한 원격 서버 연결

클라우드 VM이나 서버의 코드를 데스크탑 앱으로 작업할 수 있습니다.

환경 드롭다운에서 **+ SSH 연결 추가**를 선택하고:
- 이름: 연결 별칭
- SSH 호스트: `user@hostname` 형식
- SSH 포트: 기본값 22
- 키 파일: 인증 키 경로

> 원격 서버에도 Claude Code가 설치되어 있어야 합니다.

## CLI에서 온 사용자를 위한 비교

| CLI 명령/플래그 | 데스크탑 앱 동일 기능 |
|----------------|----------------------|
| `--model` | 전송 버튼 옆 모델 드롭다운 |
| `--resume`, `--continue` | 사이드바에서 세션 클릭 |
| `--permission-mode` | 전송 버튼 옆 모드 선택기 |
| `--dangerously-skip-permissions` | 설정 → Claude Code → 권한 무시 허용 |
| `--add-dir` | 원격 세션에서 + 버튼으로 저장소 추가 |

---

## 자주 발생하는 문제 해결

| 증상 | 해결 방법 |
|------|-----------|
| 403 인증 오류 | 앱 메뉴에서 로그아웃 후 재로그인 |
| 빈 화면 | 앱 재시작, 업데이트 확인 |
| 세션 로드 실패 | 폴더가 존재하는지, Git LFS가 설치되어 있는지 확인 |
| 도구를 찾을 수 없음 | 일반 터미널에서 도구 작동 확인, 앱 재시작 |
| Windows에서 Git 필요 | [Git for Windows](https://git-scm.com/downloads/win) 설치 후 재시작 |

## 데스크탑에서 사용 불가한 기능
