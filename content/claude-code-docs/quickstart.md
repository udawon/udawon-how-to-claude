# 빠른 시작

> 몇 분 안에 Claude Code로 AI 코딩 도움을 받아보세요!

---

## 시작 전 준비물

- 터미널(명령 프롬프트)이 열려 있어야 합니다
- 작업할 코드 프로젝트가 있어야 합니다
- Claude 구독 계정 (Pro, Max, Teams, Enterprise 중 하나)

> 터미널을 처음 사용하신다면 겁먹지 마세요. 아래 단계를 그대로 따라하면 됩니다.

---

## Step 1: Claude Code 설치

운영체제에 맞게 아래 명령어를 터미널에 입력하세요.

**macOS / Linux / WSL:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**
```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**
```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

> Windows 사용자는 먼저 [Git for Windows](https://git-scm.com/downloads/win)를 설치해야 합니다.

설치 후에는 자동으로 최신 버전으로 업데이트됩니다.

---

## Step 2: 로그인

터미널에서 `claude`를 입력하면 처음 실행 시 자동으로 로그인 화면이 나타납니다.

```bash
claude
# 첫 실행 시 로그인 안내가 나타납니다
```

또는 세션 안에서:
```
/login
```

로그인 방법:
- **Claude Pro/Max/Teams/Enterprise**: 가장 권장하는 방법
- **Claude Console (API)**: API 크레딧 방식
- **Amazon Bedrock / Google Vertex AI**: 기업용 클라우드

한 번 로그인하면 자격 증명이 저장되어 다음부터는 바로 사용 가능합니다.

---

## Step 3: 첫 세션 시작

터미널에서 프로젝트 폴더로 이동한 뒤 `claude`를 입력합니다.

```bash
cd /path/to/your/project
claude
```

Claude Code 환영 화면이 나타납니다. `/help`를 입력하면 사용 가능한 명령어를 볼 수 있습니다.

---

## Step 4: 첫 번째 질문 해보기

코드베이스를 이해하는 것부터 시작해보세요.

```
이 프로젝트가 무엇을 하는 건지 설명해줘
```

더 구체적인 질문도 가능합니다:
```
이 프로젝트에서 어떤 기술들을 사용하고 있어?
메인 진입점이 어디야?
폴더 구조를 설명해줘
```

Claude Code 자체에 대해 물어볼 수도 있습니다:
```
Claude Code로 뭘 할 수 있어?
Docker와 연동할 수 있어?
```

> Claude Code는 프로젝트 파일을 스스로 읽습니다. 직접 파일을 첨부하거나 경로를 알려줄 필요가 없습니다.

---

## Step 5: 첫 코드 변경 시도

이제 실제로 코드를 수정해봅시다.

```
메인 파일에 hello world 함수를 추가해줘
```

Claude Code가 수행하는 과정:
1. 적절한 파일을 찾습니다
2. 제안된 변경사항을 보여줍니다
3. 승인을 요청합니다
4. 수정을 적용합니다

> Claude Code는 파일을 수정하기 전에 항상 승인을 요청합니다. "전체 수락(Accept all)" 모드를 켜면 매번 묻지 않습니다.

---

## Step 6: Git과 함께 사용하기

Git 작업도 자연어로 처리할 수 있습니다.

```
내가 변경한 파일이 뭐야?
```

```
변경사항을 설명이 잘 된 메시지로 커밋해줘
```

```
feature/quickstart라는 새 브랜치를 만들어줘
```

```
마지막 5개 커밋을 보여줘
```

```
머지 충돌 해결을 도와줘
```

---

## Step 7: 버그 수정 또는 기능 추가

자연어로 원하는 것을 설명하세요.

```
사용자 등록 폼에 입력값 검증을 추가해줘
```

버그 수정:
```
사용자가 빈 폼을 제출할 수 있는 버그가 있어 - 고쳐줘
```

Claude Code가 수행하는 과정:
- 관련 코드를 찾습니다
- 맥락을 파악합니다
- 해결책을 구현합니다
- 테스트가 있으면 실행합니다

---

## Step 8: 다른 주요 워크플로우

**코드 리팩토링:**
```
authentication 모듈을 콜백 대신 async/await 방식으로 리팩토링해줘
```

**테스트 작성:**
```
calculator 함수들에 대한 유닛 테스트를 작성해줘
```

**문서 업데이트:**
```
README에 설치 방법을 업데이트해줘
```

**코드 리뷰:**
```
내 변경사항을 검토하고 개선점을 제안해줘
```

## 자주 쓰는 명령어 정리

| 명령어 | 역할 | 예시 |
|--------|------|------|
| `claude` | 대화형 모드 시작 | `claude` |
| `claude "작업"` | 일회성 작업 실행 | `claude "빌드 오류 수정해줘"` |
| `claude -p "질문"` | 한 번 물어보고 종료 | `claude -p "이 함수를 설명해줘"` |
| `claude -c` | 가장 최근 대화 이어서 | `claude -c` |
| `claude -r` | 이전 대화 선택해서 재개 | `claude -r` |
| `claude commit` | Git 커밋 생성 | `claude commit` |
| `/clear` | 대화 기록 초기화 | `/clear` |
| `/help` | 사용 가능한 명령어 보기 | `/help` |
| `exit` 또는 Ctrl+C | 종료 | `exit` |

## 예시 케이스

**케이스: 처음 사용하는 디자이너 준서 씨**

준서 씨는 개발자는 아니지만 작은 스타트업에서 간단한 웹사이트 수정을 맡게 되었습니다. Claude Code를 처음 설치하고 이렇게 입력했습니다.

```
이 프로젝트가 뭘 하는 건지 설명해줘
```

Claude가 전체 구조를 설명해 줬습니다. 그 다음:

```
메인 페이지에서 버튼 색상을 파란색에서 초록색으로 바꿔줘
```

Claude가 어떤 파일의 몇 번째 줄을 바꿀 건지 미리 보여주고, 준서 씨가 승인하자 자동으로 수정했습니다.

## 초보자를 위한 팁

**구체적으로 요청하기**

모호한 요청: `"버그 고쳐줘"`
좋은 요청: `"잘못된 자격 증명 입력 후 빈 화면이 뜨는 로그인 버그를 고쳐줘"`

**단계별로 지시하기**

복잡한 작업은 단계로 나눠서 요청하세요:
```
1. 사용자 프로필용 새 데이터베이스 테이블을 만들어줘
2. 사용자 프로필을 가져오고 업데이트하는 API 엔드포인트를 만들어줘
3. 사용자가 정보를 보고 수정할 수 있는 웹 페이지를 만들어줘
```

**구현 전에 탐색 먼저**

```
데이터베이스 스키마를 분석해줘
```
