# 모델 구성 가이드: 어떤 AI 모델을 쓸지 선택하기

Claude Code에는 여러 종류의 AI 모델이 있습니다. 마치 자동차를 고르는 것처럼, 상황에 따라 빠른 차(경차)를 탈지, 강력한 차(SUV)를 탈지 선택할 수 있습니다. 각 모델은 속도, 비용, 품질이 다릅니다.

---

## 사용 가능한 모델 별칭

복잡한 모델 이름을 외울 필요 없이, 아래 별칭을 사용하면 됩니다.

| 별칭 | 설명 | 언제 사용? |
|------|------|----------|
| `default` | 내 구독에 맞는 기본 모델 | 모르겠을 때 |
| `sonnet` | 최신 Sonnet 모델 | 일상적인 코딩 작업 |
| `opus` | 최신 Opus 모델 | 복잡한 추론, 어려운 문제 |
| `haiku` | 가장 빠르고 저렴한 모델 | 단순한 작업, 빠른 응답 |
| `sonnet[1m]` | 백만 토큰 컨텍스트의 Sonnet | 매우 긴 대화/코드베이스 |
| `opusplan` | 계획은 Opus, 실행은 Sonnet | 계획 후 구현 작업 |

> **컨텍스트(Context)란?** AI가 한 번에 기억할 수 있는 텍스트 양입니다. 더 큰 컨텍스트 = 더 긴 대화와 더 많은 코드를 동시에 처리 가능.

---

## 모델 구독별 기본값

| 구독 종류 | 기본 모델 |
|----------|---------|
| Max, Team Premium | Opus 4.6 |
| Pro, Team Standard | Sonnet 4.6 |
| Enterprise | Opus 4.6 (별도 신청 필요) |

---

## 모델 변경 방법

### 방법 1: 대화 중 변경 (즉시 적용)

```
/model sonnet
/model opus
/model haiku
```

### 방법 2: 시작할 때 지정

```bash
claude --model opus
```

### 방법 3: 환경 변수 설정

```bash
export ANTHROPIC_MODEL=sonnet
```

### 방법 4: 설정 파일에 영구 저장

`~/.claude/settings.json` 또는 `.claude/settings.json`에 추가:

```json
{
  "model": "opus"
}
```

## 특별 모드: opusplan

`opusplan`은 두 모델을 상황에 맞게 자동으로 전환합니다.

- **계획 단계**: Opus 사용 → 복잡한 추론, 아키텍처 설계
- **실행 단계**: Sonnet으로 전환 → 실제 코드 작성

이를 통해 품질과 비용을 모두 잡을 수 있습니다.

---

## 노력 수준(Effort Level) 조정

같은 모델이라도 "얼마나 깊이 생각할지"를 조절할 수 있습니다. Opus 4.6과 Sonnet 4.6에서 지원됩니다.

| 수준 | 설명 | 언제? |
|------|------|-------|
| `low` | 빠르게, 적게 생각 | 단순 작업, 빠른 답변 |
| `medium` | 균형 (기본값) | 일반 작업 |
| `high` | 깊이 생각 | 복잡하고 어려운 문제 |

**변경 방법:**
- `/model` 메뉴에서 화살표 키로 조절
- 환경 변수: `CLAUDE_CODE_EFFORT_LEVEL=low`
- 설정 파일: `"effortLevel": "high"`

---

## 백만 토큰 컨텍스트 (1M Context)

Opus 4.6과 Sonnet 4.6은 최대 100만 토큰(약 750,000 단어)까지 처리할 수 있습니다.

> **주의:** 현재 베타 기능이며, 200K 토큰을 초과하면 추가 요금이 발생합니다.

사용 방법:
```
/model sonnet[1m]
/model claude-sonnet-4-6[1m]
```

## 현재 모델 확인하기

- 화면 하단 상태 표시줄에서 확인
- `/status` 명령어로 확인

## 관리자: 모델 선택 제한하기

기업 환경에서 특정 모델만 허용하려면 `availableModels` 설정을 사용합니다.

```json
{
  "availableModels": ["sonnet", "haiku"],
  "model": "sonnet"
}
```

이렇게 하면 사용자는 sonnet과 haiku만 선택할 수 있고, 기본값은 sonnet입니다.

## 예시 케이스

**상황 1: 일상적인 코드 리뷰 작업**

> 프론트엔드 개발자 강개발 씨는 매일 코드 리뷰를 합니다. `sonnet` 모델이 속도와 품질 면에서 균형이 좋아 기본 설정으로 사용합니다. 빠르게 답변해줘서 작업 흐름이 끊기지 않습니다.

**상황 2: 복잡한 버그 추적**

> 백엔드 개발자 한백엔드 씨는 미스터리한 버그를 추적 중입니다. 이 경우는 `/model opus`로 전환하고, `effortLevel`을 `high`로 설정했습니다. Opus는 더 깊이 생각해서 원인을 찾아냈습니다.

**상황 3: 대규모 리팩토링 계획**

> 아키텍트 조아키 씨는 레거시 시스템 전체를 현대화하는 계획을 세워야 합니다. `opusplan` 모드를 사용했습니다. 계획 단계에서 Opus가 전체 구조와 전략을 설계하고, 실제 코드 작성 단계에서는 Sonnet으로 자동 전환되어 비용을 절약했습니다.

**상황 4: 매우 긴 코드베이스 분석**

> 기술 문서 담당자 문기술 씨는 10만 줄이 넘는 레거시 코드를 분석해야 합니다. `sonnet[1m]` 모드를 사용해서 전체 코드를 한번에 읽고 분석했습니다.
