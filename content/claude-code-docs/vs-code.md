# VS Code에서 Claude Code 사용하기

VS Code용 Claude Code 확장 프로그램을 사용하면 코드 편집기를 떠나지 않고도 AI 코딩 도우미를 활용할 수 있습니다. 마치 옆자리에 뛰어난 개발자가 앉아서 실시간으로 코드를 검토하고 수정해주는 것과 같습니다.

## 예시 케이스

> 디자이너 김지수 씨는 프론트엔드 개발자는 아니지만 간단한 회사 랜딩 페이지를 수정해야 합니다. VS Code에서 Claude Code 확장을 설치하고, 파일을 열고 오른쪽 상단의 불꽃 아이콘을 클릭합니다. "이 파일에서 버튼 색상을 파란색으로 바꿔줘"라고 입력하면 Claude가 정확히 어느 줄을 어떻게 바꿀지 보여주고, 승인하면 자동으로 적용됩니다.

---

## 시작 전 준비사항

| 항목 | 요구사항 |
|------|----------|
| VS Code 버전 | 1.98.0 이상 |
| 계정 | Anthropic 계정 (처음 실행 시 로그인) |

---

## 설치 방법

1. VS Code에서 `Ctrl+Shift+X` (Windows) 또는 `Cmd+Shift+X` (Mac)를 눌러 확장 프로그램 탭을 엽니다.
2. "Claude Code"를 검색합니다.
3. **설치** 버튼을 클릭합니다.
4. 설치 후 아이콘이 보이지 않으면 VS Code를 재시작합니다.

> Cursor IDE를 사용하신다면 동일한 방법으로 설치할 수 있습니다.

---

## 기본 사용법

### 1단계: Claude 패널 열기

화면 오른쪽 상단 에디터 툴바에서 불꽃(✦) 아이콘을 클릭합니다. 파일이 열려 있을 때만 나타납니다.

다른 방법으로도 열 수 있습니다:
- **왼쪽 사이드바**: 불꽃 아이콘 클릭 → 대화 목록 표시
- **명령 팔레트**: `Ctrl+Shift+P` → "Claude Code" 입력
- **상태 표시줄**: 하단 오른쪽의 **✱ Claude Code** 클릭 (파일 없어도 가능)

### 2단계: 질문 또는 요청 보내기

텍스트 입력창에 원하는 작업을 자연어로 입력합니다. 예를 들어:
- "이 함수가 어떻게 동작하는지 설명해줘"
- "로그인 버튼이 클릭되지 않는 버그를 찾아줘"
- "이 코드의 변수명을 더 이해하기 쉽게 바꿔줘"

코드를 드래그해서 선택하면 Claude가 자동으로 그 부분을 인식합니다. `Alt+K` (Windows) / `Option+K` (Mac)를 누르면 선택한 파일과 줄 번호가 자동으로 입력됩니다.

### 3단계: 변경사항 검토하기

Claude가 파일을 수정하려 할 때 좌우로 비교 화면(diff)을 보여줍니다. **승인** 또는 **거부** 버튼으로 각 변경사항을 결정할 수 있습니다.

## 프롬프트 입력창 주요 기능

| 기능 | 설명 |
|------|------|
| **권한 모드** | 입력창 하단의 모드 버튼으로 전환. 일반/계획/자동 승인 모드 선택 가능 |
| **파일 참조 (@멘션)** | `@파일명`을 입력하면 Claude가 해당 파일을 읽음. 폴더는 `@폴더명/`으로 참조 |
| **컨텍스트 표시** | 입력창 하단에서 Claude의 기억 공간 사용량 확인 가능 |
| **여러 줄 입력** | `Shift+Enter`로 줄바꿈 (전송 없이) |
| **확장 사고 모드** | `/` 메뉴에서 복잡한 문제를 더 깊이 분석하도록 설정 가능 |

### 권한 모드 비교

| 모드 | 동작 방식 |
|------|-----------|
| **일반 모드** | 모든 작업 전 Claude가 승인 요청 |
| **계획 모드** | Claude가 먼저 계획을 보여주고, 승인 후 실행 |
| **자동 승인 모드** | Claude가 묻지 않고 자동으로 편집 실행 |

---

## 여러 대화 동시 진행

명령 팔레트에서 "Open in New Tab" 또는 "Open in New Window"를 선택하면 여러 대화를 동시에 진행할 수 있습니다. 각 대화는 독립적인 기록과 맥락을 유지합니다.

---

## 과거 대화 이어서 진행하기

Claude Code 패널 상단의 드롭다운을 클릭하면 이전 대화 목록이 나타납니다. 날짜별로 정리되어 있으며, 클릭하면 이전 대화를 이어서 진행할 수 있습니다.

---

## 주요 단축키

| 단축키 | 기능 |
|--------|------|
| `Cmd+Esc` / `Ctrl+Esc` | 에디터와 Claude 사이 포커스 전환 |
| `Cmd+Shift+Esc` / `Ctrl+Shift+Esc` | 새 탭에서 대화 열기 |
| `Cmd+N` / `Ctrl+N` | 새 대화 시작 (Claude 포커스 상태에서) |
| `Option+K` / `Alt+K` | 현재 선택한 코드 참조 삽입 |

## 설정 옵션

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `useTerminal` | 꺼짐 | 터미널 방식으로 Claude 실행 |
| `initialPermissionMode` | 기본 | 초기 권한 모드 설정 |
| `autosave` | 켜짐 | Claude 작업 전 파일 자동 저장 |
| `useCtrlEnterToSend` | 꺼짐 | Enter 대신 Ctrl+Enter로 전송 |
| `respectGitIgnore` | 켜짐 | .gitignore 파일의 패턴 제외 |

## Chrome 브라우저 연동

Claude Code와 Chrome 확장을 함께 사용하면, 웹 앱을 테스트하고 콘솔 오류를 직접 확인하며 브라우저 자동화 작업을 수행할 수 있습니다.

입력창에 `@browser`를 입력한 후 작업을 요청합니다:
```
@browser localhost:3000에 접속해서 콘솔 오류 확인해줘
```

## 자주 발생하는 문제 해결

| 증상 | 해결 방법 |
|------|-----------|
| 불꽃 아이콘이 보이지 않음 | 파일을 열었는지 확인, VS Code 1.98.0 이상인지 확인 |
| 확장이 설치되지 않음 | VS Code 버전 확인, [마켓플레이스](https://marketplace.visualstudio.com/items?itemName=anthropic.claude-code)에서 직접 설치 |
| Claude가 응답하지 않음 | 인터넷 연결 확인, 새 대화 시작, 터미널에서 `claude` 실행해보기 |

---

## 보안 안내

내 코드는 모델 학습에 사용되지 않습니다. 단, 자동 승인 모드를 활성화하면 Claude가 VS Code 설정 파일을 수정할 수도 있으니, 신뢰할 수 없는 코드를 다룰 때는 일반 승인 모드를 사용하는 것을 권장합니다.
