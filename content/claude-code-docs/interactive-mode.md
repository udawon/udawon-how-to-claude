# 대화형 모드

> Claude Code 세션에서 사용할 수 있는 모든 단축키, 명령어, 입력 방식의 완전한 참고 문서입니다.

대화형 모드는 Claude Code를 직접 조종하는 **운전석**과 같습니다. 핸들(단축키)로 방향을 바꾸고, 계기판(명령어)으로 상태를 확인하고, 내비게이션(자동 제안)으로 다음 할 일을 안내받습니다. 이 문서는 운전석의 모든 버튼과 레버를 정리한 매뉴얼입니다.

---

## 키보드 단축키

> macOS 사용자는 `Option/Alt` 키 단축키(`Alt+B`, `Alt+F` 등)가 동작하려면 터미널 설정이 필요합니다.
> - **iTerm2**: 설정 → Profiles → Keys → Left/Right Option key를 "Esc+"로 설정
> - **Terminal.app**: 설정 → Profiles → Keyboard → "Use Option as Meta Key" 체크

### 일반 제어

| 단축키 | 설명 |
|--------|------|
| `Ctrl+C` | 현재 입력 또는 생성 취소 |
| `Ctrl+D` | Claude Code 세션 종료 |
| `Ctrl+G` | 기본 텍스트 에디터에서 프롬프트 편집 |
| `Ctrl+L` | 터미널 화면 지우기 (대화 기록은 유지) |
| `Ctrl+O` | 상세 출력 토글 (도구 사용 과정 보기) |
| `Ctrl+R` | 이전 명령어 검색 |
| `Ctrl+V` / `Cmd+V` | 클립보드의 이미지 붙여넣기 |
| `Ctrl+B` | 현재 실행 중인 명령어를 백그라운드로 이동 |
| `Ctrl+T` | 작업 목록 보이기/숨기기 |
| `Esc + Esc` | 이전 상태로 되돌리기 (Rewind) |
| `Shift+Tab` | 권한 모드 전환 (기본 → 자동수락 → 계획 모드) |
| `Option+P` (macOS) / `Alt+P` | 모델 전환 |
| `Option+T` (macOS) / `Alt+T` | 확장 사고 모드 켜기/끄기 |

### 텍스트 편집

| 단축키 | 설명 |
|--------|------|
| `Ctrl+K` | 커서부터 줄 끝까지 삭제 |
| `Ctrl+U` | 전체 줄 삭제 |
| `Ctrl+Y` | 삭제한 텍스트 붙여넣기 |
| `Alt+B` | 한 단어 뒤로 이동 |
| `Alt+F` | 한 단어 앞으로 이동 |

### 여러 줄 입력

긴 메시지나 코드를 여러 줄로 입력하고 싶을 때:

| 방법 | 단축키 | 호환 터미널 |
|------|--------|-----------|
| 빠른 방법 | `\` + `Enter` | 모든 터미널 |
| macOS 기본 | `Option+Enter` | macOS |
| Shift+Enter | `Shift+Enter` | iTerm2, WezTerm, Ghostty, Kitty |
| Ctrl+J | `Ctrl+J` | 모든 터미널 |
| 붙여넣기 | 직접 붙여넣기 | 코드 블록, 로그용 |

> VS Code, Alacritty, Zed, Warp 터미널에서는 `/terminal-setup` 명령어로 Shift+Enter 단축키를 설치하세요.

## 빠른 명령어 접두사

| 접두사 | 설명 |
|--------|------|
| `/` (줄 시작) | 내장 명령어 또는 스킬 실행 |
| `!` (줄 시작) | 셸 명령어 직접 실행 (Claude 거치지 않음) |
| `@` | 파일 경로 자동완성 |

## 내장 명령어 전체 목록

세션 안에서 `/`를 입력하면 사용 가능한 모든 명령어가 보입니다.

### 자주 쓰는 핵심 명령어

| 명령어 | 설명 |
|--------|------|
| `/help` | 도움말 및 사용 가능한 명령어 보기 |
| `/clear` | 대화 기록 초기화 (별칭: `/reset`, `/new`) |
| `/compact [지시사항]` | 대화를 요약해 컨텍스트 절약 |
| `/context` | 현재 컨텍스트 사용량 시각화 |
| `/cost` | 토큰 사용 통계 보기 |
| `/exit` | CLI 종료 (별칭: `/quit`) |
| `/model [모델명]` | AI 모델 선택 또는 변경 |
| `/resume [세션]` | 이전 대화 재개 (별칭: `/continue`) |

### 세션 관리

| 명령어 | 설명 |
|--------|------|
| `/rename [이름]` | 현재 세션 이름 변경 |
| `/fork [이름]` | 현재 대화를 이 시점에서 분기 |
| `/rewind` | 이전 체크포인트로 되돌리기 (별칭: `/checkpoint`) |
| `/export [파일명]` | 현재 대화를 텍스트로 내보내기 |

### Git & 코드 작업

| 명령어 | 설명 |
|--------|------|
| `/diff` | 커밋되지 않은 변경사항 인터랙티브 diff 뷰어 |
| `/pr-comments [PR번호]` | GitHub PR 댓글 가져오기 |
| `/security-review` | 현재 브랜치 변경사항 보안 취약점 분석 |
| `/init` | 프로젝트에 CLAUDE.md 초기화 |

### 설정 및 구성

| 명령어 | 설명 |
|--------|------|
| `/config` | 설정 인터페이스 열기 (별칭: `/settings`) |
| `/permissions` | 권한 보기/수정 (별칭: `/allowed-tools`) |
| `/hooks` | 훅(Hook) 설정 관리 |
| `/memory` | CLAUDE.md 메모리 파일 편집 |
| `/theme` | 색상 테마 변경 |
| `/vim` | Vim 편집 모드 켜기/끄기 |

### 도구 및 확장

| 명령어 | 설명 |
|--------|------|
| `/mcp` | MCP 서버 연결 관리 |
| `/skills` | 사용 가능한 스킬 목록 |
| `/agents` | 에이전트 설정 관리 |
| `/plugin` | Claude Code 플러그인 관리 |

### 유틸리티

| 명령어 | 설명 |
|--------|------|
| `/btw [질문]` | 대화 기록에 남기지 않는 빠른 질문 |
| `/plan` | 계획 모드로 직접 진입 |
| `/copy` | 마지막 응답을 클립보드에 복사 |
| `/insights` | 세션 분석 리포트 생성 |
| `/doctor` | Claude Code 설치 및 설정 진단 |
| `/login` / `/logout` | 계정 로그인/로그아웃 |

## Vim 편집 모드

`/vim` 명령 또는 `/config`에서 영구적으로 Vim 스타일 편집을 활성화할 수 있습니다.

### 모드 전환

| 명령 | 동작 | 현재 모드 |
|------|------|-----------|
| `Esc` | NORMAL 모드 진입 | INSERT |
| `i` | 커서 앞에 삽입 | NORMAL |
| `a` | 커서 뒤에 삽입 | NORMAL |
| `A` | 줄 끝에 삽입 | NORMAL |
| `o` | 아래에 새 줄 추가 | NORMAL |

### NORMAL 모드 탐색

| 명령 | 동작 |
|------|------|
| `h/j/k/l` | 좌/하/상/우 이동 |
| `w` | 다음 단어로 |
| `b` | 이전 단어로 |
| `0` | 줄 시작으로 |
| `$` | 줄 끝으로 |
| `gg` | 입력 시작으로 |
| `G` | 입력 끝으로 |

### NORMAL 모드 편집

| 명령 | 동작 |
|------|------|
| `x` | 문자 삭제 |

## 백그라운드 명령어 실행

오래 걸리는 작업을 백그라운드에서 실행하면 Claude와 계속 대화할 수 있습니다.

### 백그라운드 실행 방법
- Claude Code에게 백그라운드로 실행해달라고 요청
- 명령어 실행 중 `Ctrl+B` 누르기

자주 백그라운드로 실행하는 명령어:
- 빌드 도구 (webpack, vite, make)
- 패키지 매니저 (npm, yarn)
- 테스트 실행기 (jest, pytest)
- 개발 서버
- 오래 걸리는 작업 (docker, terraform)

### `!` 접두사로 직접 실행

```bash
! npm test
! git status
! ls -la
```

이 방법은:
- Claude를 거치지 않고 셸 명령어 직접 실행
- 명령어와 출력이 대화 컨텍스트에 추가됨
- 실시간 진행 상황 표시
- Tab으로 이전 `!` 명령어 자동완성 지원

## /btw — 빠른 사이드 질문

대화 기록에 남기지 않고 빠르게 물어볼 때 사용합니다.

```
/btw 그 설정 파일 이름이 뭐였지?
```

**특징:**
- Claude가 작업 중에도 사용 가능
- 대화 기록에 추가되지 않음 (오버레이로 표시 후 사라짐)
- 파일 읽기나 명령어 실행은 불가 (이미 알고 있는 정보만 답변)
- 후속 질문 없음 (단일 응답)

## 프롬프트 자동 제안

세션을 시작하면 최근 Git 히스토리를 기반으로 관련 작업 제안이 나타납니다.

- `Tab`: 제안 수락
- `Enter`: 제안 수락 후 바로 실행
- 타이핑 시작: 제안 무시

---

## 작업 목록 (Task List)

복잡한 다단계 작업 시 Claude가 자동으로 작업 목록을 만들어 진행 상황을 추적합니다.

- `Ctrl+T`: 작업 목록 보이기/숨기기
- 최대 10개 작업 표시
- 컨텍스트 압축 이후에도 작업 목록 유지

---

## PR 리뷰 상태 표시

브랜치에 열린 PR이 있으면 하단에 PR 링크가 표시됩니다.

| 색상 | 상태 |
|------|------|
| 초록 | 승인됨 |
| 노랑 | 검토 대기 중 |
| 빨강 | 수정 요청 |
| 회색 | 드래프트 |
| 보라 | 머지됨 |

`Cmd+클릭` (Mac) 또는 `Ctrl+클릭` (Windows/Linux)으로 PR을 브라우저에서 열 수 있습니다.

> PR 상태는 `gh` CLI가 설치되고 인증된 경우에만 표시됩니다 (`gh auth login`).

## 예시 케이스

<div class="example-case">
<strong>케이스: 장시간 작업 중 다른 일 하기</strong>

개발자 성훈 씨는 대규모 마이그레이션을 Claude에게 맡기고 다른 작업을 하고 싶습니다.

1. `/hooks`에서 `Notification` 이벤트 설정
2. 명령어 입력 (macOS 기준):
   ```
   osascript -e 'display notification "Claude Code 확인 필요" with title "Claude Code"'
   ```
3. `Ctrl+B`로 백그라운드로 보냄
4. 다른 작업 진행
5. 알림이 오면 돌아와서 확인

</div>

<div class="example-case">
<strong>케이스: 빠른 정보 확인</strong>

Claude가 대규모 리팩토링 중인데, 잠깐 설정 파일 이름이 기억나지 않는 승아 씨:

```
/btw 우리 eslint 설정 파일 이름이 뭐야?
```

Claude의 작업을 방해하지 않고 즉시 답변을 받습니다. 대화 기록에도 남지 않습니다.

</div>
