# Claude가 프로젝트를 기억하는 방법

> Claude Code는 매 대화를 새로 시작합니다. 하지만 두 가지 방법으로 중요한 정보를 기억하게 할 수 있습니다.

## 핵심 개념: 메모리의 두 종류

Claude Code가 정보를 기억하는 방식은 마치 두 종류의 메모장과 같습니다.

| 구분 | CLAUDE.md 파일 | 자동 메모리 |
|------|----------------|------------|
| **누가 작성하나요?** | 사용자(내가 직접) | Claude가 스스로 |
| **무엇을 담나요?** | 지시사항과 규칙 | Claude가 배운 것들 |
| **적용 범위** | 프로젝트·사용자·조직 | 해당 작업 폴더 |
| **언제 불러오나요?** | 매 대화 시작 시 | 매 대화 시작 시 (첫 200줄) |
| **어떤 용도에 적합한가요?** | 코딩 스타일, 워크플로우, 아키텍처 규칙 | 빌드 명령어, 디버깅 패턴, 발견한 선호도 |

## CLAUDE.md 파일

### CLAUDE.md란 무엇인가요?

**CLAUDE.md**는 Claude에게 보내는 "업무 지침서"입니다. 마치 신입 직원에게 "우리 팀은 이렇게 일해요"라고 알려주는 온보딩 문서처럼, Claude가 매 대화 시작 시 읽는 마크다운 파일입니다.

### 어디에 저장하나요?

저장 위치에 따라 적용 범위가 달라집니다. 더 구체적인 위치가 더 넓은 위치보다 우선합니다.

| 범위 | 저장 위치 | 용도 | 공유 대상 |
|------|-----------|------|-----------|
| **조직 전체** | Windows: `C:\Program Files\ClaudeCode\CLAUDE.md` | 회사 전체 코딩 기준, 보안 정책 | 조직의 모든 사용자 |
| **프로젝트** | `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` | 팀 공유 프로젝트 지침 | 팀원 전체 (버전 관리) |
| **개인** | `~/.claude/CLAUDE.md` | 나만의 개인 설정 | 나의 모든 프로젝트 |

### 효과적인 CLAUDE.md 작성법

**크기**: 파일 하나당 200줄 이하를 목표로 하세요. 더 길면 Claude가 덜 잘 따릅니다.

**구체성**: 검증 가능한 지시사항을 작성하세요.

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "코드를 잘 포맷하세요" | "들여쓰기는 2칸을 사용하세요" |
| "변경 전에 테스트하세요" | "커밋 전에 `npm test`를 실행하세요" |
| "파일을 정리하세요" | "API 핸들러는 `src/api/handlers/`에 위치합니다" |

### 다른 파일 불러오기 (임포트)

`@경로/파일명` 문법으로 다른 파일을 불러올 수 있습니다.

```text
프로젝트 개요는 @README, 사용 가능한 npm 명령어는 @package.json을 참고하세요.

# 추가 지침
- git 워크플로우 @docs/git-instructions.md
```

### .claude/rules/ 디렉토리로 규칙 정리하기

대규모 프로젝트에서는 규칙을 여러 파일로 나눌 수 있습니다.

```
your-project/
├── .claude/
│   ├── CLAUDE.md           # 주요 프로젝트 지침
│   └── rules/
│       ├── code-style.md   # 코드 스타일 가이드
│       ├── testing.md      # 테스트 규칙
│       └── security.md     # 보안 요구사항
```

**경로별 규칙(Path-specific rules)**을 사용하면 특정 파일 타입에만 적용되는 규칙을 만들 수 있습니다.

```markdown
---
paths:
  - "src/api/**/*.ts"
---

# API 개발 규칙

- 모든 API 엔드포인트에 입력 검증을 포함해야 합니다
- 표준 오류 응답 형식을 사용하세요
```

## 자동 메모리 (Auto Memory)

### 자동 메모리란 무엇인가요?

자동 메모리는 Claude가 스스로 메모를 남기는 기능입니다. 마치 좋은 비서가 "이 분은 npm 대신 pnpm을 좋아하시는구나"를 기억해두는 것처럼, Claude가 여러분과의 대화에서 유용한 정보를 자동으로 저장합니다.

저장되는 내용: 빌드 명령어, 디버깅 패턴, 아키텍처 메모, 코드 스타일 선호도, 워크플로우 습관 등

### 저장 위치

각 프로젝트별로 별도의 메모리 폴더가 생성됩니다:

```
~/.claude/projects/<project>/memory/
├── MEMORY.md          # 간결한 색인 (매 대화마다 불러옴)
├── debugging.md       # 디버깅 패턴 상세 메모
├── api-conventions.md # API 설계 결정사항
└── ...                # Claude가 생성한 기타 주제 파일
```

### 자동 메모리 켜고 끄기

기본적으로 켜져 있습니다. `/memory` 명령으로 토글하거나, 설정 파일에서 변경할 수 있습니다.

```json
{
  "autoMemoryEnabled": false
}
```

## /memory 명령어

`/memory` 명령을 입력하면:
- 현재 세션에 불러온 모든 CLAUDE.md 및 rules 파일 목록 확인
- 자동 메모리 켜기/끄기 토글
- 자동 메모리 폴더 바로 열기

**직접 기억시키기**: "항상 pnpm을 사용하고, npm은 쓰지 마세요"처럼 말하면 Claude가 자동 메모리에 저장합니다. CLAUDE.md에 저장하고 싶다면 "이걸 CLAUDE.md에 추가해 주세요"라고 하면 됩니다.

## 예시 케이스

### 케이스 1: 팀 프로젝트 설정

**상황**: 새로운 팀원이 프로젝트에 합류할 때마다 Claude Code 사용 방법을 설명해야 합니다.

**해결책**: 프로젝트 루트에 `.claude/CLAUDE.md`를 만들어 버전 관리에 포함합니다.

```markdown
# 쇼핑몰 프로젝트 지침

## 빌드 & 테스트
- 빌드: `npm run build`
- 테스트: `npm test`
- 로컬 서버: `npm run dev` (포트 3000)

## 코딩 규칙
- TypeScript strict 모드 사용
- 컴포넌트는 PascalCase 명명
- API 핸들러는 `src/api/handlers/`에 위치

## 아키텍처
- 상태관리: Zustand 사용
- 스타일링: Tailwind CSS
- DB 접근: Prisma ORM 사용
```

새 팀원이 Claude Code를 열면 자동으로 이 지침을 읽고 프로젝트 맥락을 이해합니다.

### 케이스 2: 개인 선호도 저장

**상황**: Claude가 매번 npm 대신 pnpm을 쓰도록 설정하고 싶습니다.

**해결책**: Claude에게 말합니다: "항상 npm 대신 pnpm을 사용해 주세요. 이걸 기억해 두세요."

Claude는 `~/.claude/projects/<프로젝트>/memory/MEMORY.md`에 이 선호도를 저장합니다.

### 케이스 3: 파일 타입별 규칙 분리

**상황**: API 파일, 컴포넌트 파일, 테스트 파일마다 다른 규칙을 적용하고 싶습니다.

**해결책**: `.claude/rules/` 디렉토리에 파일별 규칙 파일을 만들고 `paths` 프론트매터로 범위를 지정합니다.

## 자주 겪는 문제와 해결법

### Claude가 CLAUDE.md 내용을 따르지 않아요

CLAUDE.md는 강제 설정이 아니라 맥락(context)입니다. 지시사항이 모호하거나 서로 충돌하면 Claude가 임의로 선택할 수 있습니다.

**확인 방법**:
1. `/memory`를 실행해 CLAUDE.md 파일이 실제로 불려오는지 확인
2. 지시사항을 더 구체적으로 작성
3. 여러 CLAUDE.md 파일 간 충돌하는 지시사항이 없는지 확인

### CLAUDE.md가 너무 커졌어요

200줄을 초과하면 Claude의 준수율이 떨어집니다. `@path` 임포트로 내용을 분리하거나 `.claude/rules/` 파일로 나누세요.

### `/compact` 후 지시사항이 사라져요

CLAUDE.md는 컴팩션(압축) 후에도 완전히 살아남습니다. 사라진 지시사항은 대화 중에만 말했고 CLAUDE.md에 기록되지 않은 것입니다. CLAUDE.md에 추가하면 모든 세션에 걸쳐 유지됩니다.
