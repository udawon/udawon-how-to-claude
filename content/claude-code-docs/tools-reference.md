## 이게 뭔가요?

도구(Tool)란 **Claude Code가 실제로 행동을 취할 때 사용하는 기능 하나하나**를 말합니다.

목수의 도구 상자를 생각해보세요. 톱, 망치, 줄자, 드릴 — 각각 용도가 다릅니다. 나무를 자를 때는 톱을, 못을 박을 때는 망치를 꺼냅니다. Claude Code도 마찬가지입니다. 파일을 읽을 때는 Read 도구를, 코드를 실행할 때는 Bash 도구를, 파일을 수정할 때는 Edit 도구를 꺼냅니다.

중요한 점은, 일부 도구는 **사용 허가가 필요**합니다. 목수가 전기 드릴을 쓰려면 안전 교육을 받아야 하듯, Claude도 파일을 삭제하거나 명령어를 실행할 때는 사용자의 승인을 먼저 받습니다.

---

## 왜 알아야 하나요?

- Claude가 "이 도구를 사용해도 되나요?"라고 물어볼 때, **그 도구가 무슨 일을 하는지** 알아야 허용할지 판단할 수 있습니다
- Claude에게 작업을 시킬 때, **어떤 도구가 있는지 알면 더 정확한 요청**을 할 수 있습니다
- 권한(Permission, 도구 사용 허가) 설정을 할 때 각 도구의 위험도를 이해해야 합니다
- CLAUDE.md나 스킬(Skill, Claude의 전문 기능 모듈) 파일에서 도구를 언급할 때 정확한 이름을 알아야 합니다

---

## 어떻게 하나요?

### 도구 권한 설정하기

Claude Code는 위험할 수 있는 도구를 사용하기 전에 허가를 요청합니다. 매번 묻는 것이 번거롭다면 `.claude/settings.json`에서 자동 허용 규칙을 설정할 수 있습니다.

```json
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Write"],
    "deny": ["Bash(rm -rf *)"]
  }
}
```

<div class="example-case">
<strong>예시: 파일 읽기는 항상 허용하되, 삭제 명령은 차단하기</strong>

`.claude/settings.json` 파일에 위처럼 설정하면:
- Read, Glob, Grep, Write 도구는 묻지 않고 바로 실행됩니다
- `rm -rf *` 같은 위험한 Bash 명령어는 항상 차단됩니다
- 그 외 도구는 사용할 때마다 허가를 물어봅니다

</div>

### 도구가 동작하는 방식

Claude는 대화 중에 필요한 도구를 자동으로 선택합니다. "이 파일을 읽어줘"라고 하면 Read 도구를, "테스트를 실행해줘"라고 하면 Bash 도구를 사용합니다. 사용자가 직접 도구 이름을 지정할 필요는 없습니다.

## 전체 도구 목록

### 파일 도구 (File Tools)

파일을 읽고, 쓰고, 검색하는 도구들입니다.

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **Read** | 파일 내용 읽기. 이미지, PDF, 노트북 파일도 읽을 수 있음 | 아니오 |
| **Write** | 새 파일 생성 또는 기존 파일 전체 덮어쓰기 | 예 |
| **Edit** | 파일의 특정 부분만 수정 (전체를 다시 쓰지 않고 변경된 부분만 교체) | 예 |
| **Glob** | 파일 이름 패턴으로 검색 (예: `*.js`로 모든 JavaScript 파일 찾기) | 아니오 |
| **Grep** | 파일 내용에서 텍스트 검색 (ripgrep 기반의 빠른 검색) | 아니오 |
| **NotebookEdit** | Jupyter 노트북(데이터 분석용 대화형 코딩 환경) 셀 편집 | 예 |

### 실행 도구 (Execution Tools)

명령어를 실행하거나, 별도의 AI 에이전트를 띄우는 도구입니다.

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **Bash** | 터미널 명령어 실행. 작업 디렉토리는 호출 간에 유지되지만 셸 상태(변수 등)는 초기화됨 | 예 |
| **Agent** | 별도의 하위 에이전트(AI 보조원)를 생성하여 독립적인 작업을 수행 | 아니오 |

> **Bash 도구의 특성:** Bash 도구로 명령어를 실행할 때, 작업 폴더(현재 디렉토리)는 유지됩니다. 하지만 `export`로 설정한 변수나 셸 함수는 다음 호출 시 초기화됩니다. 필요한 설정은 매번 다시 지정해야 합니다.

### 사용자 상호작용 도구 (User Interaction Tools)

사용자와 대화하거나, 작업 모드를 전환하는 도구입니다.

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **AskUserQuestion** | 사용자에게 질문하고 답변을 기다림 | 아니오 |
| **EnterPlanMode** | 계획 모드(Plan Mode)로 전환 — 코드를 수정하지 않고 분석만 하는 모드 | 아니오 |
| **ExitPlanMode** | 계획 모드를 종료하고 실행 모드로 복귀 | 아니오 |

### 작업 관리 도구 (Task Management Tools)

하위 에이전트의 작업을 만들고 추적하는 도구입니다.

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **TaskCreate** | 새 작업(하위 에이전트가 수행할 일)을 생성 | 아니오 |
| **TaskGet** | 특정 작업의 현재 상태를 조회 | 아니오 |
| **TaskList** | 모든 작업 목록을 조회 | 아니오 |
| **TaskUpdate** | 작업 상태를 업데이트 | 아니오 |
| **TaskOutput** | 완료된 작업의 결과를 가져옴 | 아니오 |
| **TaskStop** | 실행 중인 작업을 중지 | 아니오 |
| **TodoWrite** | 할 일 목록(체크리스트)을 작성하여 작업 진행 상황을 추적 | 아니오 |

### 웹 도구 (Web Tools)

인터넷에서 정보를 가져오는 도구입니다.

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **WebFetch** | 지정한 URL의 웹 페이지 내용을 가져옴 | 예 |
| **WebSearch** | 웹 검색을 수행하고 결과를 반환 | 예 |

### MCP 도구 (MCP Tools)

MCP(Model Context Protocol, 외부 도구 연결 규격)를 통해 연결된 외부 서비스와 상호작용하는 도구입니다.

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **ToolSearch** | 지연 로딩(필요할 때만 불러오기)된 도구의 스키마(구조 정의)를 검색 | 아니오 |
| **ListMcpResourcesTool** | MCP 서버가 제공하는 리소스(자원) 목록을 조회 | 아니오 |
| **ReadMcpResourceTool** | MCP 서버의 특정 리소스를 읽어옴 | 아니오 |

### 기타 도구 (Other Tools)

| 도구 이름 | 기능 | 허가 필요 |
|-----------|------|-----------|
| **Skill** | 미리 정의된 스킬(전문 기능 모듈)을 실행 | 아니오 |
| **CronCreate** | 크론 작업(정해진 시간에 자동 실행되는 예약 작업)을 생성 | 예 |
| **CronDelete** | 크론 작업을 삭제 | 예 |
| **CronList** | 등록된 크론 작업 목록을 조회 | 아니오 |
| **EnterWorktree** | 워크트리(동일 저장소의 별도 작업 공간)로 진입 | 아니오 |
| **ExitWorktree** | 워크트리에서 빠져나옴 | 아니오 |
| **LSP** | Language Server Protocol(코드 편집기의 자동 완성·오류 검사 기능)과 상호작용 | 아니오 |

## 실전 예시

<div class="example-case">
<strong>실전 케이스: "파일 수정 좀 해줘"라고 했을 때 Claude가 쓰는 도구 흐름</strong>

사용자: "src/utils/helper.js에서 formatDate 함수의 날짜 형식을 YYYY-MM-DD로 바꿔줘"

Claude가 내부적으로 수행하는 순서:
1. **Read** — `src/utils/helper.js` 파일을 읽어서 현재 코드 확인
2. **Grep** — `formatDate` 함수가 정확히 어디에 있는지 검색
3. **Edit** — 해당 부분만 찾아서 날짜 형식을 수정 (사용자 승인 필요)
4. **Bash** — `npm test` 실행하여 수정이 다른 곳을 깨뜨리지 않았는지 확인 (사용자 승인 필요)

사용자는 3번과 4번에서 "허용" 버튼을 눌러야 합니다.

</div>

<div class="example-case">
<strong>실전 케이스: 웹에서 정보를 찾아서 코드에 반영하기</strong>

사용자: "최신 React Router 문법으로 라우팅 코드를 업데이트해줘"

Claude가 내부적으로 수행하는 순서:
1. **WebSearch** — "React Router latest syntax" 검색 (사용자 승인 필요)
2. **WebFetch** — 공식 문서 페이지 내용을 가져옴 (사용자 승인 필요)
3. **Glob** — 프로젝트에서 라우팅 관련 파일을 찾음 (`**/router*`, `**/routes*`)
4. **Read** — 해당 파일들의 현재 코드를 확인
5. **Edit** — 새로운 문법으로 수정 (사용자 승인 필요)

총 3번의 승인이 필요합니다. 자주 쓰는 도구(Read, Glob, Grep)는 권한 설정에서 자동 허용하면 편합니다.

</div>

## 주의할 점

- **Write vs Edit:** Write는 파일 전체를 덮어씁니다. 기존 파일의 일부만 바꾸고 싶다면 Edit를 사용하세요. Write를 잘못 쓰면 기존 내용이 사라질 수 있습니다
- **Bash 도구의 위험성:** `rm -rf`(전체 삭제), `git push --force`(강제 덮어쓰기) 같은 명령어는 되돌릴 수 없습니다. Claude가 Bash 실행 허가를 요청하면, 명령어 내용을 반드시 확인하세요
- **Bash 셸 상태 초기화:** Bash 도구로 `cd /some/path`를 실행해도, 다음 Bash 호출에서는 원래 디렉토리로 돌아갑니다. 절대 경로를 사용하는 것이 안전합니다
- **Agent 도구와 컨텍스트:** Agent가 생성한 하위 에이전트는 별도의 컨텍스트(대화 맥락)를 가집니다. 메인 대화에서 했던 내용을 자동으로 알지 못합니다
- **MCP 도구:** MCP 서버가 연결되어 있을 때만 사용 가능합니다. MCP 설정이 없으면 이 도구들은 나타나지 않습니다

## 정리

- Claude Code에는 30개 이상의 도구가 있으며, 파일 조작·명령 실행·웹 검색·작업 관리 등을 수행합니다
- 일부 도구(Write, Edit, Bash, WebFetch 등)는 실행 전 사용자 승인이 필요합니다
- 자주 쓰는 안전한 도구는 `.claude/settings.json`에서 자동 허용 설정하면 편리합니다
