# Claude Code 작동 원리

> Claude Code가 어떻게 생각하고, 판단하고, 행동하는지 이해해보세요.

---

## 핵심 비유: 요리사의 작업 방식

Claude Code의 작동 방식은 레시피를 보고 요리하는 요리사와 비슷합니다.

- **정보를 모읍니다** — 냉장고를 열어 재료를 확인하듯, 파일을 읽고 코드를 파악합니다
- **행동합니다** — 실제로 칼질하고 볶듯, 코드를 수정하고 명령어를 실행합니다
- **결과를 확인합니다** — 음식을 맛보듯, 테스트를 실행하고 결과를 검증합니다
- **이 과정을 반복합니다** — 맛이 안 나면 다시 시도하듯, 문제가 있으면 조정합니다

이것이 바로 **에이전틱 루프(Agentic Loop)**입니다.

---

## 에이전틱 루프란?

여러분이 Claude Code에게 작업을 주면, Claude는 세 단계를 반복합니다.

```
여러분의 요청
      ↓
[1. 맥락 파악] → 파일 읽기, 코드 탐색
      ↓
[2. 행동] → 파일 수정, 명령어 실행
      ↓
[3. 결과 확인] → 테스트 실행, 오류 확인
      ↓
완료될 때까지 반복
(언제든 중단하고 방향을 바꿀 수 있음)
```

예를 들어, "실패하는 테스트를 고쳐줘"라고 하면:

1. 테스트 스위트를 실행해 무엇이 실패하는지 확인
2. 에러 출력을 읽음
3. 관련 소스 파일을 검색
4. 해당 파일들을 읽고 코드 파악
5. 파일을 수정해 문제 해결
6. 테스트를 다시 실행해 확인

## Claude Code의 두 가지 핵심 요소

### 모델 (두뇌)

Claude Code는 Anthropic의 Claude 모델을 사용합니다. 이 모델이 코드를 이해하고, 어떻게 해결할지 판단합니다.

| 모델 | 특징 | 사용 상황 |
|------|------|-----------|
| **Sonnet** | 빠르고 효율적 | 대부분의 코딩 작업 |
| **Opus** | 더 깊은 추론 | 복잡한 아키텍처 결정 |

세션 중에 `/model` 명령으로 모델을 바꿀 수 있습니다.

### 도구 (손발)

도구가 없으면 Claude는 텍스트만 출력할 수 있습니다. 도구가 있어야 실제로 **행동**할 수 있습니다.

| 도구 카테고리 | Claude가 할 수 있는 것 |
|-------------|----------------------|
| **파일 작업** | 파일 읽기, 코드 편집, 새 파일 생성, 이름 변경 및 정리 |
| **검색** | 패턴으로 파일 찾기, 정규식으로 내용 검색, 코드베이스 탐색 |
| **실행** | 셸 명령어 실행, 서버 시작, 테스트 실행, Git 사용 |
| **웹** | 웹 검색, 문서 가져오기, 에러 메시지 조회 |
| **코드 인텔리전스** | 타입 오류 확인, 정의로 이동, 참조 찾기 |

## Claude Code가 접근할 수 있는 것들

`claude` 명령을 폴더에서 실행하면 Claude Code는 다음에 접근합니다.

### 프로젝트 파일
현재 폴더와 하위 폴더의 모든 파일. 파일을 직접 지정할 필요 없이 자동으로 읽습니다.

### 터미널 (명령어 실행)
여러분이 터미널에서 실행할 수 있는 모든 명령어. 빌드 도구, Git, 패키지 매니저, 스크립트 등.

### Git 상태
현재 브랜치, 커밋되지 않은 변경사항, 최근 커밋 히스토리.

### CLAUDE.md (나만의 규칙서)
프로젝트 루트에 만들어두는 마크다운 파일. 코딩 기준, 아키텍처 결정 등을 적어두면 매 세션마다 자동으로 읽습니다. **칠판**처럼 생각하세요 — 이 칠판에 적힌 것은 Claude가 항상 참조합니다.

### 자동 메모리 (Auto Memory)
세션이 끝나도 학습한 것들을 자동으로 저장합니다. 빌드 명령어, 디버깅 패턴 등이 다음 세션에도 유지됩니다.

---

## 세션(Session)이란?

세션은 **하나의 대화**입니다. 대화 내용, 파일 읽기 결과, 명령 출력 등이 모두 기록됩니다.

### 세션의 특징

- **독립적입니다**: 새 세션은 이전 대화 내용 없이 시작합니다 (자동 메모리는 유지됨)
- **저장됩니다**: 로컬에 자동 저장. 나중에 이어서 할 수 있습니다
- **되돌릴 수 있습니다**: 파일 수정 전에 스냅샷을 찍어 언제든 되돌릴 수 있습니다

### 세션 재개 방법

```bash
claude --continue    # 가장 최근 대화 이어서
claude --resume      # 이전 대화 목록에서 선택
```

### 세션 포크(분기)

```bash
claude --continue --fork-session
```

현재 대화를 복사해 새 방향으로 실험할 수 있습니다. 원본 세션은 그대로 유지됩니다.

## 컨텍스트 윈도우 (칠판의 크기)

Claude의 컨텍스트 윈도우는 **칠판**과 같습니다. 이 칠판에는:
- 대화 내용
- 읽은 파일 내용
- 명령어 출력 결과
- CLAUDE.md 내용

...이 모두 쌓입니다. 칠판이 꽉 차면 오래된 내용부터 지워집니다.

### 컨텍스트가 꽉 찰 때

Claude Code가 자동으로 오래된 도구 출력물을 먼저 지우고, 필요하면 대화를 요약합니다. 중요한 규칙은 CLAUDE.md에 저장해두세요 — 대화 기록에 의존하지 마세요.

### 수동으로 컨텍스트 관리

```
/context    # 현재 컨텍스트 사용량 확인
/compact    # 요약해서 공간 확보
/clear      # 완전히 초기화
```

## 안전 장치: 체크포인트와 권한

### 체크포인트 (되돌리기 기능)

Claude가 파일을 수정하기 전에 자동으로 스냅샷을 찍습니다. 잘못되면:
- `Esc` 두 번: 이전 상태로 되돌리기
- "이 변경사항 취소해줘"라고 요청하기

체크포인트는 세션 내 파일 변경에만 적용됩니다. 데이터베이스, API, 외부 서비스에 대한 변경은 되돌릴 수 없으므로 Claude가 실행 전에 먼저 물어봅니다.

### 권한 모드 (Shift+Tab으로 전환)

| 모드 | 설명 |
|------|------|
| **기본 모드** | 파일 수정 및 명령어 실행 전에 승인 요청 |
| **자동 수락 모드** | 파일 수정은 자동, 명령어는 여전히 승인 요청 |
| **계획 모드** | 읽기 전용. 실행 계획만 세우고 실제로는 변경하지 않음 |

## 효과적으로 사용하는 법

### Claude Code에게 물어보기
"hooks를 어떻게 설정해?"나 "CLAUDE.md를 잘 구성하려면?" 같은 질문을 직접 하면 설명해줍니다.

### 대화처럼 접근하기

완벽한 요청을 처음부터 할 필요 없습니다. 시작하고 다듬어 나가세요.

```
로그인 버그 고쳐줘
```
[Claude가 시도함]

```
그게 아니야. 문제는 세션 처리에 있어.
```
[Claude가 방향을 바꿔 다시 시도]

### 검증 수단 제공하기

Claude가 스스로 확인할 수 있게 해주세요.

```
validateEmail 함수를 만들어줘. 테스트 케이스:
- user@example.com → true
- invalid → false
- user@.com → false
구현 후 테스트를 실행해줘.
```

### 구현 전 탐색 먼저

복잡한 문제는 먼저 분석하게 하세요.

```
src/auth/를 읽고 세션 처리 방식을 파악해줘.
그 다음 OAuth 지원을 추가하는 계획을 세워줘.
```

## 예시 케이스

**케이스: 팀의 새 기능 개발**

스타트업 CTO 혜진 씨는 OAuth 로그인 기능을 추가해야 합니다.

1. 먼저 계획 모드(Shift+Tab 두 번)로 진입:
```
src/auth/ 폴더를 읽고 현재 세션 처리 방식을 파악해줘
```

2. Claude가 분석 후:
```
Google OAuth를 추가하려면 어떤 파일을 바꿔야 해? 상세 계획을 세워줘
```

3. 계획을 검토하고 일반 모드로 복귀:
```
계획대로 OAuth 흐름을 구현해줘. 콜백 핸들러 테스트를 작성하고 실패하는 게 있으면 고쳐줘
```

4. 완료 후:
```
설명이 잘 된 메시지로 커밋하고 PR을 열어줘
```
