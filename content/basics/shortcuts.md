## 핵심 단축키 한눈에 보기

| 단축키 | 기능 | 언제 쓰나 |
|--------|------|-----------|
| `Esc` | Claude 중단 (컨텍스트 유지) | 잘못된 방향으로 가고 있을 때 |
| `Esc + Esc` | Rewind 메뉴 열기 | 이전 상태로 돌아가고 싶을 때 |
| `Shift + Tab` | 모드 전환 순환 | 빠르게 모드를 바꿀 때 |
| `Ctrl + G` | 플랜을 에디터로 열기 | Plan Mode에서 플랜 직접 수정 |
| `Ctrl + O` | 상세 모드 토글 | Claude의 사고 과정 보기 |
| `Ctrl + B` | 현재 작업 백그라운드로 | 기다리지 않고 다른 작업 하고 싶을 때 |
| `Alt + T` | Thinking 모드 토글 | 추론 모드 켜기/끄기 |

## Esc - 즉시 중단

Claude가 잘못된 방향으로 가고 있을 때 즉시 멈춥니다. 컨텍스트는 유지됩니다.

```
예시 케이스:
나: "로그인 페이지 리디자인해줘"
Claude: (로그인 페이지 전체를 갈아엎기 시작...)
나: (잠깐, 이건 너무 큰 변경인데...)
-> Esc 누름

나: "전체 리디자인 말고, 버튼 색깔만 바꿔줘"
-> 이전 대화 내용은 그대로 유지된 채로 방향 수정

비유: 택시기사가 잘못된 길로 가고 있을 때
     "잠깐, 멈춰주세요" 후 새 목적지 말하기
     (택시 내리는 게 아님 - 컨텍스트 유지)
```

## Shift + Tab - 모드 전환

세 가지 모드를 순환합니다.

```
[Normal Mode] -> Shift+Tab -> [Auto-Accept Mode] -> Shift+Tab -> [Plan Mode]

각 모드:
1. Normal Mode (기본): 파일 수정 시 허락을 구함
2. Auto-Accept Mode: 모든 파일 수정을 자동 승인
   - 화면 하단에 "⏵⏵ accept edits on" 표시
   - 빠르게 많은 파일을 수정할 때 유용
   - 주의: 무엇을 바꾸는지 잘 확인해야 함

3. Plan Mode: 파일을 읽기만 하고 수정하지 않음
   - 화면 하단에 "⏸ plan mode on" 표시
   - 코드를 분석하고 계획을 세울 때 안전

예시 케이스:
상황 1: 100개 파일에 걸친 변수명 일괄 변경
-> Auto-Accept Mode로 전환 후 실행
-> 일일이 "예" 누르는 시간 절약

상황 2: 새 기능 추가 전 기존 구조 파악
-> Plan Mode로 전환
-> "인증 관련 파일들이 어떻게 연결되는지 분석해줘"
-> 실수로 파일이 수정될 걱정 없이 탐색

비유:
Auto-Accept = 도장을 미리 찍어두고 문서 처리
Plan Mode = 계약서 서명 전 꼼꼼히 읽어보기만 하는 시간
```

## Ctrl + G - 플랜을 에디터에서 직접 편집

Plan Mode에서 Claude가 작성한 플랜을 텍스트 에디터로 열어 직접 수정합니다.

```
사용 흐름:
1. Shift+Tab으로 Plan Mode 진입
2. "OAuth 구현 계획 세워줘" 요청
3. Claude가 플랜 초안 작성 완료
4. Ctrl+G 누름 -> VS Code 등 에디터에서 플랜이 열림
5. 마음에 안 드는 부분 직접 수정
6. 저장하고 Claude로 돌아옴
7. Claude가 수정된 플랜으로 구현 시작

예시 케이스:
Claude 플랜:
  1. OAuth 라이브러리 설치
  2. 구글 OAuth 설정
  3. 페이스북 OAuth 설정   <- 이건 필요 없음
  4. 테스트 작성

-> Ctrl+G로 열어서 "3. 페이스북 OAuth 설정" 삭제
-> 저장
-> Claude가 수정된 플랜대로 구현

비유: 여행 계획서를 여행사가 초안 작성하면
     내가 필요없는 일정은 지우고 OK 사인 보내는 것
```

## Ctrl + O - 사고 과정 보기

Claude가 어떻게 생각하는지 내부 추론 과정을 표시합니다.

```
활성화 시: Claude의 생각이 회색 이탤릭체로 화면에 표시됨
비활성화 시: 최종 답변만 표시 (기본 상태)

예시:
> "버그 원인이 뭔지 찾아줘"

[Ctrl+O 활성화 상태]

  *이 에러 메시지를 보면 null 참조 문제인 것 같다...*
  *user 객체가 undefined일 때 .name을 호출하려 했나...*
  *어디서 user가 undefined가 되는지 추적해야겠다...*

  **분석 결과:** user.name을 호출하기 전에 user가 존재하는지
  확인하는 코드가 없어서 발생한 문제입니다.

언제 유용한가:
- Claude가 왜 그런 결론을 냈는지 이해하고 싶을 때
- ultrathink와 함께 쓰면 고급 분석 과정을 볼 수 있음
- 디버깅 시 Claude의 추론이 맞는 방향인지 확인할 때
```

## Ctrl + B - 백그라운드로 전환

오래 걸리는 작업을 백그라운드로 보내고 다른 작업을 할 수 있습니다.

```
예시 케이스:
나: "전체 테스트 실행하고 실패한 것 모두 고쳐줘"
Claude: (작업 시작, 수분이 걸릴 예정)
-> Ctrl+B 누름
-> Claude가 백그라운드에서 계속 작업
-> 나는 다른 작업 진행

나중에 확인:
-> 작업 완료 알림이 오면 결과 확인

주의사항:
- 백그라운드 실행 중 Claude에게 질문하면 답변을 못 받을 수 있음
- 권한 요청이 필요하면 미리 승인해둬야 중단 없이 진행

비유: 세탁기 돌려놓고 다른 집안일 하기
     세탁이 끝나면 알람이 울림
```

## Esc + Esc (또는 /rewind) - 체크포인트로 되돌리기

> 자세한 내용은 [체크포인트 활용법](/docs/workflow/checkpoint-rewind) 참고

```
빠른 사용법:
1. Esc를 두 번 빠르게 누름
2. Rewind 메뉴가 열림
3. 선택:
   - 대화만 되돌리기
   - 코드만 되돌리기
   - 둘 다 되돌리기
   - 해당 시점부터 요약

예시 케이스:
나: "로그인 기능 리팩토링해줘"
Claude: (리팩토링 완료)
나: (테스트해보니 뭔가 이상하다)
-> Esc + Esc
-> "코드 + 대화 모두 되돌리기" 선택
-> 리팩토링 전 상태로 복구

비유: 게임에서 저장 포인트로 돌아가기
     Claude가 모든 변경 전에 자동으로 저장해둠
```

## 단축키 조합 활용법

### 빠른 코드 탐색 → 플랜 작성 → 구현

```
1. Shift+Tab (Plan Mode 진입)
2. "인증 시스템 전체 구조 파악하고 개선 계획 세워줘"
3. Ctrl+G (플랜 에디터에서 수정)
4. Shift+Tab (Normal Mode로 복귀)
5. "이 플랜대로 구현해줘"
```

### 실험적 변경 → 결과 확인 → 필요시 되돌리기

```
1. "이 함수를 async/await 패턴으로 바꿔줘" (시도)
2. 테스트 실패 확인
3. Esc + Esc (Rewind)
4. "async/await 대신 Promise 체이닝으로 바꿔줘" (다른 방법 시도)
```
