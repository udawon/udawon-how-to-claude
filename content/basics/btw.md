## /btw가 뭔가?

Claude가 코드를 열심히 작성하고 있는 **도중에**, 기다리지 않고 바로 질문할 수 있는 기능입니다.

질문과 답변은 **대화 기록에 남지 않고**, 작업도 **중단되지 않습니다.**

<div class="example-case">

비유: 신입사원이 보고서를 쓰고 있는데, 보스가 어깨 너머로
"<span class="keyword-highlight">아까 그 수치 어디서 가져온 거야?</span>" 하고 물어보는 것.

신입은 고개만 돌려 대답하고, 보고서 작업은 그대로 계속 진행.
</div>

## 사용법

Claude가 작업 중일 때(응답을 생성하는 중일 때) 입력합니다.

```
/btw 지금 뭐 하고 있어?
/btw 아까 바꾼 파일 이름이 뭐였지?
/btw 이거 다 끝나면 내가 뭐 해야 돼?
```

응답은 **팝업처럼 떴다가 사라지는 형태**로 표시됩니다.
메인 대화에는 흔적이 남지 않습니다.

## 기존 방식과 비교

| | 기존 (기다리기) | /btw (바로 질문) |
|---|---|---|
| 작업 중단 | O (끝날 때까지 대기) | X (작업 계속 진행) |
| 대화 기록 | 남음 | 안 남음 |
| 후속 질문 | 가능 | 불가 (1회성) |
| 파일 읽기/수정 | 가능 | 불가 |

핵심 차이: `/btw`는 **기억력은 있지만 손은 못 쓰는 상태**로 답변합니다.
대화 내용은 기억하지만, 파일을 열거나 코드를 실행하는 건 못 합니다.

## 실전 예시 케이스

### 1. 작업 현황 파악

<div class="example-case">

상황: Claude가 홈페이지 코드를 한창 수정하고 있어서 언제 끝날지 모름

→ `/btw 지금 전체 작업 중 어디쯤이야?`
→ "<span class="keyword-highlight">3개 파일 중 2번째 작업 중입니다. 나머지 1개 남았어요.</span>"

비유: 이사 중인 짐꾼에게 "지금 몇 짐째야?" 물어보는 것.
짐꾼은 대답하면서도 짐을 계속 나름.
</div>

### 2. 이전 결정사항 다시 확인

<div class="example-case">

상황: 아까 버튼 색상을 정했는데 기억이 안 남

→ `/btw 아까 버튼 색상 뭘로 정했었지?`
→ "<span class="keyword-highlight">메인 버튼은 #2563EB (파란색), 보조 버튼은 #6B7280 (회색)으로 정했습니다.</span>"

비유: 회의 중에 "아까 그거 뭐라고 했었지?" 하고 옆 사람한테 속삭이는 것.
회의는 계속 진행됨.
</div>

### 3. 팀원에게 공유할 설명 요청

<div class="example-case">

상황: Claude가 코드를 리팩토링하는 중인데, 팀장에게 보고해야 함

→ `/btw 지금 하고 있는 변경사항을 비개발자한테 설명하려면 뭐라고 하면 돼?`
→ "<span class="keyword-highlight">사이트 로딩 속도를 개선하기 위해 내부 구조를 정리하고 있다고 하시면 됩니다.</span>"

비유: 정비소에서 "차주한테 뭐라고 설명하면 되냐?"고
정비사에게 물어보는 것. 정비사는 대답하면서 수리 계속.
</div>

### 4. 다음 단계 미리 확인

<div class="example-case">

상황: Claude가 백엔드 연동 중인데, 작업 끝나면 내가 할 일이 있는지 궁금

→ `/btw 이거 끝나면 내가 직접 해야 할 거 있어?`
→ "<span class="keyword-highlight">Vercel에서 환경변수 3개를 등록하셔야 합니다. 작업 끝나면 가이드 드릴게요.</span>"

비유: 인테리어 업자가 벽지를 바르는 중에
"끝나면 제가 뭐 준비해놓으면 돼요?" 하고 미리 물어보는 것.
</div>

### 5. 롤아웃 체크리스트 요청

<div class="example-case">

상황: 배포 작업이 진행 중인데, 배포 전 확인 사항을 미리 정리하고 싶음

→ `/btw 배포 전에 확인해야 할 체크리스트 정리해줘`
→ "<span class="keyword-highlight">1. 환경변수 확인 2. 빌드 에러 없는지 3. DB 마이그레이션 완료 여부 ...</span>"

비유: 출장 짐 싸는 중에 "출발 전 체크리스트 뭐였지?" 하고
메모장을 힐끗 보는 것. 짐은 계속 쌈.
</div>

## 언제 /btw를 쓰고, 언제 기다려야 하나?

| 상황 | 추천 방법 |
|------|-----------|
| "지금 뭐 하고 있어?" 같은 간단한 확인 | `/btw` |
| "아까 합의한 내용 뭐였지?" 같은 맥락 질문 | `/btw` |
| "이 파일 내용 좀 보여줘" | 기다렸다가 메인 대화에서 요청 |
| "방향을 바꿔서 이렇게 해줘" | <kbd>Esc</kbd>로 중단 후 새로 지시 |
| 답변 내용을 이어서 더 파고 싶을 때 | 기다렸다가 메인 대화에서 후속 질문 |

## 기술적 특징

- **대화 기록에 안 남음**: 메인 컨텍스트를 오염시키지 않아서, 길고 복잡한 작업 흐름이 깨지지 않습니다.
- **도구 사용 불가**: 파일 읽기, 코드 실행, 웹 검색 등은 못 합니다. 오직 이미 대화에 있는 정보로만 답변합니다.
- **1회성**: 후속 질문 불가. 한 번 묻고 한 번 답하면 끝. 더 파고 싶으면 메인 대화에서 하세요.
- **비용 효율적**: 부모 대화의 프롬프트 캐시를 재사용하므로 추가 비용이 적습니다.
- **서브에이전트와 정반대**: 서브에이전트는 도구는 있지만 대화 맥락이 없고, `/btw`는 대화 맥락은 있지만 도구가 없습니다.
