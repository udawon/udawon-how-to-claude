## 한 줄 요약

**서브 에이전트 = 특정 심부름을 맡길 수 있는 전문 인턴**

Claude Code에게 "이거 해줘"라고 말하면, Claude가 혼자 다 하는 대신
전문 인턴(서브 에이전트)에게 일부를 맡기고, 결과만 받아서 나에게 알려줍니다.

---

## 서브 에이전트가 뭔가요?

### 비유로 이해하기

```
[나] → "매출 보고서 만들어줘"
  ↓
[Claude = 팀장]
  ├→ [인턴 A] "데이터 찾아와"    → 데이터 전달
  ├→ [인턴 B] "차트 만들어"      → 차트 전달
  └→ [인턴 C] "경쟁사 조사해"    → 조사 결과 전달
  ↓
[Claude = 팀장] 종합해서 보고서 완성 → [나]에게 전달
```

핵심 특징:
- **인턴끼리는 서로 모릅니다** — 각자 맡은 일만 하고 팀장(Claude)에게 보고
- **결과만 돌아옵니다** — 인턴이 100페이지 조사해도, 나에게는 요약만 전달
- **여러 명이 동시에 일합니다** — 순서대로 하면 30분 걸릴 일을 10분에 끝냄

### 실생활 예시

| 상황 | 서브 에이전트 없이 | 서브 에이전트 사용 |
|------|------------------|------------------|
| 파일 100개에서 검색 | Claude가 하나씩 찾음 (오래 걸림) | 검색 전문 인턴이 빠르게 처리 |
| 3개 경쟁사 분석 | 하나씩 순서대로 분석 | 인턴 3명이 동시에 분석 |
| 테스트 결과 확인 | 수백 줄 결과를 내가 읽음 | 인턴이 읽고 "3건 문제" 요약만 전달 |

## 기본 내장 서브 에이전트

Claude Code에는 이미 만들어진 인턴 3명이 있습니다. 별도 설정 없이 자동으로 사용됩니다.

### 1. Explore (탐색 전문)

```
역할: 프로젝트 파일을 빠르게 찾고 분석
모델: Haiku (빠르고 저렴한 모델)
권한: 읽기만 가능 (수정 불가)

비유: 서류함에서 자료 찾아오는 인턴
      "이 서류 어디 있었지?" → 찾아서 가져다줌
```

**예시 대화:**
```
나: "이 프로젝트에서 결제 관련 파일 전부 찾아줘"

→ Claude가 Explore 인턴에게 시킴
→ 인턴이 프로젝트 전체를 빠르게 뒤져서 목록 작성
→ Claude가 "결제 관련 파일 7개 찾았습니다"라고 정리해서 알려줌
```

### 2. Plan (조사/계획 전문)

```
역할: 구현 전 조사하고 계획 세우기
모델: 메인 대화와 같은 모델
권한: 읽기만 가능 (수정 불가)

비유: 시장조사 하는 인턴
      "이거 어떻게 만들면 좋을지 조사해와" → 보고서 작성
```

**예시 대화:**
```
나: "로그인 기능 어떻게 구현하면 좋을지 계획 세워줘"

→ Claude가 Plan 인턴에게 시킴
→ 인턴이 코드 구조를 조사하고 실현 가능한 방법 분석
→ Claude가 "3가지 방법이 있는데, 이유는..." 정리해서 알려줌
```

### 3. General (만능)

```
역할: 복잡한 작업 처리 (읽기 + 수정 모두 가능)
모델: 메인 대화와 같은 모델
권한: 모든 도구 사용 가능

비유: 경력 있는 인턴
      "이 문제 알아서 분석하고 고쳐와" → 직접 수정까지 해옴
```

**예시 대화:**
```
나: "이 에러 원인 찾아서 고쳐줘"

→ Claude가 General 인턴에게 시킴
→ 인턴이 에러 분석, 원인 파악, 코드 수정까지 처리
→ Claude가 "이런 원인이었고 이렇게 고쳤습니다"라고 알려줌
```

## 나만의 서브 에이전트 만들기

기본 인턴 3명 외에, **내가 원하는 전문 인턴을 직접 만들 수 있습니다.**

### 방법 1: 대화로 만들기 (가장 쉬움)

```
따라하기:
  1. Claude Code에서 /agents 입력
  2. "Create new agent" 선택
  3. 범위 선택:
     - User-level: 모든 프로젝트에서 사용
     - Project-level: 이 프로젝트에서만 사용
  4. "Generate with Claude" 선택
  5. 원하는 설명 입력:
     "코드를 검토하고 문제점을 알려주는 검토 전문가"
  6. 도구/모델 선택 후 저장
```

**비유:**
```
신입 인턴을 채용하는 과정:

1. "인사팀(=/agents)" 방문
2. "새 인턴 뽑을게요"
3. "전 부서에 배치할까요? 이 팀에만?" → 범위 선택
4. "어떤 업무를 시킬 건가요?" → 업무 설명
5. "이 인턴이 사용할 수 있는 도구는?" → 권한 설정
6. 채용 완료! 바로 일 시작 가능
```

### 방법 2: 파일로 직접 만들기

서브 에이전트는 마크다운(.md) 파일 하나로 정의됩니다.

**파일 위치:**
| 위치 | 적용 범위 | 비유 |
|------|----------|------|
| `.claude/agents/` | 현재 프로젝트만 | 이 팀 전용 인턴 |
| `~/.claude/agents/` | 모든 프로젝트 | 회사 전체에서 쓰는 인턴 |

**예시: 코드 리뷰어 만들기**

파일명: `.claude/agents/code-reviewer.md`
```markdown
---
name: code-reviewer
description: 코드를 검토하고 품질, 보안, 모범 사례를 확인합니다
tools: Read, Glob, Grep
model: sonnet
---

당신은 시니어 코드 리뷰어입니다.
호출되면 코드를 분석하고 품질, 보안, 모범 사례에 대한
구체적이고 실행 가능한 피드백을 제공하세요.
```

**비유:**
```
인턴 업무 매뉴얼을 작성하는 것:

--- (위쪽: 인사 카드)
name: 이름
description: 담당 업무
tools: 사용할 수 있는 도구
model: 역량 수준
---

(아래쪽: 업무 지침서)
"당신은 ~입니다. 이런 상황에서 이렇게 하세요."
```

## 설정 항목 상세

서브 에이전트를 만들 때 설정할 수 있는 항목들입니다.

### 필수 항목

| 항목 | 설명 | 비유 |
|------|------|------|
| `name` | 에이전트 이름 (영문, 하이픈) | 인턴 이름표 |
| `description` | 언제 이 에이전트를 쓸지 설명 | 인턴 직무기술서 |

### 선택 항목

| 항목 | 설명 | 비유 |
|------|------|------|
| `tools` | 사용 가능한 도구 목록 | 인턴에게 허용된 장비 |
| `disallowedTools` | 사용 금지할 도구 | 인턴에게 금지된 장비 |
| `model` | 사용할 AI 모델 (`sonnet`, `opus`, `haiku`) | 인턴 역량 등급 |
| `permissionMode` | 권한 수준 | 인턴 보안 등급 |
| `maxTurns` | 최대 작업 횟수 | 인턴 근무시간 제한 |
| `skills` | 미리 주입할 스킬 | 인턴 사전 교육 자료 |
| `memory` | 기억 저장 (`user`, `project`, `local`) | 인턴 업무일지 |
| `background` | 백그라운드 실행 여부 | 인턴이 조용히 일하기 |
| `isolation` | `worktree` 설정 시 독립 작업 공간 | 인턴 전용 책상 |
| `hooks` | 작업 전후 자동 실행 스크립트 | 인턴 업무 체크리스트 |

### 모델 선택 가이드

```
haiku  = 파트타임 인턴   → 빠르고 저렴. 단순 검색/조사에 적합
sonnet = 정규직 인턴     → 균형 잡힌 성능. 대부분의 작업에 적합
opus   = 경력직 인턴     → 가장 뛰어남. 복잡한 분석에 적합
inherit = 팀장과 같은 등급 → 메인 대화의 모델을 따라감 (기본값)
```

### 권한 모드

```
default           = 매번 허락받고 일하기 (기본값)
acceptEdits       = 파일 수정은 자동 허용
dontAsk           = 허용된 것만 하고, 나머지는 자동 거부
bypassPermissions = 모든 것 자동 허용 (주의!)
plan              = 읽기만 가능 (계획 모드)
```

**비유:**
```
인턴 권한 등급:

default  = "이거 해도 될까요?" 매번 물어보는 신입
acceptEdits = "서류 수정은 알아서 할게요" 중급 인턴
dontAsk  = "허락된 것만 하고 나머진 패스" 규칙 준수형
bypass   = "다 알아서 하겠습니다" 자율형 (위험할 수 있음)
plan     = "조사만 하고 보고서 쓸게요" 리서치 인턴
```

## 기억 기능 (Memory)

서브 에이전트가 **세션을 넘어서 학습한 내용을 기억**하게 할 수 있습니다.

```yaml
---
name: code-reviewer
description: 코드 검토 전문가
memory: user
---
```

### 기억 범위

| 범위 | 저장 위치 | 비유 |
|------|----------|------|
| `user` | `~/.claude/agent-memory/이름/` | 모든 프로젝트에서 쓰는 메모장 |
| `project` | `.claude/agent-memory/이름/` | 이 프로젝트 전용 메모장 |
| `local` | `.claude/agent-memory-local/이름/` | 개인용 메모장 (공유 안 함) |

**비유:**
```
인턴이 업무일지를 쓰는 것:

user    = 회사 전체에서 열람 가능한 노트
project = 이 팀 내에서만 보는 노트
local   = 인턴 개인 메모 (남에게 안 보여줌)

"지난번에 이런 패턴 발견했으니 다음에도 적용하자"
→ 같은 실수를 반복하지 않고 점점 나아짐
```

## 포그라운드 vs 백그라운드

서브 에이전트는 두 가지 방식으로 실행됩니다.

### 포그라운드 (기본)

```
나: "이거 조사해줘"
→ 서브 에이전트 시작
→ (기다리는 중...)      ← 이 동안 나는 다른 작업 불가
→ 결과 도착
→ 대화 계속

비유: 인턴한테 심부름 시키고 기다리는 것
     "갖다 올 때까지 여기서 기다릴게"
```

### 백그라운드

```
나: "이거 조사해줘 (백그라운드로)"
→ 서브 에이전트 시작
→ 나는 다른 작업 계속...  ← 기다리지 않아도 됨
→ 서브 에이전트 완료 알림
→ 결과 확인

비유: 인턴한테 심부름 시키고 내 일 계속하는 것
     "끝나면 알려줘, 나는 내 일 할게"
```

**백그라운드 전환:** 진행 중인 작업을 `Ctrl+B`로 백그라운드로 보낼 수 있습니다.

## 실전 활용 패턴

### 패턴 1: 대량 결과 격리

```
나: "테스트 전부 실행하고 실패한 것만 알려줘"

→ 서브 에이전트가 수백 줄의 테스트 결과를 처리
→ 나에게는 "3건 실패: login, payment, signup" 요약만 전달

왜 좋은가?
  서브 에이전트 없이 하면 → 수백 줄이 내 대화창에 쌓여서 맥락을 잃음
  서브 에이전트 사용하면 → 깔끔한 요약만 받아서 바로 다음 작업 가능
```

### 패턴 2: 동시 조사

```
나: "인증, 데이터베이스, API 모듈을 동시에 조사해줘"

→ 서브 에이전트 3개가 동시에 출발
   [인턴 A] → 인증 모듈 조사 완료
   [인턴 B] → 데이터베이스 모듈 조사 완료
   [인턴 C] → API 모듈 조사 완료
→ Claude가 3개 결과를 종합해서 보고

왜 좋은가?
  순서대로 하면 → 15분 x 3 = 45분
  동시에 하면 → 약 15분 (3배 빠름)
```

### 패턴 3: 단계별 연결

```
나: "코드 검토 먼저 하고, 문제 있으면 최적화해줘"

→ 1단계: code-reviewer 에이전트가 문제점 발견
→ 2단계: optimizer 에이전트가 발견된 문제를 수정
→ 최종 결과를 나에게 보고

비유: 인턴 A가 불량품 찾으면 → 인턴 B가 수리하는 것
```

## 서브 에이전트 vs 메인 대화, 언제 뭘 쓸까?

| 상황 | 추천 | 이유 |
|------|------|------|
| 빠른 수정 | 메인 대화 | 인턴 부를 필요 없음 |
| 왔다갔다 대화 필요 | 메인 대화 | 인턴은 결과만 가져옴 |
| 결과가 길고 요약만 필요 | 서브 에이전트 | 대화창이 깔끔해짐 |
| 특정 도구만 쓰게 제한 | 서브 에이전트 | 권한 제한 가능 |
| 독립적인 작업 | 서브 에이전트 | 맡기고 기다리기 |

## 관리하기

### 목록 보기

```
Claude Code에서: /agents
터미널에서: claude agents
```

### 이어서 작업하기 (Resume)

서브 에이전트가 작업을 끝낸 뒤, 이어서 추가 작업을 시킬 수 있습니다.

```
나: "아까 그 코드 리뷰 에이전트에게 인증 로직도 봐달라고 해줘"
→ Claude가 이전 에이전트를 이어서 실행 (맥락 유지)
```

**비유:** 인턴이 보고서를 제출한 뒤 "아, 이 부분도 추가로 조사해줘"라고 시키는 것. 인턴이 처음부터 다시 시작하지 않고, 이전 내용을 기억한 채로 이어서 작업합니다.

### 비활성화하기

특정 서브 에이전트를 끄고 싶다면 settings.json에 추가:

```json
{
  "permissions": {
    "deny": ["Agent(내-에이전트-이름)"]
  }
}
```

**비유:** "이 인턴은 당분간 쉬게 해"

## 주의사항

| 제한 | 설명 | 비유 |
|------|------|------|
| 인턴이 인턴을 못 부름 | 서브 에이전트가 또 다른 서브 에이전트를 만들 수 없음 | 인턴이 또 인턴을 고용할 순 없음 |
| 결과가 돌아오면 맥락 소비 | 여러 서브 에이전트의 결과가 많으면 대화창이 커짐 | 인턴 10명이 각각 보고서 올리면 책상이 가득 참 |
| 팀장 대화 기록 안 줌 | 서브 에이전트는 이전 대화 내용을 모름 | 인턴은 회의 내용을 모르고, 시킨 것만 앎 |
