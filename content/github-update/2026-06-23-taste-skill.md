## 이게 뭔가요?

taste-skill(테이스트 스킬)은 **AI에게 코드를 시켜 만든 웹사이트·앱 화면의 '디자인 완성도'를 끌어올려 주는 스킬(skill, AI에게 미리 가르쳐 두는 작업 설명서) 모음**입니다. 만든 사람은 Leonxlnx(레온)이고, 오픈소스(누구나 무료로 쓰고 고칠 수 있는 공개 소프트웨어)로 공개되어 있습니다.

요즘은 ChatGPT(챗지피티)나 Claude(클로드) 같은 AI에게 "쇼핑몰 첫 화면 만들어줘"라고 시키면 코드를 뚝딱 만들어 줍니다. 그런데 그렇게 나온 화면은 하나같이 어디서 본 듯 비슷하고 밋밋합니다. 글자는 다닥다닥 붙어 있고, 여백은 어색하고, 움직임도 없어서 "AI가 대충 찍어냈구나" 하는 티가 납니다. 영어권에서는 이런 결과물을 **slop(슬롭, 성의 없이 양산된 결과물)**이라고 부르는데, taste-skill은 바로 이 'slop'을 없애겠다고 내세우는 도구입니다(스스로를 "anti-slop", 즉 '반(反)-슬롭' 도구라고 소개합니다).

마치 신입 디자이너가 막 그린 시안을 **베테랑 아트 디렉터가 옆에서 "여백을 더 줘라, 글꼴을 바꿔라, 여기에 부드러운 움직임을 넣어라"라고 다듬어 주는 것**처럼 — taste-skill은 AI가 화면 코드를 만들 때 따라야 할 '디자인 감각 규칙'을 미리 손에 쥐여 줍니다. 그래서 같은 AI를 쓰더라도 훨씬 세련된 화면이 나오게 됩니다.

## 왜 필요한가요?

AI로 화면을 만드는 사람들에게는 세 가지 공통된 불만이 있습니다.

**문제 1: 결과물이 다 비슷비슷하다**
AI가 만든 화면은 기본 틀(boilerplate, 보일러플레이트 — 어디나 똑같이 들어가는 뻔한 기본 구성)을 벗어나지 못해, 누가 만들어도 형제처럼 닮습니다. taste-skill은 레이아웃(화면 배치), 타이포그래피(글꼴·글자 배치), 모션(움직임·애니메이션), 여백을 의도적으로 다르게 잡아 줘서 '나만의 화면'에 가깝게 만들어 줍니다.

**문제 2: 비전문가는 '뭐가 어색한지'를 콕 집지 못한다**
디자인을 배우지 않은 사람은 화면이 어딘가 촌스러워도 정확히 무엇을 고쳐야 할지 모릅니다. taste-skill은 그 '디자인 감각'을 규칙으로 정리해 AI에게 대신 적용해 주므로, 사용자가 디자인 용어를 몰라도 결과가 좋아집니다.

**문제 3: 쓰는 AI 도구마다 따로 설정해야 한다**
어떤 사람은 Cursor(커서)를, 어떤 사람은 Claude Code(클로드 코드)를, 어떤 사람은 그냥 ChatGPT를 씁니다. taste-skill은 이 도구들 어디에든 똑같이 끼워 쓸 수 있는 '들고 다니는(portable, 포터블) 스킬' 형태라, 한 번 익히면 도구를 바꿔도 그대로 통합니다. React(리액트)·Vue(뷰)·Svelte(스벨트) 같은 어떤 웹 제작 프레임워크(framework, 화면을 만드는 기본 골격 도구)와도 어울립니다.

## 핵심 기능

taste-skill의 가장 큰 특징은 **목적에 따라 골라 쓰는 여러 개의 전문 스킬**과, **취향을 숫자로 조절하는 다이얼(dial)**입니다.

### 1. 코드를 만들어 주는 '구현 스킬' (총 10종)

상황과 원하는 분위기에 맞춰 골라 쓰는 스킬들입니다.

- **design-taste-frontend**: 대표 스킬(v2, 실험판). 요청을 분석해 디자인 시스템(색·글꼴·간격을 일관되게 묶은 규칙 세트)으로 정리하고, GSAP(지샙 — 웹에서 부드러운 움직임을 만드는 대표 애니메이션 라이브러리) 기반 움직임 코드의 뼈대까지 잡아 줍니다.
- **design-taste-frontend-v1**: 이전 버전. 안정적으로 쓰고 싶은 사람을 위해 그대로 남겨 두었습니다.
- **gpt-taste**: ChatGPT·Codex(코덱스, OpenAI의 코딩 AI)에 더 맞춘 변형. 배치 변화를 더 과감하게, 움직임 지시를 더 강하게 줍니다.
- **image-to-code**: 먼저 참고용 화면 '이미지'를 만들고, 그 이미지를 분석한 뒤 실제 코드로 옮기는 '이미지 먼저' 방식입니다.
- **redesign-existing-projects**: 이미 있는 화면을 진단해 배치·여백·정보 위계(중요한 것을 더 크고 눈에 띄게 두는 순서)·스타일을 손봐 줍니다.
- **high-end-visual-design**: 부드러운 대비, 넉넉한 여백, 고급 글꼴, 스프링(탄성) 움직임으로 '비싸 보이는' 차분한 화면을 만듭니다.
- **full-output-enforcement**: AI가 "여기는 알아서 채우세요" 식으로 빈칸(placeholder)을 남기지 않고, 코드를 끝까지 완성해 내도록 강제합니다.
- **minimalist-ui**: Notion(노션)·Linear(리니어) 같은 깔끔하고 절제된 '에디토리얼(잡지 같은)' 화면을 만듭니다.
- **industrial-brutalist-ui**: 딱딱하고 기계적인 느낌, 강한 대비, 실험적 배치의 '브루탈리즘(투박·강렬)' 스타일을 만듭니다.
- **stitch-design-taste**: 구글 Stitch(스티치, 구글의 화면 디자인 도구)와 호환되며, 디자인 설명서(DESIGN.md) 파일로 내보낼 수도 있습니다.

### 2. 참고용 이미지를 만들어 주는 '이미지 생성 스킬' (총 3종)

코드가 아니라 디자인 시안 이미지를 뽑아 주는 스킬들입니다.

- **imagegen-frontend-web**: 글꼴·여백이 살아 있는 웹사이트 시안 이미지
- **imagegen-frontend-mobile**: iOS·Android(안드로이드) 앱 화면·흐름 시안 이미지
- **brandkit**: 로고 방향·색 조합·글꼴 등 브랜드 정체성 시안

### 3. 취향을 숫자로 조절하는 3개의 다이얼

대표 스킬에는 1부터 10까지 숫자로 분위기를 조절하는 손잡이가 있습니다. 마치 **오디오의 볼륨·저음·고음 노브를 돌려 내 취향에 맞추듯** 화면의 성격을 미세 조정할 수 있습니다.

| 다이얼 | 낮을 때 | 높을 때 |
|--------|---------|---------|
| DESIGN_VARIANCE (배치 실험성) | 가운데 정렬·깔끔·정석 | 비대칭·모던·파격 |
| MOTION_INTENSITY (움직임 강도) | 마우스 올렸을 때 살짝 반응 | 스크롤·자석 효과 등 풍부한 애니메이션 |
| VISUAL_DENSITY (정보 밀도) | 넉넉하고 여유로운 화면 | 정보가 빽빽한 대시보드형 화면 |

## 설치 방법

특정 프로그램을 따로 깔 필요 없이, 한 줄 명령으로 스킬을 가져옵니다. 아래 명령은 CLI(키보드로 명령어를 입력해 컴퓨터를 조작하는 검은 화면)에 입력합니다.

```bash
# 저장소의 모든 스킬을 한 번에 설치
npx skills add https://github.com/Leonxlnx/taste-skill
```

원하는 스킬 하나만 골라 설치할 수도 있습니다.

```bash
# 예: 대표 스킬 하나만 설치
npx skills add https://github.com/Leonxlnx/taste-skill --skill "design-taste-frontend"
```

> npx(엔피엑스)는 따로 설치하지 않고 그 자리에서 도구를 실행해 주는 명령입니다. Node.js(노드 — 자바스크립트를 컴퓨터에서 돌리는 환경)가 깔려 있으면 바로 쓸 수 있습니다.

명령어 사용이 부담스럽다면, 각 스킬의 설명서(SKILL.md) 파일을 직접 프로젝트에 복사해 넣거나, 그 내용을 ChatGPT·Codex 대화창에 붙여 넣는 방법도 있습니다. taste-skill은 Codex, Cursor, Claude Code, ChatGPT에서 모두 쓸 수 있습니다.

## 이런 분께 유용합니다

- AI로 웹·앱 화면을 만들지만 결과가 늘 밋밋해 고민인 분
- 디자인을 배운 적은 없지만 '있어 보이는' 화면을 만들고 싶은 1인 창업가·소상공인
- 외주 비용 없이 제품 소개 페이지·포트폴리오·관리자 화면을 직접 만들려는 분
- 이미 만들어 둔 낡은 화면을 빠르게 현대적으로 손보고 싶은 개발자
- Cursor·Claude Code·ChatGPT 등 도구를 오가며 일관된 디자인 품질을 유지하고 싶은 팀

## 주의할 점

- **현재 대표 버전(v2)은 실험 단계**입니다. 만든 사람도 "안정적으로 써야 하는 실제 서비스에는 v1로 고정(pin)해 쓰라"고 안내합니다. 새 기능을 빨리 써 보고 싶다면 v2, 안정성이 중요하면 v1을 선택하세요.
- **결과를 100% 보장하지는 않습니다.** AI에게 더 좋은 지침을 주는 도구일 뿐, 최종 화면 품질은 어떤 AI를 쓰는지와 요청 내용에 따라 달라집니다. 나온 결과는 사람이 한 번 검토하는 것이 좋습니다.
- **이름이 같은 가상화폐(코인)와는 무관합니다.** 만든 사람이 README에서 "이 프로젝트는 공식 암호화폐·토큰이 없으며, 이름을 사칭한 코인은 우리와 관련 없다"고 분명히 밝혔습니다. 사기 주의.
- **디자인 '취향'은 주관적입니다.** 브루탈리즘·미니멀 등 스타일 선택은 결국 보는 사람의 호불호가 갈리므로, 다이얼을 조절하며 여러 번 시도해 보는 것이 좋습니다.
