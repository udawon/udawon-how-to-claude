## 이게 뭔가요?

마치 공장의 표준 작업 지침서처럼, AI 에이전트가 코드 작업을 할 때 반드시 따라야 하는 절차를 문서로 정의해두는 도구입니다. AI에게 "버그를 고쳐줘"라고 말하면 매번 다른 방식으로 접근하는 문제를 해결합니다. Archon은 버그 수정, 기능 개발, PR(Pull Request, 코드 검토 요청) 리뷰 같은 작업마다 YAML(야믈, 사람이 읽기 쉬운 설정 파일 형식)로 작성된 절차서를 AI에게 쥐여주어 결과가 항상 예측 가능하게 만드는 "AI 개발 인프라 코드(Infrastructure-as-Code for AI)" 엔진입니다.

## 왜 주목받고 있나요?

"AI가 코드를 짜는데 결과가 매번 달라서 실무에 못 쓰겠다"는 개발팀들의 공통 불만을 정면으로 겨냥했습니다. GitHub Actions나 Dockerfile처럼 인프라 설정을 코드로 관리하는 방식을 AI 에이전트의 행동 규칙에도 그대로 적용한 아이디어가 엔지니어링 커뮤니티에 설득력 있게 받아들여졌습니다. AI가 하는 일을 버전 관리(Git)로 추적하고, 팀원 모두가 같은 방식으로 AI를 사용할 수 있다는 점이 실무 팀들의 관심을 끌었습니다.

---

## 핵심 기능

### 1. YAML 기반 워크플로우 정의
이슈 수정, 기능 개발, PR 리뷰 등 17개의 기본 워크플로우가 기본으로 제공됩니다. 각 워크플로우는 AI가 수행해야 할 단계를 순서대로 명세한 YAML 파일이며, 팀에 맞게 직접 수정하거나 새로 만들 수 있습니다. 절차서 자체가 코드이므로 Git으로 버전 관리됩니다.

### 2. Git Worktree 격리 실행
각 작업을 독립된 브랜치(branch, 코드의 독립 작업 공간)에서 병렬로 실행합니다. 한 AI가 기능 A를 개발하는 동안 다른 AI가 버그 B를 수정해도 서로 충돌하지 않습니다. 마치 여러 요리사가 각자 다른 조리대를 쓰는 것과 같습니다.

### 3. 멀티플랫폼 인터페이스
웹 UI, CLI(명령줄 인터페이스), Slack, Telegram, Discord, GitHub 이슈 등 다양한 채널에서 AI 에이전트에게 작업을 요청할 수 있습니다. Slack에서 "#bug 로그인 오류 수정"이라고 입력하면 Archon이 워크플로우를 실행합니다.

### 4. 결정론적(Deterministic) + AI 노드 혼합 처리
모든 단계를 AI에게 맡기지 않습니다. 테스트 실행, 파일 저장, 브랜치 생성처럼 결과가 명확해야 하는 단계는 확정적 로직으로, 코드 작성이나 리뷰처럼 판단이 필요한 단계만 AI로 처리합니다. 이 혼합 방식이 전체 결과의 예측 가능성을 높입니다.

### 5. Claude + Codex 듀얼 지원
Anthropic의 Claude와 OpenAI의 Codex(코덱스)를 함께 활용할 수 있습니다. 작업 유형에 따라 더 적합한 AI 모델을 워크플로우별로 지정할 수 있어 비용과 성능을 함께 최적화할 수 있습니다.

<div class="example-case">
<strong>예시 1: GitHub 이슈 자동 처리</strong>

개발팀에서 GitHub에 "결제 페이지 로딩 느림"이라는 이슈를 등록하면, Archon이 자동으로 버그 수정 워크플로우를 시작합니다. AI가 관련 코드를 분석하고 수정안을 작성한 뒤 PR을 올립니다. 팀원은 PR만 검토하면 됩니다.

## 이런 분께 유용합니다

- AI 코딩 도구를 팀에 도입했지만 결과가 매번 달라 신뢰하기 어려운 엔지니어링 팀
- GitHub Actions처럼 AI 에이전트 동작도 코드로 관리하고 싶은 DevOps(개발-운영 통합) 담당자
- 반복적인 코드 리뷰나 버그 수정 작업을 AI에게 위임하고 싶은 소규모 개발팀 리더

---

## 주의할 점

워크플로우를 YAML로 직접 작성하거나 수정하려면 기술적 이해가 필요합니다. 비개발자가 혼자 설정하기는 어렵습니다. 또한 AI 에이전트가 코드 저장소에 직접 접근하는 구조이므로 권한 설정을 꼼꼼히 해야 합니다. 프로덕션(실제 서비스) 환경에 바로 적용하기보다 테스트 환경에서 충분히 검증한 후 도입하는 것을 권장합니다.
