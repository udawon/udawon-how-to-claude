## 이게 뭔가요?

**Computer Use(컴퓨터 사용)**는 Claude가 여러분의 컴퓨터 화면을 보면서 **마우스 클릭, 키보드 입력, 스크롤** 등을 직접 해주는 기능입니다.

쉽게 말하면, **원격 비서가 내 컴퓨터 앞에 앉아서 대신 일해주는 것**과 같습니다. 파일을 열고, 브라우저를 조작하고, 개발 도구를 실행하는 것까지 — 별도 설치 없이 Claude가 알아서 해줍니다.

**Dispatch(디스패치)**는 여기에 **스마트폰 원격 지시 기능**을 더한 것입니다. 출퇴근길 지하철에서 폰으로 "이것 좀 해줘"라고 Claude에게 보내면, 집이나 사무실에 켜져 있는 컴퓨터에서 Claude가 작업을 수행합니다. 나중에 컴퓨터 앞에 앉으면 완료된 결과물을 바로 확인할 수 있습니다.

이 두 기능은 **Claude Cowork**(데스크톱 앱)과 **Claude Code**(개발자용 CLI 도구)에서 모두 사용할 수 있습니다.

## 왜 알아야 하나요?

**알면 좋은 점:**
- **내가 자리를 비워도 Claude가 일함** — 출퇴근 중, 회의 중에도 Claude가 컴퓨터에서 작업을 계속함
- **도구 연동이 안 돼도 괜찮음** — Slack, Google Calendar 같은 직접 연동이 없는 앱도 화면을 보고 직접 조작 가능
- **반복 작업 자동화** — "매일 아침 이메일 확인해서 요약해줘", "매주 지표 뽑아줘" 같은 정기 작업 설정 가능
- **폰 하나로 원격 지시** — PC가 없어도 스마트폰에서 바로 Claude에게 업무 할당 가능

**모르면 불편한 점:**
- API(프로그램끼리 대화하는 통로) 연동이 안 되는 앱은 수동으로 직접 조작해야 함
- 컴퓨터 앞에 앉아 있어야만 Claude에게 일을 시킬 수 있음
- 반복 작업을 매번 처음부터 다시 요청해야 함

## 어떻게 하나요?

### 방법 1: Computer Use — Claude가 내 컴퓨터를 조작하게 하기

Claude는 작업을 수행할 때 **가장 정확한 방법을 먼저 시도**합니다:
1. Slack, Google Calendar 같은 **직접 연동(Connector)**이 있으면 그걸 먼저 사용
2. 직접 연동이 없으면 **화면을 보고 마우스·키보드로 직접 조작**

**사용 조건:**
- Claude Pro 또는 Claude Max 구독 필요
- 현재 **macOS만 지원** (Windows는 아직 미지원)
- 데스크톱 앱이 **켜져 있고 깨어 있는 상태**여야 함

**작동 방식:**
- Claude가 새로운 앱에 접근하려 할 때 **반드시 사용자 허락을 먼저 구함**
- 언제든지 Claude를 **중간에 멈출 수 있음**
- 일부 앱은 **보안상 기본적으로 접근 차단**되어 있음

<div class="example-case">
<strong>예시: IDE에서 코드 수정 → 테스트 → PR 올리기</strong>

1. Claude에게 "이 버그 수정해줘"라고 요청
2. Claude가 IDE(코드 편집기)를 열고 해당 파일을 찾음
3. 코드를 수정하고, 터미널에서 테스트를 실행
4. 테스트 통과 확인 후, Git으로 PR(Pull Request, 코드 검토 요청)을 생성

→ 사용자는 중간 과정을 지켜볼 수도, 다른 일을 하다 결과만 확인할 수도 있음
</div>

### 방법 2: Dispatch — 스마트폰에서 Claude에게 원격 지시하기

Dispatch는 **폰과 데스크톱을 하나의 대화로 연결**해줍니다.

**사용 방법:**
1. 데스크톱 앱(Claude Cowork 또는 Claude Code)이 **켜진 상태**로 두기
2. 스마트폰 Claude 앱에서 **모바일 앱 페어링**(연결) 설정
3. 폰에서 "이것 좀 해줘"라고 메시지 보내기
4. Claude가 컴퓨터에서 작업 수행
5. 나중에 컴퓨터에서 결과 확인

<div class="example-case">
<strong>예시: 출근길에 모닝 브리핑 만들기</strong>

**상황:** 김대리가 지하철 타고 출근하는 중. 사무실 컴퓨터는 켜져 있음.

1. 폰에서 Claude에게 메시지: "오늘 받은 이메일 확인해서 중요한 것만 요약해줘"
2. Claude가 사무실 컴퓨터에서 이메일 앱을 열고 확인
3. 중요한 이메일들을 정리한 모닝 브리핑 문서를 작성
4. 김대리가 사무실에 도착하면 이미 완성된 브리핑을 바로 확인

→ 출근길 30분이 낭비되지 않고, 도착 즉시 업무에 집중 가능
</div>

### Dispatch 활용 아이디어

| 상황 | 폰에서 보내는 지시 | Claude가 하는 일 |
|------|-------------------|-----------------|
| 출근길 | "이메일 확인해서 요약해줘" | 이메일 앱 열고 → 중요 메일 정리 |
| 회의 중 | "이 데이터 분석 보고서 만들어줘" | 스프레드시트 열고 → 차트 포함 보고서 작성 |
| 퇴근 후 | "내일 회의 자료 준비해줘" | 관련 문서 찾고 → 발표자료 초안 작성 |
| 주말 | "3D 프린팅 프로젝트 다음 단계 진행해줘" | 설계 파일 수정 → 프린터 설정 조정 |

## 주의할 점

### 보안 관련

- **민감한 데이터 주의** — 비밀번호, 금융 정보 등이 있는 앱에서는 Computer Use 사용을 자제하세요
- **일부 앱은 기본 차단** — 보안상 이유로 특정 앱은 Claude가 접근하지 못하도록 되어 있습니다
- **프롬프트 인젝션(악의적 명령 주입) 방어** — Anthropic이 자동으로 모델 내부 활성화를 스캔하여 이런 공격을 탐지합니다. 하지만 100% 완벽하지는 않으므로, **신뢰할 수 있는 앱부터 시작**하세요

### 기능 제한

- **현재 macOS만 지원** — Windows, Linux는 아직 사용 불가
- **데스크톱 앱이 깨어 있어야 함** — 컴퓨터가 잠자기 모드에 들어가면 Claude도 작업을 못 함
- **아직 리서치 프리뷰(실험 단계)** — 복잡한 작업은 가끔 실패하거나 두 번째 시도가 필요할 수 있음
- **화면 조작은 직접 연동보다 느림** — Slack API를 사용하는 것보다 화면에서 Slack을 클릭하는 게 당연히 느림

### 구독 조건

- **Claude Pro 또는 Claude Max 구독**이 필요합니다 (무료 플랜에서는 사용 불가)

## 실전 예시

<div class="example-case">
<strong>실전 케이스: 개발자의 Dispatch + Computer Use 활용</strong>

**상황:** 박개발자가 퇴근 후 카페에서 커피를 마시고 있음. 갑자기 버그 리포트가 올라옴.

1. **폰에서 Dispatch로 지시:** "방금 올라온 버그 리포트 확인하고, 관련 코드 찾아서 수정 가능한지 분석해줘"
2. **Claude가 사무실 컴퓨터에서:**
   - 버그 트래커를 열어 리포트 내용 확인
   - IDE에서 관련 코드 파일 검색
   - 원인 분석 결과를 정리
3. **박개발자가 나중에 확인:** "이 부분이 원인이고, 이렇게 수정하면 됩니다"라는 Claude의 분석 보고서가 준비되어 있음
4. 다음 날 출근하자마자 수정 작업에 바로 착수 가능

→ 카페에서 노트북 꺼내지 않고도, 버그 분석이 미리 완료됨
</div>

<div class="example-case">
<strong>실전 케이스: 비개발자의 Dispatch 활용</strong>

**상황:** 이마케터가 매주 월요일마다 지난주 마케팅 성과를 정리해야 함.

1. **Dispatch 정기 작업 설정:** "매주 월요일 오전 8시에 Google Analytics랑 광고 대시보드 확인해서 주간 성과 요약 만들어줘"
2. **매주 월요일 아침, Claude가 자동으로:**
   - 브라우저에서 Google Analytics 열기
   - 주요 지표(방문자 수, 전환율 등) 확인
   - 광고 대시보드에서 비용·클릭수 확인
   - 깔끔한 주간 보고서 문서 작성
3. **이마케터가 9시에 출근하면:** 보고서가 이미 완성되어 있음

→ 매주 1~2시간씩 걸리던 반복 작업이 자동화됨
</div>

## 정리

- **Computer Use** = Claude가 내 컴퓨터의 마우스·키보드를 직접 조작하여 앱을 사용하는 기능 (현재 macOS만 지원)
- **Dispatch** = 스마트폰에서 Claude에게 원격으로 업무를 지시하고, 결과를 나중에 확인하는 기능
- 두 기능을 결합하면 **자리를 비워도 Claude가 컴퓨터에서 알아서 일하는 환경**이 만들어짐 (Claude Pro/Max 구독 필요)
