## 이게 뭔가요?

Claude Code에 한꺼번에 4가지 신기능이 추가됐습니다:

1. **`/loop`** — "10분마다 메일 확인해줘"처럼 반복 작업을 자동으로 돌리는 기능
2. **Scheduled Tasks** — "매일 아침 9시에 이 작업 실행해줘"처럼 매일/매주 반복 예약
3. **Google Workspace CLI** — 구글 문서, 시트, 드라이브를 터미널에서 직접 제어
4. **Skills 2.0 (Evals)** — 만든 스킬이 진짜 잘 작동하는지 자동으로 테스트하고 점수 매기기

비유하자면:

> `/loop` = **알람 앱**에서 반복 알람 설정하는 것 (단, 3일 뒤 자동 종료)
> Scheduled Tasks = **매일 아침 자동으로 울리는 알람** (앱만 켜두면 계속)
> Skills 2.0 = **시험지로 실력 테스트** → 부족한 부분만 골라서 보충

## 왜 알아야 하나요?

지금까지 Claude Code는 **내가 말할 때만** 움직였습니다. "메일 확인해줘" 하면 한 번 확인하고 끝. 다시 보려면 또 말해야 했죠.

이제는:
- 한 번 설정하면 **자동으로 반복** 실행 (loop & schedule)
- 스킬을 만들면 **자동으로 품질 체크**까지 (evals)

즉, Claude Code가 **비서처럼 알아서 일하는** 단계로 진화한 겁니다.

### 기능 1: `/loop` — 세션 내 반복 자동화

`/loop`은 현재 세션에서 일정 간격으로 같은 작업을 반복 실행합니다.

```
/loop 10m 내 이메일에서 중요한 메일 확인해줘
```

이렇게 입력하면 **10분마다** Claude가 자동으로 메일을 확인합니다.

<div class="example-case">
<strong>💬 예시: 다양한 loop 활용</strong>

**반복 모니터링**
```
/loop 5m 이 폴더의 에러 로그 확인해줘
```

**일회성 리마인더**
```
/loop 1m 1분 뒤에 회의 시작이라고 알려줘
```
→ 리마인더는 1회만 실행되는 일회성 cron job으로 생성됩니다.

**매일 콘텐츠 작업**
```
/loop every day 내 유튜브 새 영상 확인하고 뉴스레터 초안 만들어줘
```

</div>

#### `/loop`의 한계 (중요!)

| 제한 사항 | 설명 |
|-----------|------|
| **3일 후 만료** | 안전장치 — 실수로 20개 루프가 영원히 돌지 않도록 |
| **세션 종료 시 사라짐** | 터미널을 닫으면 루프도 함께 종료 |
| **놓친 실행은 복구 안 됨** | 세션이 닫혀있던 동안의 루프는 그냥 건너뜀 |

> `/loop`은 "지금 이 프로젝트에서 며칠간 자동 체크" 용도입니다. 장기 자동화에는 Scheduled Tasks를 쓰세요.

#### 내부 작동 원리

`/loop`은 내부적으로 **cron job**을 생성합니다. 관련 도구 3가지:

- `cron create` — 새 반복 작업 생성
- `cron list` — 현재 실행 중인 작업 목록
- `cron delete` — 작업 삭제

### 기능 2: Scheduled Tasks — 매일/매주 자동 실행

`/loop`이 "지금 잠깐 반복"이라면, Scheduled Tasks는 **"매일 아침 9시에 실행"** 같은 장기 자동화입니다.

```
/schedule every day 내 유튜브 새 영상 확인하고 콘텐츠 리퍼포징 스킬로 뉴스레터와 트윗 만들어줘
```

<div class="example-case">
<strong>💬 예시: 스케줄 작업 설정 (데스크톱 앱)</strong>

Claude Code 데스크톱 앱 또는 Claude Code Work에서:

1. **New Task** → Schedule Tasks 클릭
2. 이름: `콘텐츠 리퍼포징`
3. 설명: `새 유튜브 영상을 뉴스레터로 변환`
4. 프롬프트: `유튜브 새 영상 확인 → 콘텐츠 리퍼포징 스킬 실행`
5. 빈도: 매일 오전 9시
6. 모델 선택 → 저장

</div>

#### `/loop` vs Scheduled Tasks 비교

| 항목 | `/loop` | Scheduled Tasks |
|------|---------|-----------------|
| 지속 시간 | 최대 3일 | 영구 |
| 세션 종료 시 | 사라짐 | 유지됨 |
| 놓친 실행 | 복구 안 됨 | **다시 실행해줌** |
| 실행 환경 | 현재 세션에서 실행 | 매번 새 세션 생성 |
| 사용 가능 환경 | 터미널, VS Code, 데스크톱 | **데스크톱 앱만** (현재) |

> 핵심: `/loop` = 단기 모니터링, Scheduled Tasks = 장기 루틴 자동화

#### 현재 제한사항

- **데스크톱 앱에서만** 사용 가능 (터미널, VS Code 미지원 — 곧 확장 예상)
- 컴퓨터가 켜져 있고 앱이 열려 있어야 실행됨
- 단, 놓친 작업은 앱을 다시 열면 자동 실행 (loop과 다른 점)

### 기능 3: Google Workspace CLI (오픈소스)

Google이 공식 오픈소스로 발표한 **Workspace CLI**는 하나의 도구로 Google 생태계 전체에 접근할 수 있습니다.

> 자세한 설정 방법은 [Google Workspace 연동 가이드](/youtube-update/google-workspace)를 참고하세요.

#### 이전 방식 vs 새 CLI

| 항목 | 이전 (MCP 서버) | 새 CLI |
|------|-----------------|--------|
| 접근 범위 | Gmail, Calendar만 | Drive, Docs, Sheets, Slides 전부 |
| 문서 포맷 | Raw Markdown (서식 깨짐) | **정상 포맷** (헤더, 이미지, 링크 완벽) |
| 설정 복잡도 | API 직접 연동 필요 | 터미널 설치 한 줄 |
| 레시피 | 없음 | **100개+ 내장 레시피** |

<div class="example-case">
<strong>📌 실전 케이스: 마크다운 → Google Docs 변환</strong>

Claude Code가 생성한 마크다운 콘텐츠를 Google Docs로 바로 변환:

```
"이 마크다운 파일을 Google Docs로 만들어줘"
```

→ 헤더, 볼드, 링크가 **제대로 포맷팅된** Google Docs가 생성됩니다.

기존 API 방식은 원시 마크다운이 그대로 텍스트로 들어가서 직접 서식을 잡아야 했지만, 새 CLI는 bash 명령어로 Google에 직접 통신하기 때문에 깔끔한 결과물이 나옵니다.

</div>

### 기능 4: Skills 2.0 — 스킬 평가(Evals) 시스템

지금까지 스킬을 만들면: 실행 → 결과 확인 → 수정 → 다시 실행... 이 과정을 감으로 반복했습니다.

Skills 2.0은 **자동 채점 시스템**을 도입합니다. 시험지를 만들어서 스킬을 여러 번 돌리고, 기준별로 점수를 매겨주는 거예요.

#### 사용 방법 (3단계)

**1단계: 스킬 만들기**

Anthropic의 `skill-creator` 스킬을 사용하거나 직접 작성:
- 명확한 이름과 트리거 설명
- 필요한 도구/커넥터 지정
- 레퍼런스 파일 연결 (브랜드 보이스, ICP 등)
- 단계별 프로세스 정의

**2단계: 평가 기준 설정 & 테스트**

```
내 카피라이팅 스킬을 테스트해줘.
기준:
1. 설득 기법 레퍼런스 파일을 참고하는가?
2. 호기심 갭(curiosity gap)을 사용하는가?
3. 창업자 스토리를 포함하는가?

"학교 커뮤니티 랜딩페이지 카피"로 5번 테스트해줘.
```

→ Claude가 **서브에이전트 5개를 동시에** 실행하여 병렬 테스트 수행

**3단계: 결과 분석 & 개선**

<div class="example-case">
<strong>📌 실전 케이스: 카피라이팅 스킬 평가 결과</strong>

테스트 결과가 **HTML 리포트**로 생성됩니다:

- 각 실행별 출력 결과 비교 (탭으로 전환)
- 기준별 통과/실패 표시
- 구체적인 개선 제안 (예: "호기심 갭이 여러 문장에 걸쳐 유지되지 않음")
- 벤치마크: 실행 시간, 토큰 사용량 비교

**결과 예시:**
| 기준 | 통과율 |
|------|--------|
| 고통을 구체화하는가 | 5/5 ✅ |
| 감정적 혜택까지 파고드는가 | 5/5 ✅ |
| 호기심 갭 사용 | 2/5 ❌ |
| 창업자 스토리 포함 | 3/5 ⚠️ |

→ 호기심 갭 관련 스킬 지시문을 강화하거나 레퍼런스 파일 수정

</div>

#### A/B 테스트도 가능

```
카피라이팅 스킬 적용 vs 미적용으로 A/B 테스트해줘.
비교 기준: 실행 시간, 토큰 수, 설득력 점수
```

→ 스킬이 실제로 품질을 높이는지, 아니면 토큰만 낭비하는지 확인 가능

#### 핵심 팁

- 한 번에 **1~3가지 기준**만 테스트 (너무 많으면 변수가 겹침)
- 테스트 → 개선 → 다시 테스트 사이클 반복
- 9/10 이상 통과하면 해당 기준은 졸업, 다음 기준으로 이동

## 실전 예시

<div class="example-case">
<strong>📌 실전 케이스: 콘텐츠 리퍼포징 전체 자동화</strong>

4가지 기능을 조합하면 이런 워크플로우가 가능합니다:

1. **Scheduled Tasks**로 매일 아침 9시에 실행
2. 유튜브 새 영상 확인 (스킬 활용)
3. 콘텐츠 리퍼포징 스킬로 뉴스레터 + 트윗 생성
4. **Google Workspace CLI**로 결과물을 Google Docs에 저장
5. **Skills 2.0 Evals**로 주기적으로 스킬 품질 체크

```
/schedule every day at 9am
유튜브 새 영상 확인 → 콘텐츠 리퍼포징 스킬 실행 → Google Docs에 저장
```

**결과**: 매일 아침 출근하면 뉴스레터 초안이 Google Docs에 준비되어 있음!

</div>

## 주의할 점

- `/loop`은 **최대 3일**만 유지됨 — 장기 작업은 반드시 Scheduled Tasks 사용
- Scheduled Tasks는 현재 **데스크톱 앱에서만** 가능 (VS Code, 터미널 미지원)
- Google Workspace CLI는 아직 **베타** — "not an officially supported Google product" 표기 있음 (Google 공식 오픈소스이나 정식 출시 전)
- Skills Evals 테스트 시 기준을 **너무 많이 넣지 마세요** — 1~3개가 적당

## 정리

- **`/loop`**: 지금 이 세션에서 며칠간 반복 체크할 때
- **Scheduled Tasks**: 매일/매주 꾸준히 자동 실행할 때
- **Google Workspace CLI**: 구글 문서/시트/드라이브를 터미널에서 직접 제어
- **Skills 2.0 Evals**: 스킬 품질을 자동 채점하고 빠르게 개선
