# Karpathy 4원칙: Claude Code가 코딩 전에 먼저 생각하게 만드는 플러그인

> **이 문서는 "Karpathy Skills" CLAUDE.md 플러그인을 다룹니다.**
> 저장소에 있는 다른 Karpathy 문서들(LLM 지식 베이스 메모리, Auto Research 파이프라인)과는 다른 주제입니다.

---

## 이게 뭔가요?

능숙한 직원과 신입의 차이를 생각해 보세요. 신입은 "이것 좀 해줘"라고 하면 일단 시작합니다. 무엇을 만들어야 하는지 파악하기도 전에요. 반면 능숙한 직원은 먼저 "어떤 방향으로 가면 될까요? 이렇게 이해했는데 맞나요?"를 물어봅니다.

AI 코딩 도구도 마찬가지입니다. 기본 상태의 Claude Code는 요청을 받으면 바로 코드를 씁니다. Andrej Karpathy(안드레이 카르파시, AI 분야 선구자, OpenAI 초기 멤버이자 전 Tesla AI 책임 연구자)가 이 문제를 지적했고, 개발자 커뮤니티가 이를 해결하는 CLAUDE.md(Claude Code 동작 규칙 파일) 플러그인을 만들었습니다.

GitHub 스타 약 9,000개, 포크(코드 복사) 614개, 주간 설치 2,100회.

## AI 코딩의 3가지 근본 문제

Karpathy가 지적한 문제들입니다.

| 문제 | 증상 |
|------|------|
| **침묵하는 가정** | 확인 없이 추측해서 코드를 작성. 모호한 요청도 그냥 진행 |
| **불필요한 복잡성** | 아무도 요청하지 않은 추상화(재사용 구조), 미래를 위한 설계 추가 |
| **범위 밖 수정** | 버그 하나 고치러 갔다가 관련 없는 함수 3개를 같이 리팩터링 |

---

## 4가지 원칙

### 원칙 1 — 코딩 전에 먼저 생각하기 (Think before coding)

코드를 작성하기 전에 가정(assumption, 전제조건)을 명시적으로 밝힙니다. 요청이 모호하면 여러 해석을 제시하고, 장단점을 설명하고, 추측 대신 질문합니다.

"예스 머신"이 아니라 같이 문제를 고민하는 협업자로 동작합니다.

### 원칙 2 — 단순함 우선 (Simplicity first)

최소 동작 코드(Minimum Viable Code)만 작성합니다. 요청받지 않은 것은 추가하지 않습니다.

- 한 번만 쓰이는 헬퍼 함수(helper function, 보조 함수) 금지
- 예방적 디자인 패턴(design pattern, 코드 설계 공식) 적용 금지
- 미래를 위한 추상화(abstraction, 공통 구조로 묶기) 금지

버튼 하나 추가하는 작업이라면? 버튼 핸들러와 파일 저장 코드만 작성합니다. 범용 저장 레이어(persistence layer)는 없습니다.

### 원칙 3 — 외과적 수정 (Surgical changes)

변경된 모든 줄은 사용자의 요청으로 추적 가능해야 합니다. 인접한 코드를 "개선"하지 않습니다. 버그를 고치면 그 버그의 코드만 바꿉니다.

### 원칙 4 — 목표 기반 실행 (Goal-driven execution)

명령형(imperative, "X를 해줘") 대신 목표형(declarative, "성공 기준은 Y야, 달성해줘")으로 프롬프트(입력 지시문)를 작성합니다.

LLM(대형 언어 모델)은 명령을 그대로 실행하는 건 잘 못하지만, 특정 목표에 도달할 때까지 반복하는 건 탁월합니다.

```
# 명령형 (비추천)
잘못된 입력값 처리 로직을 추가해줘

# 목표형 (추천)
잘못된 입력값에 대한 테스트를 작성하고, 그 테스트가 통과할 때까지 코드를 수정해줘
```

## Before / After 비교

<div class="example-case">

### 예시 1: 입력값 검증 추가 요청

**프롬프트:** "입력값 검증(validation)을 추가해줘"

**플러그인 없이:**
- 커스텀 에러 클래스 생성
- 검증 프레임워크(framework, 기능 묶음) 구축
- 재시도 로직(retry logic) 추가
- 약 200줄 코드

**플러그인 적용 후:**
1. "어떤 입력값을 검증해야 하나요?" 먼저 질문
2. 요청에 맞는 30줄 코드 작성
3. 동작 확인용 테스트 추가

</div>

<div class="example-case">

### 예시 2: 버그 수정 요청

**상황:** 특정 함수에 버그 발생

**플러그인 없이:**
- 버그 수정
- 인접한 함수 3개 리팩터링(refactoring, 코드 재구성)
- "이 부분도 개선했어요" 추가 변경

**플러그인 적용 후:**
- 버그가 있는 그 한 줄만 수정
- 다른 코드 일절 건드리지 않음
- 변경 이유를 명확히 설명

## 설치 방법

### 방법 A — Claude Code 마켓플레이스 (가장 간단)

Claude Code 내에서 마켓플레이스(Marketplace)를 열고 "karpathy skills"를 검색해 설치합니다.

**단축키:**

| 운영체제 | 마켓플레이스 열기 |
|----------|------------------|
| Mac / Windows | Claude Code 세션에서 `/plugin install karpathy-skills@claude-plugins-official` 입력 |

### 방법 B — 프로젝트에 직접 추가

터미널(명령 입력 창)에서 프로젝트 폴더로 이동한 뒤, GitHub에서 `CLAUDE.md` 파일을 내려받아 프로젝트 루트에 저장합니다. 저장소 주소: `github.com/forrestchang/andrej-karpathy-skills`

이 파일 하나가 Claude Code의 동작 방식을 바꿉니다. 런타임 의존성(runtime dependency, 실행 시 필요한 외부 라이브러리)이 전혀 없습니다.

## 핵심 정리

이 플러그인의 본질은 **명령형에서 목표형으로의 전환**입니다.

AI 코딩 도구에게 "X를 해줘"라고 말하는 대신, "Y가 충족되면 성공이야"라고 말하는 방식입니다. AI는 목표를 향해 스스로 반복하며 개선하고, 그 과정에서 불필요한 코드를 만들지 않습니다.

설정 파일 한 줄이 Claude Code를 실행기가 아닌 협업자로 바꿉니다.
