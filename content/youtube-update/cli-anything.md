## 이게 뭔가요?

보통 우리가 프로그램을 쓸 때는 마우스로 클릭하잖아요. 포토샵에서 이미지 편집하려면 메뉴 클릭, 필터 클릭, 저장 클릭... 이런 식으로요.

**CLI-Anything**은 이런 "클릭으로 쓰는 프로그램"을 **"말로 쓰는 프로그램"으로 바꿔주는 도구**입니다. 여기서 CLI(키보드로 명령어를 입력하는 화면)란 마우스 대신 글자를 타이핑해서 컴퓨터를 조작하는 방식을 말해요.

비유하자면 이래요:

> 원래: 리모컨 버튼을 하나하나 눌러서 TV 조작
> CLI-Anything 적용 후: "넷플릭스 열어서 어제 보던 거 이어 틀어줘"라고 말하면 알아서 실행

Claude Code가 터미널(명령어 창)에서 살고 있잖아요. CLI-Anything은 다른 프로그램들도 터미널에서 조작할 수 있게 만들어서, Claude Code가 **직접 그 프로그램을 제어**할 수 있게 해주는 겁니다.

이 도구는 홍콩대학교 연구실에서 만들었어요. LightRAG, DragAnything 같은 유명 AI 프로젝트를 만든 팀이라, 실력이 검증된 곳입니다. GitHub(코드 저장소 서비스)에 올라가자마자 **10,000개 스타**를 달성할 정도로 화제가 됐어요.

## 왜 알아야 하나요?

지금까지 AI 에이전트가 소프트웨어를 제어하려면 세 가지 방법밖에 없었는데, 다 문제가 있었어요:

### 기존 방법들의 한계

**방법 1: 화면 자동화** — 화면을 보고 마우스를 자동으로 클릭하는 방식

마치 누군가가 내 컴퓨터를 원격으로 조종하는 것처럼 동작해요. 문제는 **메뉴 위치가 1픽셀만 바뀌어도 고장**난다는 거예요. 카페에서 주문할 때 매번 같은 자리에 키오스크가 있어야 하는 것처럼, 너무 깨지기 쉬워요.

**방법 2: MCP(외부 도구 연결)** — 누군가 미리 만들어둔 "연결 다리"를 사용하는 방식

잘 되긴 하는데, **누군가 미리 만들어둬야** 해요. Slack, GitHub 같은 유명한 도구는 MCP가 있지만, 세상 대부분의 프로그램에는 MCP가 없습니다. 택시 앱은 있는데 내가 타고 싶은 마을버스는 앱이 없는 것과 비슷해요.

**방법 3: Claude에게 직접 부탁하기** — 파일을 Claude에게 넘기고 "이거 편집해줘" 하는 방식

간단한 건 되는데, Claude가 그 프로그램의 파일 형식을 정확히 모르면 **파일이 깨지는 경우가 많아요**. 외국어를 대충 아는 사람한테 번역 맡기면 의미가 이상해지는 것과 같아요.

**CLI-Anything**은 이 세 가지 한계를 모두 해결합니다. 프로그램의 소스코드(설계도)를 읽어서 Claude가 정확하게 제어할 수 있는 명령어 도구를 자동으로 만들어주거든요.

### CLI-Anything을 쓰면:

- **Draw.io** 같은 다이어그램 도구를 Claude Code가 직접 조작
- **Blender** 같은 3D 프로그램도 명령어로 제어 가능
- **Audacity** 같은 오디오 편집기도 자동화

MCP(외부 도구 연결)보다 **토큰(AI가 처리하는 텍스트 단위) 비용이 약 30배 절약**됩니다. MCP는 연결할 때마다 수천 개의 토큰을 사용하는데, CLI는 그럴 필요가 없거든요. 비유하면 매번 통역사를 고용하는 것(MCP) vs 내가 직접 그 나라 말을 배우는 것(CLI)의 차이예요.

## 어떻게 하나요?

### 준비물

- Python 3.10 이상
- Claude Code 설치 완료
- 연결하고 싶은 오픈소스 프로그램의 Git(코드 버전 관리 도구) 저장소

### 1단계: CLI-Anything 플러그인(추가 기능을 붙이는 확장 도구) 설치

```bash
claude plugin add cli-anything
claude plugin install cli-anything
```

설치 확인은 `/plugin`에서 installed 목록에 나오면 OK.

> 설치 후 `/reload plugins` 또는 Claude Code 재시작 필요!

### 2단계: 대상 프로그램 클론

연결하고 싶은 프로그램의 소스코드를 가져옵니다.

```bash
git clone https://github.com/jgraph/drawio.git
```

### 3단계: CLI 도구 자동 생성

```bash
cli-anything ./drawio
```

이 한 줄이면 끝! 내부적으로 7단계 파이프라인이 자동 실행됩니다:

1. 코드 분석 → 2. 설계 → 3. 구현 계획 → 4. 테스트 작성 → 5. 문서화 → 6. 빌드 → 7. 개선

약 20분 정도 걸리고, 완료되면 "all 9 success criteria met"이라고 나옵니다.

### 4단계: 사용하기

이제 Claude Code에서 바로 활용 가능!

## 이 프로그램에 CLI-Anything을 쓸 수 있을까?

CLI-Anything을 시도하기 전에 **딱 하나만 확인**하세요:

> **"이 프로그램이 화면(GUI) 없이도 작동하는 부분이 있는가?"**

쉽게 말하면:

- 프로그램이 파일을 저장하거나, API(프로그램끼리 대화하는 통로)가 있거나, 백그라운드에서 뭔가 처리하는 기능이 있으면 → **CLI-Anything 사용 가능**
- 화면을 직접 보면서 마우스로만 조작할 수 있는 프로그램이면 → **CLI를 만들어도 실제로 동작하지 않을 수 있음**

| Draw.io | 파일 형식(.drawio)으로 저장/편집 가능 | ✅ 적합 |

## MCP vs CLI-Anything: 뭘 써야 하나요?

둘 다 "Claude가 외부 도구를 쓸 수 있게 하는 방법"이지만, **서로 다른 상황에서 쓰는 도구**예요. 대체 관계가 아니라 **보완 관계**입니다.

| 상황 | 추천 도구 |
|------|-----------|
| Slack, GitHub 같이 **MCP 서버가 이미 있는** 유명 도구 | MCP 사용 |
| MCP가 없는 **오픈소스 프로그램** (회사 내부 도구 등) | CLI-Anything 사용 |
| MCP가 있지만 **토큰 비용을 줄이고 싶을 때** | CLI-Anything 고려 |

비유하면 이래요:
- **MCP** = 배달앱 (있는 식당은 편하게 주문 가능, 없는 식당은 이용 불가)
- **CLI-Anything** = 직접 레시피 만들기 (어떤 식당이든 레시피만 있으면 요리 가능)

## 주의할 점

- **오픈소스만 가능**: 소스코드가 공개된 프로그램만 CLI로 만들 수 있습니다. 포토샵 같은 상용 소프트웨어는 안 돼요.
- **첫 생성에 시간 필요**: CLI 만드는 데 15~20분 걸려요 (200MB 규모 기준). 하지만 한 번 만들면 계속 쓸 수 있습니다.
- **GUI 전용 프로그램은 한계**: 화면 없이 동작하지 않는 프로그램은 CLI를 만들어도 효과가 제한적이에요.
- **MCP와 동시 사용**: 같은 프로그램에 MCP와 CLI-Anything을 동시에 쓸 필요는 없어요. 하나를 골라서 쓰세요.

## 정리

- CLI-Anything = "아무 프로그램이든 Claude Code가 조종할 수 있게 만드는 도구"
- 오픈소스 프로그램 + 명령어 한 줄 = 자동 CLI 생성
- MCP와 대체가 아닌 **보완 관계** — MCP 없는 도구에 특히 유용
- MCP 대비 **토큰 30배 절약** — 비용 효율적
