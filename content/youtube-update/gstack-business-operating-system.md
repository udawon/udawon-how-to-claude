## 이게 뭔가요?

gstack은 Y Combinator(세계 최대 스타트업 엑셀러레이터) CEO인 Garry Tan이 만든 오픈소스 프레임워크(뼈대 도구)입니다. Claude Code에 **13가지 비즈니스 스킬**을 설치하면, AI가 단순히 코드만 짜는 게 아니라 CEO 의사결정, 제품 관리, QA(품질 검증) 테스트, 보안 점검까지 전체 업무 사이클을 자동으로 처리합니다.

쉽게 말하면, Claude Code에 **"회사 운영 매뉴얼"**을 통째로 넣어주는 것입니다. 비서에게 업무 매뉴얼을 주면 알아서 일하듯이, gstack의 스킬들이 Claude Code에게 "이런 상황에선 이렇게 해"라는 지침을 제공합니다.

출시 2주 만에 GitHub에서 **3,000 Fork, 25,000 스타**를 기록하며 큰 주목을 받았습니다.

## 왜 알아야 하나요?

- **혼자서도 팀 규모의 업무를 처리할 수 있습니다** — CEO 리뷰, 비즈니스 분석, QA 테스트를 AI가 대신하니까요
- **Garry Tan의 실전 경험이 녹아 있습니다** — 60일간 60만 줄 이상의 코드를 작성하고, 하루 2만 줄 이상을 커밋한 실전 사용자의 노하우입니다
- **스킬 시스템의 활용 범위를 넓혀줍니다** — 코딩 도우미를 넘어 비즈니스 운영 도구로 Claude Code를 확장하는 구체적인 방법을 보여줍니다
- **설치가 간단합니다** — GitHub URL만 주면 Claude Code가 알아서 스킬을 설치합니다

## 어떻게 하나요?

### 방법 1: gstack 설치하기

Claude Code에서 gstack GitHub 저장소 URL을 주고 "스킬 활성화해줘"라고 말하면 됩니다. 별도의 복잡한 설치 과정이 필요 없습니다.

1. Claude Code를 실행합니다
2. gstack GitHub 저장소 링크를 전달합니다 (https://github.com/garrytan/gstack)
3. "이거 스킬 설치해줘"라고 말합니다
4. Claude Code가 자동으로 스킬을 다운로드하고 설정합니다

<div class="example-case">
<strong>예시: gstack 설치 후 바로 사용하기</strong>

```
사용자: 이 저장소 스킬 활성화해줘 https://github.com/garrytan/gstack
Claude: (자동으로 스킬 다운로드 및 설정)

사용자: /plan-ceo-review 이번 주 진행 상황 정리해줘
Claude: (CEO 관점에서 프로젝트 상태를 분석하고 리포트 작성)
```

설치 후에는 `/plan-ceo-review`, `/retro` 같은 스킬 커맨드를 바로 사용할 수 있습니다.

</div>

### 방법 2: 핵심 스킬만 골라서 활용하기

gstack 전체를 설치하지 않더라도, Garry Tan의 접근법에서 영감을 받아 자신만의 스킬을 만들 수도 있습니다. gstack의 핵심 철학을 참고하면 됩니다.

## gstack의 13가지 스킬 카테고리

gstack은 크게 4가지 영역에 걸쳐 13개의 스킬을 제공합니다.

### 기획 & 의사결정
- **plan CEO review** — CEO 관점에서 프로젝트를 분석합니다. 사업 방향, 우선순위, 리스크를 점검합니다
- **plan review** — 일반적인 계획 검토. 실행 가능성과 빠진 부분을 확인합니다
- **plan proposal** — 새로운 기능이나 프로젝트 제안서를 작성합니다

### 코드 & 품질
- **code review** — 코드 변경사항을 자동으로 리뷰합니다
- **execute document** — 문서 기반으로 코드를 실행합니다
- **security audit** — 보안 취약점을 자동으로 검사합니다

### QA 테스트
- **QA smoke test** — 빠른 기본 동작 확인 테스트
- **QA general** — 일반적인 품질 검증
- **QA security** — 보안 관점의 철저한 테스트 (SQL 인젝션, 크로스사이트 스크립팅, LLM 프롬프트 인젝션 등)

### 운영 유틸리티
- **retro** (회고) — 프로젝트 회고를 자동으로 진행합니다. 24시간~30일 범위에서 커밋 분석
- **auto upgrade** — gstack 자체를 자동으로 업그레이드합니다
- **health check** — 프로젝트 건강 상태를 점검합니다
- **key management** — 암호화 키를 안전하게 관리합니다

## gstack의 핵심 설계 철학

Garry Tan은 두 가지 핵심 원칙을 강조합니다:

### 원칙 1: AI에게 명확한 역할을 부여하라

> "AI의 힘으로 10배에서 100배 생산성을 올리되, 각 부분을 해결할 때 항상 구조적으로 접근한다"

CEO에게는 CEO 리뷰를, QA 담당자에게는 테스트를 시키듯이, AI에게도 **역할별로 스킬을 나눠서** 맡기는 방식입니다.

### 원칙 2: 자동화를 신뢰하되 검증하라

> "100% 커버리지를 목표로 하고 항상 QA와 회귀 테스트를 먼저 작성하는 것이 gstack의 DNA"

코드를 작성하면 반드시 테스트가 따라붙고, 테스트를 통과해야 다음 단계로 넘어갑니다.

## 기술적 특징 (개발자 참고)

### 왜 Bun을 선택했나?

gstack은 Node.js 대신 Bun(번, 빠른 자바스크립트 런타임)을 사용합니다.

| 비교 항목 | Node.js | Bun |
|-----------|---------|-----|
| CLI(명령어 도구) 시작 시간 | ~100ms | ~1ms |
| 설치 후 파일 | node_modules 폴더 필요 | 단일 바이너리 |
| SQLite 지원 | 별도 설치 필요 | 내장 |
| TypeScript | 별도 변환 필요 | 네이티브 지원 |
| HTTP 서버 | 외부 프레임워크 필요 | 내장 |

### 접근성 트리 방식 vs 스크린샷 방식

gstack은 웹 페이지를 분석할 때 **스크린샷 대신 접근성 트리**(Accessibility Tree)를 사용합니다. 접근성 트리란, 웹 페이지의 구조를 텍스트로 변환한 것입니다. 마치 시각장애인용 화면 읽기 프로그램이 페이지를 읽는 것처럼, 화면을 "텍스트로 보는" 방식입니다.

| 비교 항목 | 스크린샷 방식 | 접근성 트리 방식 |
|-----------|-------------|----------------|
| 첫 분석 시간 | 2~3초 | ~3초 (비슷) |
| 이후 분석 시간 | 매번 2~3초 | 100~200ms |
| 20개 요소 분석 시 | 40초 이상 | 수초 이내 |
| 토큰(AI 사용량) | 이미지라 많음 | 텍스트라 적음 |
| 보안 | 화면 정보 노출 | 구조만 노출 |

### 보안 아키텍처

gstack은 보안에 특히 신경을 씁니다:

1. **macOS Keychain 연동** — 비밀번호를 운영체제의 보안 저장소에 저장합니다 (Mac 기준)
2. **AES 암호화** — 키체인 비밀번호에서 13회 반복 해싱으로 16바이트 암호화 키를 생성합니다
3. **메모리 보호** — 암호화 키는 메모리에서만 존재하고, 디스크에 기록되지 않습니다
4. **자동 파기** — 프로그램 종료 시 캐시와 키가 자동으로 삭제됩니다

## 실전 예시

<div class="example-case">
<strong>실전 케이스: 워크트리로 10개 프로젝트 동시 진행</strong>

gstack은 Git 워크트리(worktree, 하나의 저장소에서 여러 작업 공간을 만드는 기능)를 활용하여 **동시에 여러 작업을 병렬로 진행**할 수 있습니다.

```
터미널 1: QA 스모크 테스트 중
터미널 2: 리뷰 완료 → PR 생성 중
터미널 3: 회고 작성 중
터미널 4~10: 다른 브랜치에서 각각 다른 작업 진행
```

각 워크트리는 독립적이기 때문에 서로 충돌 없이 작업할 수 있습니다. 이것이 Garry Tan이 "하루에 1만 줄에서 2만 줄의 코드를 커밋할 수 있었던 비결"입니다.

</div>

<div class="example-case">
<strong>실전 케이스: plan CEO review로 의사결정하기</strong>

```
사용자: /plan-ceo-review 다음 주 출시 계획 검토해줘

Claude: (CEO 관점에서 분석)
- 현재 프로젝트 상태: 코드 완성도 85%
- 리스크: 결제 모듈 테스트 미완료
- 권장: 출시를 3일 연기하고 QA 완료 후 진행
- 우선순위: 1) 결제 QA 2) 성능 테스트 3) 문서 정리
```

CEO가 아니더라도, **프로젝트를 경영자 시각으로 점검**할 수 있어서 혼자 일하는 개발자나 소규모 팀에게 특히 유용합니다.

</div>

## 주의할 점

- **macOS에서 가장 잘 작동합니다** — 보안 기능(Keychain 연동)이 macOS 기반으로 설계되어 있습니다. Windows에서는 일부 보안 기능이 다르게 동작할 수 있습니다
- **Playwright(웹 자동화 도구)가 필요합니다** — 접근성 트리 기능을 사용하려면 Playwright가 설치되어 있어야 합니다
- **MCP 서버를 최소화하세요** — gstack 제작자는 MCP(외부 도구 연결 기능) 서버를 많이 쓰지 않기를 권장합니다. 토큰을 많이 소모하고, gstack의 접근성 트리 방식이 더 효율적이기 때문입니다
- **gstack은 계속 업데이트됩니다** — 오픈소스 프로젝트이므로 지속적으로 스킬이 추가되고 있습니다. 최신 버전을 유지하세요

## 정리

- gstack은 Claude Code에 비즈니스 운영 스킬 13개를 설치하여 CEO 리뷰부터 QA까지 자동화하는 프레임워크입니다
- Bun + SQLite + 접근성 트리 방식으로 빠르고 안전하게 동작합니다
- Y Combinator CEO의 실전 경험에서 나온 도구이므로, 특히 1인 개발자나 소규모 팀이 "혼자서 팀처럼 일하기"에 활용할 수 있습니다
