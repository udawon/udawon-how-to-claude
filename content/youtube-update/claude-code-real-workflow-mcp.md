# Claude Code 실전 워크플로우 — 타입 분석부터 MCP 연동까지

## 이게 뭔가요?

채팅창에 코드를 붙여넣고 답변을 기다리는 방식은 마치 퀵서비스를 쓸 때마다 물건을 직접 들고 가서 건네주는 것과 같습니다. Claude Code는 다릅니다. 터미널 안에 살면서 파일 시스템 전체를 이해하고, 직접 파일을 열고 수정하고 테스트까지 실행하는 **코딩 에이전트** (자율적으로 작업을 수행하는 AI 도구)입니다.

이 문서에서는 실제 개발 현장에서 바로 쓸 수 있는 4가지 케이스와 컨텍스트 관리, MCP 서버 연동까지 다룹니다.

---

## 설치

**Mac / Linux:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows:**
PowerShell에서 공식 설치 스크립트를 실행하거나, WinGet 패키지 관리자를 사용합니다. (자세한 방법은 claude.ai/code 공식 문서 참조)

설치 후 `claude`를 입력하면 **API 키** (Anthropic 서비스 접근 인증 코드)를 입력하라는 메시지가 나옵니다. 입력하면 터미널과 AI 사이의 암호화 연결이 완성됩니다.

이후 프로젝트 루트 폴더에서 `claude`를 실행하면 됩니다. 에이전트가 폴더 구조를 분석하기 시작합니다.

> **참고:** 실행 명령어는 `claude`입니다 (`claude-code`가 아닙니다).

## 실전 케이스 4가지

<div class="example-case">

### 케이스 1 — 타입 오류 분석 및 수정

**상황:** 인증 설정 파일에 **타입 에러** (TypeScript 등에서 변수 타입이 맞지 않아 발생하는 오류)가 의심될 때.

**프롬프트:**
```
인증 설정 파일을 분석하고 인터페이스의 타이핑 오류를 감지해줘
```

에이전트가 백그라운드에서 파일을 열고 타입 정의 관련 경고 목록을 출력합니다.

**이어서:**
```
필요한 수정을 적용하고 프로젝트 표준에 맞게 파일을 포맷해줘
```

우측 패널에 변경 사항이 초록(추가)·빨강(삭제)으로 실시간 표시됩니다.

**효과:** 타입 오류를 찾아 수동으로 고치던 작업이 프롬프트 2개로 끝납니다.

</div>

<div class="example-case">

### 케이스 2 — 의존성 감사

**상황:** `package.json`의 라이브러리 버전이 오래되어 호환성 충돌이 의심될 때.

**프롬프트:**
```
의존성 파일을 검토하고 호환성 충돌을 일으키는 오래된 버전을 찾아줘
```

에이전트가 **패키지 트리** (프로젝트가 사용하는 외부 라이브러리 목록과 관계)를 스캔하고 6개월 이상 업데이트되지 않은 라이브러리를 감지합니다.

에이전트가 자동으로 제안하는 것:
1. 버전 업데이트 마이그레이션 스크립트
2. API 변경 사항 반영 코드 수정
3. 아무것도 깨지지 않았는지 확인하는 통합 테스트 생성

**효과:** 수동 버전 확인, 변경 로그 읽기, 마이그레이션 스크립트 작성이 한 번에 처리됩니다.

</div>

### 케이스 3 — 단위 테스트 자동 생성

**상황:** 인증 모듈의 **단위 테스트** (개별 함수/모듈이 올바르게 동작하는지 검증하는 코드)가 없을 때.

**프롬프트:**
```
인증 모듈의 단위 테스트를 생성해줘. 모든 엣지 케이스를 커버해야 해
```

에이전트가 기존 코드의 모든 분기 조건을 분석하고, 각 경계 조건에 대한 **파라미터 테스트** (여러 입력값을 자동으로 순환하며 검증하는 테스트 방식)를 작성한 뒤 즉시 실행합니다. 첫 실행에서 100% 통과율을 보이는 경우가 많습니다.

### 케이스 4 — 비동기 리팩토링

**상황:** 구형 콜백 방식 코드를 **async/await** (비동기 처리를 동기 코드처럼 읽기 쉽게 작성하는 문법)로 바꾸고 싶을 때.

**프롬프트:**
```
이 모듈을 async/await를 쓰도록 리팩토링하고 전체에 걸쳐 타입 일관성을 유지해줘
```

에이전트가 처리하는 순서:
1. 모든 모듈 의존성 매핑 (예상치 못한 사이드이펙트 방지)
2. 타입 일관성 유지하며 리팩토링 적용
3. 기존 테스트 전부 통과 여부 검증

## 컨텍스트 관리

### /compact — 세션 요약

복잡한 작업을 마친 뒤 에이전트와 대화를 계속하면 **컨텍스트** (에이전트가 기억하는 대화 내용과 코드 정보)가 쌓여 느려지거나 한계에 도달합니다.

```
/compact
```

이 명령어를 실행하면 세션을 요약하고 중복 토큰을 제거합니다. 컨텍스트를 잃지 않고 작업을 이어갈 수 있습니다.

**사용 타이밍:** 큰 기능 하나를 완성했을 때, 또는 에이전트 응답이 느려지기 시작할 때.

| 키보드 단축키 | Mac | Windows |
|--------------|-----|---------|
| 터미널 새 창 | `Cmd + T` | `Ctrl + Shift + T` |
| 터미널 클리어 | `Cmd + K` | `Ctrl + L` |
| 명령어 히스토리 | `↑ / ↓` | `↑ / ↓` |

### CLAUDE.md — 팀 규칙 자동 적용

프로젝트 루트에 `CLAUDE.md` 파일을 만들면 에이전트가 매 세션 시작 시 자동으로 읽습니다.

```markdown
# CLAUDE.md

## 네이밍 규칙
- 컴포넌트: PascalCase
- 유틸 함수: camelCase
- 상수: UPPER_SNAKE_CASE

## 허용된 디자인 패턴
- Repository 패턴으로 데이터 접근 추상화
- 서비스 레이어 분리 필수

## 테스트 컨벤션
- 단위 테스트는 __tests__ 폴더에 배치
- 통합 테스트 파일명은 *.integration.test.ts
```

**효과:** 팀원마다 에이전트에게 규칙을 매번 설명할 필요 없이, 한 번 정의하면 모든 세션에 일관되게 적용됩니다.

## MCP 서버 연동

**MCP** (Model Context Protocol, 에이전트가 외부 도구와 직접 통신하는 프로토콜)를 활용하면 Claude Code가 데이터베이스, 모니터링 도구 등과 직접 연결됩니다.

### 데이터베이스 연동 예시

로컬 DB에 MCP 서버를 연결한 뒤:

```
프로덕션 데이터베이스의 쿼리 플랜을 분석하고 느린 쿼리를 찾아줘
```

에이전트가 자동으로 수행하는 흐름:

```
쿼리 플랜 검사
      │
      ▼
병목 지점 감지
      │
      ▼
인덱스 추가 마이그레이션 작성
      │
      ▼
검증 테스트 생성 및 실행
```

수동으로 쿼리 플랜을 읽고, 인덱스 전략을 고민하고, 마이그레이션을 작성하던 작업이 프롬프트 하나로 끝납니다.

### CI 도구 연동

모니터링 도구를 연결하면 인시던트(서비스 장애)를 자율적으로 분석하고 해결책을 제안하게 할 수도 있습니다.

| 상황 | 도구 | 효과 |
|------|------|------|
| 타입 오류 수정 | 직접 프롬프트 | 파일 열고 수정, 포맷까지 자동 |

| 테스트 생성 | 직접 프롬프트 | 엣지 케이스 포함 즉시 실행 |
| 리팩토링 | 직접 프롬프트 | 의존성 분석 후 안전하게 변환 |
| 컨텍스트 정리 | `/compact` | 세션 압축, 속도 유지 |
| 규칙 일관성 | `CLAUDE.md` | 매 세션 팀 컨벤션 자동 적용 |
| DB 직접 분석 | MCP 서버 | 쿼리 → 마이그레이션 → 검증 자동화 |

## 시작 순서

1. `curl -fsSL https://claude.ai/install.sh | bash` (Mac/Linux) 또는 공식 문서의 Windows 설치 안내 참고
2. 프로젝트 루트에서 `claude` 실행 및 API 키 입력
3. `CLAUDE.md` 파일 생성 후 팀 규칙 정의
4. 위 4가지 케이스 중 하나를 바로 시도
5. 큰 작업 완료 후 `/compact` 습관화
