## 이게 뭔가요?

"브랜드 가이드(Charte Graphique)"는 우리 브랜드의 색·폰트·로고·말투 같은 디자인 규칙을 한 권으로 모은 문서입니다. 광고 대행사나 디자이너가 클라이언트에게 넘기는 그 "브랜드북"과 같은 개념입니다.

이 영상에서는 그 브랜드북을 **그냥 보관용 PDF가 아니라 AI가 매번 자동으로 따르게 만드는** 방법을 보여줍니다. 핵심 흐름은 두 단계입니다.

1. 내 사이트·로고·이미지를 Claude에 업로드 → 브랜드 가이드 문서(.md, 마크다운 파일) 자동 생성
2. 그 문서를 **Skill(스킬, Claude가 특정 상황에서 자동으로 꺼내 쓰는 전문 능력 모음)**로 변환 → 그 다음부터는 "PDF로 견적서 만들어줘" 한 줄에 같은 색·같은 폰트·같은 말투가 자동 적용

마치 신입 디자이너에게 우리 브랜드북을 한 번 외우게 시킨 뒤로는 "그냥 알아서 우리 브랜드 톤으로 만들어줘"가 되는 것과 같습니다.

## 왜 알아야 하나요?

1인 사업자·마케터·디자이너·프리랜서에게 가장 큰 시간 낭비는 "매번 같은 톤을 다시 설명하는 일"입니다. 견적서, 제안서, 인스타 이미지, 블로그 썸네일을 만들 때마다 "우리 브랜드 컬러는 #xxx, 폰트는 OOO, 말투는 직설적"을 다시 입력하고 있다면 이 영상의 방법으로 그 반복이 사라집니다.

영상 제작자가 강조한 가장 중요한 한 줄은 이것입니다.

> 21세기 핵심 역량은 prompt(프롬프트, AI에게 보내는 요청 메시지)를 잘 쓰는 게 아니라 **강력한 컨텍스트(context, AI에게 미리 알려주는 배경 정보)를 주는 것**.

브랜드 가이드 Skill은 그 "강력한 컨텍스트"를 한 번 만들어 두고 평생 재활용하는 가장 빠른 방법입니다.

## 어떻게 하나요?

### 1단계: 95% 확신 들 때까지 만들지 마라 — 시작 프롬프트

Claude Opus(클로드의 가장 똑똑한 모델, 매일 쓰는 결과물이라 가장 좋은 모델 권장)를 켜고 다음 프롬프트를 그대로 입력합니다.

<div class="example-case">
<strong>한국어로 그대로 따라 쓸 수 있는 시작 프롬프트</strong>

```
안녕하세요 Claude. 제 브랜드 가이드를 만들고 싶습니다.
제 사이트, 로고, 브랜드 이미지를 공유하겠습니다.

조건:
- 모든 요소를 95% 확신할 때까지 만들지 마세요.
- 만들기 전에 제가 검증할 plan(계획)을 먼저 제시하세요.
- 부족한 정보가 있으면 저에게 질문하세요.
- 출력 형식은 마크다운(.md) 파일.
```

이 프롬프트는 영상 제작자가 수많은 클라이언트 작업을 거쳐 정착한 패턴입니다. "바로 만들어줘"가 아니라 "계획부터 보여줘"가 핵심.
</div>

**왜 이게 중요한가요?** 영상 제작자의 경험: AI는 대화가 길어질수록 결과 품질이 떨어집니다. 처음에 계획을 세우고 한 번에 잘 만드는 편이 크레딧(AI 사용료)도 아끼고 결과물도 더 좋습니다.

### 2단계: 자료를 충분히 던지기

Claude에 다음을 첨부합니다.

| 자료 | 비고 |
|------|------|
| 로고 파일 | 변형 버전 있으면 모두 |
| 사이트 URL | "이 주소를 분석해줘" |
| 제품·패키지 사진 | 다양한 각도 |
| 사람이 제품을 들고 있는 사진 | 사용 맥락 |
| 기존 SNS 게시물 캡처 | 톤·말투 참고용 |

**원칙**: 자료가 많을수록 좋습니다. 영상에서는 사이트 캡처·다양한 패키지 사진·로고 변형까지 모두 던졌습니다.

### 3단계: Claude가 질문하는 대로 답하기

영상에서 Claude가 실제로 던진 질문 예시.

- 정확한 폰트(typography)를 아시나요?
- 컬러 코드(hex code, 색을 표시하는 6자리 영문/숫자)가 있나요?
- 추가 로고 변형이 있나요?
- **이 가이드의 사용 목적은 무엇인가요?**

마지막 질문이 가장 중요합니다. "내부 데모용인지, 클라이언트 전달용인지, 메모리에 저장해도 되는지"에 따라 결과가 달라집니다.

### 4단계: 계획 검증 → 승인 → 생성

Claude가 만든 plan을 읽어보고 마음에 들면 "validé(승인)"라고 답합니다. 영상에서는 약 1분 30초 만에 .md 파일이 완성되었습니다.

완성된 .md 파일에는 이런 정보가 담깁니다.

- 로고 사용 규칙(색·두께·인쇄용 사양)
- 컬러 팔레트(hex code 포함)
- 타이포그래피(예: Satoshi Black, Satoshi Regular)
- **말투(tone of voice)** — "직설적, 펀치 있는, 돌려 말하지 않음"

영상 제작자는 강조합니다.

> 브랜드 가이드는 단순히 시각 디자인이 아니다. **말투까지 들어가야 진짜 브랜드 가이드**.

### 5단계: .md를 Skill로 변환

이제 핵심 단계입니다. Claude에게 다음과 같이 요청합니다.

<div class="example-case">
<strong>Skill 생성 프롬프트 (한국어)</strong>

```
이 브랜드 가이드를 항상 자동으로 적용하는 Skill을 만들어주세요.
앞으로 제가 PDF·이미지·웹 페이지 같은 산출물을 만들 때마다
이 Skill이 호출되어 디자인이 일관되게 유지되어야 합니다.
```

Claude는 내부적으로 "Skill creator"(스킬을 만드는 도구)를 호출해 적절한 Skill 파일을 자동으로 작성합니다.
</div>

생성된 Skill은 **Personalize(개인화) 메뉴 → Skills**에 자동 등록됩니다. 이후로는 Claude가 알아서 이 Skill을 꺼내 씁니다.

### 6단계: 자동 호출 시연

Skill 등록 후에는 더 이상 "우리 컬러는 OOO, 폰트는 XXX"라고 알려줄 필요가 없습니다.

영상 시연 장면.

- "Grunch라는 브랜드의 PDF 광고 한 장 만들어줘" → 브랜드 컬러·폰트가 자동 적용된 PDF 완성
- 영상 제작자 본인의 회사(The French Bot)는 별도 "견적서 Skill"도 만들어 두어, **음성 한 줄**로 견적서가 만들어집니다. 회사 정보, 가격, 컬러가 모두 자동.

## 도구를 넘어서: 같은 .md를 다른 AI에 가져다 쓰기

영상의 보너스 팁: 만들어진 .md 파일은 **Claude만의 자산이 아닙니다.**

영상 제작자는 같은 .md를 ChatGPT(오픈AI의 AI, 이미지 생성에 강함)의 이미지 생성에 그대로 던졌고, "Grunch 프로모션 이미지 만들어줘"라는 요청에 동일한 브랜드 톤으로 이미지가 나왔습니다.

**시사점**: .md 파일 한 개가 **single source of truth(단일 진실 공급원, 모든 도구가 같은 정보를 참조하게 하는 단일 원천)**가 됩니다.

| 사용처 | 적용 방식 |
|--------|-----------|
| Claude | Skill로 등록 → 자동 호출 |
| ChatGPT | 대화 시작 시 첨부 |
| Manus(다른 AI 도구) | 프로젝트 컨텍스트로 업로드 |
| 디자이너 신입 인수인계 | 그대로 전달 |

## 실전 예시

<div class="example-case">
<strong>실전 케이스: 1인 한약사가 영양상담 콘텐츠를 통일된 톤으로</strong>

상황: 인스타 운영, 블로그 글, 환자에게 보내는 PDF 안내문, 강의 자료를 매번 다른 사람이 만든 것처럼 보임.

순서.

1. Claude Opus를 켜고 시작 프롬프트(위 예시) 입력
2. 첨부: 인스타 캡처 5장 + 로고 + 평소 환자에게 보내는 카카오톡 답변 텍스트 3개(말투 학습용)
3. Claude가 던지는 질문에 답: "주된 사용처는 환자에게 보내는 안내문 + 인스타 콘텐츠"
4. 생성된 brand-guide.md 안에 다음이 자동 포함:
   - 컬러: 파스텔 그린 + 흰색
   - 폰트: 가독성 좋은 산세리프
   - 말투: "환자에게 친근하지만 전문가다운, 약사 직함 노출"
   - 금지어: "100% 효과", "부작용 없음" 같은 광고 표현
5. "이 가이드를 Skill로 만들어줘" → 등록
6. 다음 주, "타이레놀과 이부프로펜의 차이를 환자가 받아갈 PDF로 만들어줘" 한 줄 → 같은 디자인·같은 말투의 PDF 완성

</div>

<div class="example-case">
<strong>실전 케이스: 프리랜서 마케터의 견적서 자동 생성</strong>

상황: 신규 클라이언트 문의가 올 때마다 견적서를 다시 디자인하고, 회사 정보를 다시 채워 넣음.

순서.

1. 위와 같은 방식으로 brand-guide.md 생성
2. 추가로 quote-skill.md(견적서 전용 Skill)를 별도로 만들기 — 회사명·사업자번호·계좌·표준 단가표 포함
3. 두 Skill 모두 Personalize → Skills에 등록
4. 새 문의가 오면 Claude에 음성으로 한 줄: "신규 견적서 — A사, 인스타 광고 패키지 스타터, 주소는 서울 OO구"
5. Claude가 두 Skill을 동시에 호출 → 회사 디자인 + 표준 견적 양식이 결합된 PDF가 1분 안에 완성

영상에서 시연된 것과 같은 흐름입니다. 단, 영상 결과물은 PDF를 넘어 **클릭 가능한 인터랙티브 페이지**(상담 예약 링크 포함)로 출력되었습니다.

</div>

## 주의할 점

**자료 첨부는 아끼지 말 것.** 영상 제작자의 표현: "context를 강하게 줄수록 결과가 galactique(galactic, 압도적)이 된다." 사이트 캡처 5장이면 5장, 10장이면 10장 — 많을수록 좋습니다.

**Opus 모델 사용 권장.** 매일 평생 쓸 자산이므로 가장 좋은 모델로 한 번 잘 만드는 편이 합리적. 영상에서는 Opus 4.6을 사용했고, 4.7도 괜찮다고 언급.

**개인 메모리 저장 여부 미리 답하기.** Claude가 "이 가이드를 메모리에 저장해도 되나요?"를 묻습니다. 데모용·일회성이면 "저장하지 마"라고 명시.

**Skill은 만들고 끝이 아님.** 회사 톤이 바뀌면 .md 파일을 갱신하고 Skill을 다시 등록해야 합니다. 한 번 만들면 1~2년은 가지만, 무한히 가는 자산은 아닙니다.

**말투(tone of voice) 칸을 빼먹지 말 것.** 시각 디자인만 일치해도 글이 다른 사람이 쓴 것처럼 들리면 브랜드가 깨집니다. 한국 사용자라면 "직설적·존대 위주·이모지 금지" 같은 명시적 규칙을 넣으세요.

## 정리

- 브랜드 가이드를 .md 파일 한 개로 만들고 Skill로 등록하면 PDF·이미지·견적서가 자동으로 같은 톤이 된다.
- 핵심 프롬프트 한 줄: "95% 확신 들 때까지 만들지 말고, plan부터 보여줘."
- 같은 .md는 ChatGPT·Manus 등 다른 도구에서도 그대로 재사용된다 — 단일 진실 공급원.
