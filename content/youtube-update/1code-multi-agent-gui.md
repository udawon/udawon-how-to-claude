## 이게 뭔가요?

Claude Code를 터미널에서 쓸 때 불편한 점이 있어요. 에이전트 3개를 동시에 돌리려면 터미널 창을 3개 열어야 하고, 어떤 에이전트가 뭘 하고 있는지 추적하기가 어렵죠.

**1Code**는 이 문제를 해결하는 **에이전트 관제탑**입니다.

> 비유: Claude Code가 주방장이라면, 1Code는 **주방장 여러 명을 동시에 관리하는 주방 매니저**예요. 누가 어떤 요리를 하고 있는지, 진행률이 어떤지 한눈에 보이는 대시보드가 있는 거죠.

YC W26 스타트업이 만든 오픈소스 프로젝트로, 출시 2개월 만에 GitHub 스타 5,000개를 넘겼습니다.

## 왜 알아야 하나요?

Claude Code CLI(터미널)만 쓸 때의 불편함:
- 에이전트 3개 = 터미널 3개를 열어야 함
- 에이전트들이 같은 파일을 건드리면 **충돌 발생**
- 코드 변경사항(diff)을 실시간으로 보기 어려움
- 노트북 닫으면 **작업 중단**

1Code를 쓰면:
- 하나의 앱에서 **에이전트 5개 동시 실행**
- 자동 Git 워크트리(작업 공간) 격리 → 충돌 방지
- 실시간 diff 미리보기
- **백그라운드 실행** — 노트북 닫아도 에이전트 계속 작업

## 어떻게 하나요?

### 설치하기

```bash
# 1. 소스 클론
git clone https://github.com/1code/1code.git
cd 1code

# 2. 의존성 설치
npm install

# 3. Claude Code 바이너리 다운로드 후 빌드
npm run build
```

> Pro 구독(월 $20)이면 빌드 없이 프리빌트 앱을 바로 받을 수 있습니다. Mac, Windows, Linux 모두 지원.

### 핵심 기능 4가지

**1. Git 워크트리 자동 격리**

새 세션을 시작하면 1Code가 자동으로 브랜치와 워크트리를 만들어줍니다.

<div class="example-case">
<strong>💬 예시: 에이전트 2개로 동시 작업</strong>

- 에이전트 A → `feature-a` 브랜치에서 로그인 기능 개발
- 에이전트 B → `feature-b` 브랜치에서 결제 기능 개발

메인 브랜치는 절대 건드리지 않아요. 작업 끝나면 PR(풀 리퀘스트)로 병합하면 됩니다.
</div>

**2. 대시보드 (칸반 보드)**

각 에이전트 세션이 카드로 표시되고, 상태에 따라 자동 이동:
- 대기 → 진행 중 → 리뷰 → 완료

Jira나 Linear 쓰시는 분이라면 익숙한 패턴이에요.

**3. 플랜 모드**

에이전트가 바로 코드를 짜지 않고 3단계를 거칩니다:
1. 불확실한 부분을 **질문**
2. 구조화된 **실행 계획**을 마크다운으로 보여줌
3. 개발자가 **승인해야** 실행 시작

**4. REST API로 원격 실행**

```bash
curl -X POST http://localhost:1code/api/sessions \
  -d '{"repo": "https://github.com/my/project", "prompt": "버그 수정해줘"}'
```

CI 실패 시 자동 수정 에이전트를 돌리거나, 새 이슈에 자동 코드 작성 워크플로우도 가능합니다.

## 주의할 점

- **Claude Code 바이너리 필요**: 1Code는 내부에서 Claude Code를 실행하는 구조. Claude Code가 먼저 설치되어 있어야 함
- **토큰 비용 주의**: 에이전트 5개를 동시에 돌리면 토큰 소비가 5배. 비용 관리 필요
- **OpenAI Codex도 지원**: Claude Code뿐 아니라 Codex도 한 앱에서 사용 가능
- **오픈소스(Apache 2.0)**: 무료 사용 가능, Pro 구독은 프리빌트 앱 + 클라우드 샌드박스 제공

## 정리

- 1Code = Claude Code 에이전트 여러 개를 동시에 관리하는 데스크탑 앱
- 자동 워크트리 격리로 충돌 없는 병렬 개발 가능
- 대시보드, 플랜 모드, REST API로 프로급 에이전트 관리
