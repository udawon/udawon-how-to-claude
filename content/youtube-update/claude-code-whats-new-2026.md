## 이게 뭔가요?

Anthropic(앤트로픽)이 공식 유튜브 채널에서 공개한 **Claude Code 신기능 총정리 영상**입니다.

식당 메뉴판이 6개월에 한 번씩 통째로 바뀌는 게 아니라, 매주 새 메뉴가 조용히 추가되는 상황을 상상해보세요. 눈치 빠른 단골손님은 신메뉴를 바로 알아채고 주문하지만, 대부분의 손님은 "원래부터 있던 건가?" 하고 지나칩니다. Claude Code도 비슷합니다 — 기능이 빠르게 추가되지만 모르면 쓰지 못하죠.

이 영상은 Anthropic이 직접 나서서 **"이 기간에 이런 것들이 추가됐어요"** 라고 정리해준 공식 요약본입니다. 2026년 3월부터 5월까지 출시된 핵심 기능 8종을 다룹니다.

## 왜 알아야 하나요?

**알면 좋은 점:**
- 일일이 업데이트 노트를 찾지 않아도 **한 번에 주요 신기능을 파악**할 수 있습니다
- 각 기능이 **어떤 문제를 해결**하려고 만들어졌는지 Anthropic의 의도를 직접 들을 수 있습니다
- "이런 기능이 있었어?" 싶었던 것들을 발견하고 바로 써먹을 수 있습니다

**모르면 불편한 점:**
- 이미 내 문제를 해결해주는 기능이 있는데 모르고 수동으로 처리하게 됩니다
- 특히 Auto mode, Agent view 같은 핵심 기능을 놓치면 작업 효율이 크게 차이납니다

## 2026년 주요 신기능 8종

### 1. Auto mode — 권한 프롬프트 자동 처리 (3월)

매번 "이 작업 허용하시겠어요?" 물어보는 게 피곤하셨죠? **Auto mode(자동 모드)**가 이 문제를 해결합니다.

자동 분류기가 작업을 분석해서:
- **안전한 작업** → 자동 승인 (파일 읽기, 검색 등)
- **위험한 작업** → 자동 차단 (파일 대량 삭제 등)
- **모호한 작업** → 사용자에게 확인 요청

기존의 두 극단(매번 물어보기 vs 모든 걸 허용하는 위험 모드) 사이의 **중간 지점**을 제공합니다.

| 구분 | 기존 방식 | Auto mode |
|------|----------|-----------|
| 안전한 작업 | 매번 승인 필요 | 자동 실행 |
| 위험한 작업 | 실수로 허용 가능 | 자동 차단 |
| 모호한 작업 | 매번 판단 필요 | 자동 판단 후 필요시만 확인 |

### 2. Agent view — 모든 세션을 한 화면에서 (5월)

여러 Claude Code 세션을 동시에 돌리고 있다면 **`claude agents`** 명령어 하나로 전체 현황을 볼 수 있습니다.

```
claude agents
```

화면에 표시되는 정보:
- 현재 **실행 중**인 세션
- **내 응답을 기다리는** 세션 (권한 확인 필요 등)
- **완료된** 세션

공사 현장의 관제탑처럼, 여러 일꾼(에이전트)이 무엇을 하고 있는지 한눈에 파악하는 화면입니다.

### 3. /goal — 완료 조건 지정 자동 실행 (5월)

**`/goal`** 명령어로 완료 조건을 지정하면, Claude Code가 그 조건이 충족될 때까지 여러 턴에 걸쳐 스스로 작업을 계속합니다.

```
/goal 모든 테스트가 통과할 때까지 버그를 수정해
```

이후 Claude Code가 테스트 실행 → 오류 분석 → 수정 → 재실행을 반복하고, 모든 테스트가 통과하면 자동으로 멈춥니다. 중간에 매번 "다음에 뭐 할까요?" 물어보지 않습니다.

### 4. Ultraplan — 클라우드에서 계획 세우기 (4월)

큰 작업을 시작하기 전에 **계획을 먼저 웹에서 검토**할 수 있습니다.

```
claude plan "회원가입 기능 전체 구현"
```

**3단계 흐름:**
1. CLI(명령어 창)에서 계획 초안 생성
2. 웹 에디터에서 계획을 검토·수정·코멘트
3. 원격으로 실행하거나 로컬로 가져와서 실행

건축 설계도를 먼저 검토한 뒤 공사를 시작하는 것과 같습니다. 무작정 코드부터 짜는 것보다 실수가 훨씬 줄어듭니다.

### 5. /ultrareview — 버그 탐색 에이전트 함대 (4월)

**`/ultrareview`** 는 클라우드에서 여러 에이전트가 동시에 코드를 분석해서 버그를 찾아주는 기능입니다.

```
/ultrareview
```

에이전트들이 클라우드에서 병렬로 코드 전체를 검토하고, 발견한 문제점이 자동으로 CLI나 데스크탑 앱으로 전달됩니다. 혼자 코드 리뷰하는 것보다 훨씬 빠르고 놓치는 것도 적습니다.

> 현재 리서치 프리뷰(연구 미리보기) 단계입니다.

### 6. Computer use — GUI 직접 조작 (3월)

**Computer use(컴퓨터 사용)**은 Claude Code가 실제 앱 화면을 클릭하고 조작할 수 있는 기능입니다.

지금까지는 코드와 터미널(명령어 창)만 다룰 수 있었는데, 이제 Figma(디자인 툴), 브라우저, 일반 앱처럼 **마우스로만 조작하는 프로그램**도 Claude Code가 직접 다룰 수 있습니다.

> 현재 CLI(명령어 창)에서 리서치 프리뷰 단계입니다. 데스크탑 앱에서는 별도로 지원됩니다.

### 7. Claude Opus 4.7 기본 탑재 + xhigh 에포트 (4월)

Max 플랜과 Team Premium 사용자에게 **Claude Opus 4.7**이 기본 모델로 적용됩니다.

함께 추가된 **`xhigh` 에포트(effort) 레벨**은 복잡한 코딩 작업에 권장하는 설정으로, Claude가 더 많은 시간과 자원을 투입해서 품질 높은 결과물을 냅니다. `/effort` 슬라이더로 직접 조절할 수 있습니다.

### 8. Windows 지원 개선 — Git Bash 없이도 작동 (4월)

기존에는 Windows에서 Claude Code를 사용하려면 **Git Bash**(리눅스 명령어 환경)가 필수였습니다. 이제는 Git Bash 없이도 **PowerShell**(윈도우 기본 명령어 창)로 바로 사용할 수 있습니다.

Windows 사용자에게는 설치 과정이 크게 단순해진 업데이트입니다.

## 그 밖의 업데이트

| 기능 | 설명 |
|------|------|
| **Routines(루틴)** | 예약된 시간이나 GitHub 이벤트에 클라우드 에이전트 자동 실행 |
| **모바일 푸시 알림** | 오래 걸리는 작업이 끝나거나 확인이 필요할 때 폰에 알림 |
| **Monitor 도구** | 백그라운드 이벤트를 실시간으로 대화창에 스트리밍 |
| **세션 리캡** | 화면을 안 보다가 돌아왔을 때 그동안 있었던 일 요약 표시 |
| **커스텀 테마** | `/theme`으로 색상 테마 직접 제작 및 플러그인으로 배포 |
| **fast mode → Opus 4.7** | Fast mode(빠른 모드)가 기존 Opus 4.6에서 Opus 4.7로 업그레이드 |
| **`/loop` 자동 페이싱** | 반복 간격을 지정하지 않으면 Claude가 상황에 맞게 간격 자동 조절 |
| **`claude project purge`** | 프로젝트의 로컬 상태 데이터 일괄 삭제 정리 |

## 어떻게 시작하나요?

**Auto mode 켜기:**
```
/auto
```
또는 설정에서 Auto mode를 활성화합니다.

**Agent view 열기:**
```
claude agents
```

**목표 지정 실행:**
```
/goal [완료 조건을 한 문장으로]
```

**코드 리뷰 요청:**
```
/ultrareview
```

## 주의할 점

- **Auto mode** — 2026년 3월 리서치 프리뷰로 출시된 후, Max 구독자에게는 별도 플래그 없이 정식 사용 가능한 상태로 전환됐습니다. 분류기가 완벽하지 않을 수 있으므로 중요한 작업에서는 결과를 확인하는 것이 좋습니다
- **Computer use도 리서치 프리뷰** — 아직 모든 앱이나 상황에서 안정적이지 않을 수 있습니다
- **Ultraplan과 Routines은 클라우드 기능** — 인터넷 연결 없이는 사용 불가합니다
- **Opus 4.7 기본 탑재는 Max/Team Premium 전용** — 일반 플랜은 해당되지 않을 수 있습니다

## 정리

- **Auto mode**: 권한 프롬프트를 자동 분류·처리해서 불필요한 확인 질문을 줄여줌
- **Agent view**: `claude agents` 명령어로 모든 세션 현황을 한 화면에서 관리
- **Ultraplan**: 클라우드 웹 에디터에서 계획을 검토 후 실행하는 3단계 워크플로우
- **/ultrareview**: 클라우드 에이전트 함대가 코드 버그를 병렬 탐색
- **Computer use**: GUI 앱(Figma, 브라우저 등)을 Claude Code가 직접 클릭·조작
- **Windows 개선**: Git Bash 없이 PowerShell만으로 Claude Code 사용 가능
