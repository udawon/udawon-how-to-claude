## 케이스 A: 아이디어만 있을 때

> 코드를 잘 몰라도 됩니다. Claude가 질문하고, 당신은 답하면 됩니다.

### 1단계: 폴더 만들고 Claude 실행

```bash
# 터미널(명령 프롬프트)에서
mkdir cafe-booking
cd cafe-booking
claude
```

```
비유: 빈 공책을 펼쳐놓는 것
     아직 아무것도 없어도 됨. 시작이 반
```

---

### 2단계: Claude에게 인터뷰 받기 (핵심!)

바로 "만들어줘"라고 하지 마세요. 먼저 Claude가 당신을 인터뷰하게 만드세요.

```
"카페 예약 사이트를 만들고 싶어.
 AskUserQuestion 도구로 나를 인터뷰해줘.
 내가 미처 생각 못한 부분까지 파고들어줘.
 다 끝나면 SPEC.md에 정리해줘."
```

**Claude가 물어볼 것들 (예시):**
```
Claude: 예약을 받을 카페가 한 곳인가요, 여러 곳인가요?
나: 일단 제 카페 한 곳이요

Claude: 예약 취소/변경은 손님이 직접 할 수 있어야 하나요?
나: 아, 그것도 필요하겠네요

Claude: 예약 확인을 문자나 카카오톡으로 보내야 하나요?
나: 카카오톡 알림이면 좋겠어요

Claude: 테이블 수와 시간대별 예약 가능 인원은 어떻게 되나요?
나: 테이블 5개, 각 2시간 단위로요
```

```
인터뷰 없이 바로 시작하면:
"카페 예약 사이트 만들어줘"
-> Claude가 만들고 나서 "아, 카카오톡 알림도 필요한데"
-> 중간에 방향이 바뀌어 시간 낭비

인터뷰 후 시작하면:
모든 요구사항을 미리 정리 → 처음부터 제대로 구현
```

---

### 3단계: 환경 설정 (CLAUDE.md)

새 세션을 시작하고 프로젝트 규칙을 잡습니다.

```
(새 세션에서)
"@SPEC.md 를 읽고 이 프로젝트의 CLAUDE.md를 만들어줘.
 Next.js + Tailwind CSS로 만들 거야."
```

Claude가 자동으로 만들어주는 것:
```markdown
# CLAUDE.md (자동 생성 예시)
## 프로젝트
카페 예약 사이트 (단일 카페, 테이블 5개, 2시간 단위)

## 기술 스택
- Next.js 14 App Router
- Tailwind CSS
- SQLite (간단한 DB)

## 규칙
- 한국어로 응답
- 모바일 우선 디자인
```

---

### 4단계: 구조 설계 (Plan Mode)

코드를 짜기 전에 전체 구조를 합의합니다.

```
Shift+Tab으로 Plan Mode 진입 (화면 아래 "⏸ plan mode on" 확인)

"SPEC.md를 기반으로 이 프로젝트의 전체 구조를 설계해줘.
 어떤 페이지가 필요하고, 어떤 순서로 만들면 좋을지."
```

Claude의 플랜 예시:
```
1. 메인 페이지 (예약 폼)
2. 예약 현황 페이지 (관리자용)
3. 예약 확인 페이지 (손님용)
4. 카카오톡 알림 연동
```

```
Ctrl+G 로 플랜을 에디터에서 직접 수정 가능
마음에 안 드는 순서나 항목이 있으면 직접 편집
```

Shift+Tab으로 Normal Mode 복귀 후 구현 시작.

---

### 5단계: 세션 이름 붙이기

```
/rename cafe-booking-main
```

```
이름을 붙여두면:
내일 이어서 작업할 때 → claude --resume cafe-booking-main
PR 만들었을 때 → claude --from-pr 123
```

---

### 6단계: 하나씩 구현하기

한 번에 다 만들려 하지 마세요. 하나씩, 확인하면서 진행합니다.

```
"메인 페이지의 예약 폼부터 만들어줘.
 날짜, 시간, 인원, 이름, 연락처 입력 필드가 필요해."
```

**잘못된 방향이면 즉시 Esc:**
```
Claude가 너무 복잡하게 만들기 시작하면?
-> Esc 눌러서 중단
-> "더 단순하게, 디자인 신경 쓰지 말고 기능만 먼저"
```

**실험하고 싶으면 마음껏:**
```
체크포인트가 자동으로 저장되니까
"한 번 예약 완료 애니메이션 넣어봐" 시도해보고
마음에 안 들면 Esc + Esc → Rewind로 되돌리기
```

---

### 7단계: 검증하기

```
(새 세션 열기 - Reviewer 세션)

"@src/ 폴더 전체를 봐줘.
 처음 보는 코드라고 생각하고 냉정하게 리뷰해줘.
 예약이 동시에 들어오면 어떻게 되는지,
 잘못된 날짜를 입력하면 어떻게 되는지 확인해줘."
```

```
같은 세션에서 리뷰하면 관대해짐
새 세션 = 새로운 눈 = 냉정한 리뷰
```

---

### 8단계: Git 커밋 & 마무리

```
"변경사항을 커밋해줘. 커밋 메시지는 한국어로."
```

```
Claude가 자동으로:
- 변경 파일 확인
- 의미있는 커밋 메시지 작성
- git add + git commit 실행
```

---

### 전체 흐름 한눈에 보기

```
[터미널] 폴더 생성 + claude 실행
    ↓
[Claude] 인터뷰 → SPEC.md 작성
    ↓
[새 세션] CLAUDE.md 생성
    ↓
[Plan Mode] 구조 설계 + Ctrl+G로 수정
    ↓
[Normal Mode] /rename으로 세션 이름 저장
    ↓
[구현] 하나씩 + 체크포인트 활용
    ↓
[새 세션] 냉정한 코드 리뷰
    ↓
[커밋] 마무리
```

## 케이스 B: 기술 스택을 정한 개발자

> "Next.js + Supabase로 카페 예약 사이트 만들 거야" 처럼 방향이 정해진 경우.

### 1단계: 프로젝트 초기화

```bash
mkdir cafe-booking && cd cafe-booking
npx create-next-app@latest . --typescript --tailwind --app
claude
```

---

### 2단계: /init으로 CLAUDE.md 자동 생성

```
/init
```

Claude가 프로젝트를 분석해서 CLAUDE.md 초안을 만들어줍니다. 여기에 추가로:

```
"CLAUDE.md에 이것도 추가해줘:
 - Supabase를 DB로 사용
 - 한국어로 응답
 - 컴포넌트는 src/components/ 에 분리"
```

---

### 3단계: Plan Mode로 빠르게 설계

```
Shift+Tab → Plan Mode

"카페 예약 시스템 설계해줘.
 Supabase 테이블 구조, API 라우트, 컴포넌트 구조 포함.
 @SPEC.md 있으면 참고해."
```

마음에 들면 Shift+Tab으로 Normal Mode 전환 후 구현.

---

### 4단계: 기능별 Worktree로 병렬 개발

여러 기능을 동시에 개발해야 한다면:

```bash
# 터미널 1: 예약 UI 개발
claude --worktree feature-booking-ui

# 터미널 2: 관리자 페이지 개발
claude --worktree feature-admin-panel
```

```
두 작업이 완전히 분리된 환경에서 진행
서로 파일 충돌 없음
```

---

### 5단계: 복잡한 부분은 ultrathink

```
"ultrathink - 동시에 같은 시간대에 예약이 들어오면
 어떻게 처리해야 하지? Supabase에서 트랜잭션 처리 방법과
 낙관적 락 vs 비관적 락 트레이드오프 분석해줘."
```

---

### 전체 흐름 한눈에 보기

```
[터미널] create-next-app + claude
    ↓
[/init] CLAUDE.md 자동 생성 + 보완
    ↓
[Plan Mode] 빠른 설계
    ↓
[Worktree] 기능별 병렬 개발 (필요시)
    ↓
[ultrathink] 복잡한 부분 깊이 분석
    ↓
[커밋 + PR]
```

## 케이스 C: 기존 프로젝트 합류

> "이 프로젝트 인수인계받았는데 뭐가 뭔지 모르겠어" 상황.

### 1단계: 전체 파악부터

```bash
cd 기존-프로젝트-폴더
claude
```

```
"이 프로젝트 전체 구조를 파악해줘.
 어떤 기술을 쓰고, 주요 기능이 뭔지, 폴더 구조는 어떻게 됐는지
 신입 개발자에게 설명하듯이 알려줘."
```

---

### 2단계: 핵심 코드 흐름 추적

```
"사용자가 예약을 하면 어떤 파일들이 실행되는지
 처음부터 끝까지 추적해줘."

"인증은 어떻게 처리되고 있어?"

"DB 스키마가 어떻게 생겼어? ERD로 설명해줘."
```

```
비유: 새 직장 첫날
     "우리 팀이 뭘 하는 팀이에요?" 먼저 물어보는 것
     코드부터 보면 길을 잃음
```

---

### 3단계: 서브에이전트로 병렬 탐색

큰 프로젝트는 서브에이전트를 활용해 빠르게 파악합니다.

```
"서브에이전트들을 써서 병렬로 탐색해줘:
 - 에이전트 1: 프론트엔드 구조와 주요 컴포넌트
 - 에이전트 2: 백엔드 API 구조와 DB 스키마
 - 에이전트 3: 최근 3개월 git 로그에서 주요 변경사항

 각자 탐색 후 종합 보고서 만들어줘."
```

```
서브에이전트가 탐색하는 동안
메인 컨텍스트가 오염되지 않음
요약본만 돌아옴 = 효율적
```

---

### 4단계: CLAUDE.md 생성 (파악 후)

```
"이 프로젝트의 주요 특성, 기술 스택, 주의사항을
 CLAUDE.md로 정리해줘.
 내가 나중에 이 프로젝트 작업할 때 Claude가
 놓치면 안 되는 것들 위주로."
```

---

### 5단계: 작은 것부터 시작

처음부터 큰 기능을 건드리지 마세요.

```
"기존 코드 스타일과 패턴을 파악했으니,
 이 작은 버그부터 고쳐줘: [버그 설명]
 기존 패턴을 최대한 따라줘."
```

---

### 전체 흐름 한눈에 보기

```
[전체 파악] "이 프로젝트 설명해줘"
    ↓
[핵심 흐름 추적] 주요 기능 동작 방식 이해
    ↓
[서브에이전트] 병렬 탐색으로 빠른 파악
    ↓
[CLAUDE.md] 파악한 내용 정리
    ↓
[작은 것부터] 기존 패턴 따라 시작
```

## 공통: 자주 쓰는 패턴 모음

### 막혔을 때
```
"어디서 에러가 나는지 모르겠어. 로그를 보여줄게: [에러 복붙]"
"ultrathink로 이 버그 원인을 체계적으로 추적해줘"
```

### 방향을 바꾸고 싶을 때
```
Esc → 중단 (컨텍스트 유지)
Esc + Esc → Rewind 메뉴 (이전 상태로 복구)
/clear → 컨텍스트 초기화 (완전히 새로 시작)
```

### 며칠에 걸친 작업
```
/rename 의미있는이름     (세션에 이름 붙이기)
claude --resume 이름     (다음날 이어서)
```

### 중요한 기능 전 안전망
```
"구현 전에 git commit 해줘"   (체크포인트)
또는
Esc + Esc → Rewind 활용      (Claude 자동 체크포인트)
```
