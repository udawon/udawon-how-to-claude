> 이 문서는 실제 있었던 일입니다.
> Claude와 브레인스토밍을 했는데, 결과물이 텍스트 파일로만 남았습니다.
> "이걸 눈으로 보고 싶다"는 생각 하나로 시작해서, 마인드맵 뷰어 웹앱이 됐습니다.

---

## 시작 계기

Claude와 팀 협업 도구 아이디어를 정리하는 브레인스토밍을 했습니다.
대화가 끝나고 `SPEC.md`, `PRD.md`, `TASKS.md` 세 문서와 `mindmap.json` 파일이 생겼습니다.

텍스트 파일을 열어봤습니다. 내용은 잘 정리됐는데, 한눈에 보기가 어려웠습니다.
아이디어들이 어떻게 연결되는지, 구조가 어떻게 생겼는지 — 시각적으로 보고 싶었습니다.

"이 마인드맵 파일을 브라우저에서 보고 싶어.
 React랑 Vite로 만들어줘."

이렇게 시작했습니다.

---

## 1단계: 폴더 만들고 Claude 실행

```bash
mkdir udawon-cyclist
cd udawon-cyclist
claude
```

기술 스택을 React + Vite로 지정한 이유:
- React: 이름 정도는 들어봤고, 웹앱에 많이 쓴다고 알고 있었음
- Vite: 빠르다고 들었음

마인드맵을 어떻게 그리는지, TypeScript가 필요한지, 상태 관리는 어떻게 하는지 — 이런 건 전혀 몰랐고, 알 필요도 없었습니다. Claude가 결정했습니다.

---

## 2단계: Claude에게 인터뷰 받기

바로 "만들어줘"라고 하지 않았습니다.
요구사항을 제대로 정리하기 위해 Claude가 먼저 물어보게 했습니다.

"마인드맵 뷰어를 만들고 싶어.
 AskUserQuestion 도구로 나를 <span class="keyword-highlight">인터뷰해줘</span>.
 내가 생각 못한 부분까지 파고들어줘."

**Claude가 실제로 물어본 것들:**

<div class="example-case">

Claude: 마인드맵 노드를 클릭하면 어떻게 됐으면 해요?
나: 노드 내용을 옆에 패널로 보여줬으면 해요

Claude: 마인드맵이 하나인가요, 여러 프로젝트를 관리할 건가요?
나: 지금은 하나인데, 나중에 여러 개 관리하고 싶어요

Claude: 다크모드가 필요한가요?
나: 있으면 좋겠어요

Claude: 모바일에서도 봐야 하나요?
나: 주로 PC지만 반응형으로 만들어주세요

Claude: 마인드맵 외에 문서(SPEC, PRD)도 뷰어에서 보고 싶으신가요?
나: 아, 그것도 같이 보면 좋겠다
</div>

마지막 질문이 중요했습니다. 생각해보니 SPEC.md와 PRD.md도 같이 보고 싶었는데, 혼자였으면 처음에 못 챙겼을 내용이었습니다.

---

## 3단계: CLAUDE.md 설정

인터뷰가 끝나고 새 세션을 열어 프로젝트 규칙을 잡았습니다.

(새 세션에서)
"방금 정한 요구사항으로 이 프로젝트의 <span class="keyword-highlight">CLAUDE.md를 만들어줘</span>.
 React + Vite + TypeScript로 만들 거야."

Claude가 만들어준 CLAUDE.md 주요 내용:
```markdown
## 프로젝트
마인드맵 뷰어 + 문서 뷰어 포트폴리오 사이트

## 기술 스택
- React + Vite (TypeScript)
- React Flow: 마인드맵 시각화
- React Router: 페이지 라우팅
- Zustand: 상태 관리
- Tailwind CSS: 스타일링

## 규칙
- 한국어로 응답
- 반응형 필수 (모바일 사이드바 토글)
- 다크모드 지원
```

React Flow가 뭔지 몰랐습니다. Claude가 마인드맵 시각화에 가장 적합한 라이브러리를 선택해줬습니다.

## 4단계: Plan Mode로 전체 구조 설계

코드를 짜기 전에 어떤 화면이 필요한지, 어떤 순서로 만들지 먼저 합의했습니다.

<kbd>Shift</kbd>+<kbd>Tab</kbd>을 눌러 권한 모드를 전환 (기본 → 자동수락 → Plan Mode 순환)

"이 프로젝트의 전체 구조를 설계해줘.
 어떤 페이지가 필요하고, 파일 구조는 어떻게 잡으면 좋을지."

Claude의 플랜:
```
레이아웃
├── Header (다크모드 토글, 프로젝트명)
├── Sidebar (프로젝트 목록, 문서 탭)
└── 메인 영역
    ├── WelcomePage (처음 화면)
    ├── MindmapPage (React Flow 시각화)
    └── MarkdownPage (SPEC/PRD/TASKS 문서)

라우팅: /:slug/mindmap, /:slug/spec, /:slug/prd, /:slug/tasks
```

플랜을 보고 "TASKS 문서도 탭에 추가해줘"라고 수정했습니다.
<kbd>Ctrl</kbd>+<kbd>G</kbd>로 플랜을 직접 편집하고 <kbd>Shift</kbd>+<kbd>Tab</kbd>으로 구현 모드로 전환했습니다.

## 5단계: 핵심 기능 구현 — 마인드맵

가장 핵심인 마인드맵부터 만들었습니다.

"mindmap.json을 읽어서 React Flow로 마인드맵을 그려줘.
 노드는 카테고리별로 색깔이 다르게, 그룹으로 묶어서 보여줘."

Claude가 만든 것:
- `MindmapCanvas.tsx`: React Flow 기반 마인드맵
- `groupLayout.ts`: 노드를 카테고리 그룹으로 자동 배치하는 알고리즘
- `MindmapNode.tsx`: 커스텀 노드 디자인
- `DetailPanel.tsx`: 노드 클릭 시 나타나는 상세 패널

**여기서 처음 마인드맵이 화면에 떴습니다.**

텍스트 파일로만 보던 브레인스토밍 결과가, 노드와 연결선으로 화면 가득 펼쳐졌습니다.
노드를 클릭하면 오른쪽에 상세 내용이 나타났습니다.

비유: 머릿속에만 있던 지도가
     처음으로 종이 위에 그려진 순간

---

## 6단계: 확장 — 포트폴리오 사이트로

마인드맵 하나만 보는 건 아쉬웠습니다. 프로젝트가 쌓일 테니까요.

"지금은 sticky-office 프로젝트 하나만 있는데,
 나중에 프로젝트가 여러 개가 될 거야.
 사이드바에서 프로젝트를 선택하고, 각 프로젝트의 마인드맵/문서를 볼 수 있게 해줘."

Claude가 한 번에 만든 것들:
- `projects/index.json`: 프로젝트 목록 관리
- `useProjectStore.ts`: Zustand로 프로젝트 상태 관리
- `Sidebar.tsx`: 프로젝트 선택 + 문서 탭 네비게이션
- 반응형 처리: 모바일에서는 사이드바 토글, 데스크탑에서는 항상 표시

결과:

```
sidebar
├── 우리 팀의 업무 프레임워크  ← 클릭 시 선택
│   ├── Mindmap
│   ├── SPEC
│   ├── PRD
│   └── TASKS
└── (다음 프로젝트 추가 예정)
```

## 7단계: 다크모드와 마무리 디테일

"다크모드 토글을 헤더에 넣어줘.
 새로고침해도 설정이 유지됐으면 해."

"모바일에서 마인드맵이 너무 작아 보여.
 처음 뜰 때 줌 레벨을 조정해줘."

"마인드맵에 노드가 36개인데 처음에 너무 축소돼서 글씨가 안 보여.
 최소 줌 레벨을 제한해줘."

이런 식으로 쓰면서 발견한 문제를 하나씩 말했고, Claude가 수정했습니다.

## 완성된 것

| 기능 | 내용 |
|------|------|
| 마인드맵 시각화 | React Flow, 카테고리별 그룹 + 색상, 줌/패닝 |
| 노드 상세보기 | 클릭 시 오른쪽 패널에 내용 표시 |
| 문서 뷰어 | SPEC / PRD / TASKS 마크다운 렌더링 |
| 프로젝트 네비게이션 | 사이드바에서 여러 프로젝트 관리 |
| 다크모드 | Zustand + localStorage로 설정 유지 |
| 반응형 | 모바일 사이드바 토글, 데스크탑 상시 표시 |

## 전체 흐름 한눈에 보기

```
[아이디어] "브레인스토밍 결과를 눈으로 보고 싶다"
    ↓
[폴더 생성] mkdir + claude 실행
    ↓
[인터뷰] Claude가 요구사항 파고들기 → 마크다운 뷰어도 추가
    ↓
[새 세션] CLAUDE.md 생성 (React Flow 등 Claude가 기술 선택)
    ↓
[Plan Mode] 전체 구조 합의 → Ctrl+G로 수정
    ↓
[구현 ①] 마인드맵 핵심 기능 → 처음 화면에 뜨는 순간
    ↓
[구현 ②] 포트폴리오 확장 → 멀티 프로젝트 구조
    ↓
[구현 ③] 다크모드, 반응형, 디테일 개선
    ↓
[커밋] 마무리
```

## 이 프로젝트에서 배운 것

**인터뷰가 범위를 잡아준다**

"마인드맵 뷰어 만들어줘"라고만 했으면 단순한 뷰어가 나왔을 겁니다.
Claude가 "문서(SPEC, PRD)도 보고 싶으세요?"라고 물었고,
덕분에 처음부터 포트폴리오 사이트 방향으로 설계됐습니다.

**모르는 기술은 Claude가 고른다**

React Flow, Zustand, react-router — 이름도 몰랐습니다.
"React + Vite로 만들어줘"라고만 했고, 나머지는 Claude가 상황에 맞게 선택했습니다.

**쓰면서 발견한 문제를 바로 말한다**

"마인드맵 처음에 너무 작아 보여", "글씨가 안 읽혀" — 이런 피드백을 그때그때 말했습니다.
완벽하게 기획하고 시작한 게 아니라, 만들면서 보이는 문제를 고치면서 완성됐습니다.
