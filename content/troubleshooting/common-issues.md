## 컨텍스트 초과 (이전 내용을 잊어버림)

**증상**: 대화가 길어지면 이전에 합의한 내용이나 수정한 파일을 잊어버림

**해결**:
- Claude Code 대화창에서 `/compact` 입력하여 대화 압축
- CLAUDE.md에 중요한 내용 기록 (대화창에서 "CLAUDE.md에 이 내용 추가해줘"라고 요청)
- 작업 단위를 작게 나누기

**예시 케이스:**
```
문제 상황:
  30분 동안 대화하면서 여러 파일을 수정함
  -> Claude가 처음에 합의한 디자인 스타일을 잊어버리고 다른 스타일로 만듦

해결:
  1. 중요한 합의 사항을 CLAUDE.md에 적어두기:
     "디자인 스타일: 둥근 모서리, 파란색 계열"
  2. /compact 입력
  3. "CLAUDE.md 읽고 이어서 작업해줘"
  -> 파일에 기록되어 있으니 잊어버려도 다시 읽을 수 있음

비유: 긴 회의에서 중요한 결정을 화이트보드에 적어두는 것.
     기억에 의존하면 나중에 "우리 뭐라고 했지?" 하게 됨
```

## 도구 권한 거부 (매번 승인 팝업)

**증상**: 파일 수정할 때마다 "허용할까요?" 확인이 계속 뜸

**해결**:
- 권한 모드 설정 조정
- 안전한 도구는 자동 허용 설정

**예시 케이스:**
```
문제 상황:
  "제목 바꿔줘"라고 했는데 "파일 수정을 허용할까요?" 팝업
  "색상 바꿔줘"라고 했는데 또 팝업
  매번 "허용"을 눌러야 해서 번거로움

해결:
  파일 읽기는 자동 허용 (읽는 건 안전하니까)
  파일 수정은 필요에 따라 자동 허용 또는 수동 확인

비유: 아파트 출입문에 비밀번호가 있는데,
     거주자는 자동으로 열리게 해두는 것.
     외부인만 확인하면 됨.
```

## 잘못된 파일 수정 (엉뚱한 곳을 고침)

**증상**: 의도하지 않은 파일이나 코드를 수정함

**해결**:
- 작업 범위를 명확히 지정
- 수정 후 변경사항 확인

**예시 케이스:**
```
문제 상황:
  "버튼 스타일 수정해줘"라고 했더니
  버튼 외에 다른 요소들까지 같이 수정함

예방법:
  "메인 페이지의 '로그인' 버튼만 수정해줘.
   다른 파일은 건드리지 마."

수정 후 확인법:
  "방금 뭘 바꿨는지 알려줘"
  -> Claude가 변경 목록을 보여줌

복구법:
  "방금 수정한 거 되돌려줘"
  -> Claude가 원래대로 복구

비유: "거실 벽지 바꿔주세요"라고 했는데 주방 벽지까지 바꾼 경우
     -> "거실 벽지만 바꿔주세요. 주방은 그대로 두세요."라고 명확히
```

## 오버엔지니어링 (단순한 요청에 과도한 작업)

**증상**: 간단한 요청에 너무 복잡한 구조를 만듦

**해결**:
- "최소한의 변경으로" 키워드 사용
- "기존 패턴을 따라서" 키워드 사용

**예시 케이스:**
```
문제 상황:
  "글씨 색상 바꿔줘"라고 했더니
  테마 시스템을 새로 만들고, 설정 파일을 추가하고...
  단순한 작업이 대공사가 됨

예방법:
  "메인 페이지 제목 색상을 파란색으로 바꿔줘.
   최소한의 변경으로, 다른 건 건드리지 마."

비유: "문고리 좀 고쳐주세요"라고 했는데
     현관문 전체를 교체하려는 인테리어 업체
     -> "문고리만 고쳐주세요. 다른 건 손대지 마세요."
```

## Claude가 같은 실수를 반복

**증상**: 같은 유형의 실수를 매 세션마다 반복함

**해결**:
- CLAUDE.md에 규칙으로 등록

**예시 케이스:**
```
문제 상황:
  Claude가 매번 영어로 주석을 쓰거나,
  매번 불필요한 파일을 추가로 생성함

해결:
  CLAUDE.md에 적어두기:
  "## 규칙
   - 주석은 반드시 한국어로 작성할 것
   - 새 파일은 꼭 필요한 경우에만 생성할 것"

비유: 실수가 반복되면 매번 주의를 주는 대신
     사무실 벽에 주의사항을 붙여두는 것.
     들어올 때마다 자동으로 보게 됨.
```
