## 기본 흐름

1. **프로젝트 폴더 만들기** 후 Claude Code 실행
2. **CLAUDE.md 작성** - 규칙과 선호사항 정의
3. **만들고 싶은 것 설명** - 자연어로 설명
4. **하나씩 구현** - 한 번에 하나의 기능씩

**예시 케이스 (처음부터 따라하기):**
```
[1단계] 일반 터미널(명령 프롬프트)을 열고:
mkdir my-blog        <- 'my-blog'이라는 폴더를 만든다
cd my-blog           <- 그 폴더 안으로 들어간다
claude               <- Claude Code를 실행한다

[2단계] Claude Code 대화창이 열리면, 여기서부터는 자연어로 대화:
"이 프로젝트는 개인 블로그야.
 CLAUDE.md 먼저 만들어줘."
-> Claude가 설정 파일을 자동으로 생성해줌

[3단계] 이어서 Claude Code 대화창에서:
"블로그를 만들 건데, 어떤 구조가 좋을지 제안해줘.
 아직 만들지는 마."
-> Claude가 폴더 구조, 페이지 구성 등을 제안

[4단계] 제안이 마음에 들면:
"좋아, 이 구조로 시작해줘."
-> Claude가 기본 틀을 생성

[5단계] 하나씩 구현:
"메인 페이지부터 만들자."
-> 확인 후 다음 기능으로

비유: 집을 지을 때
     1. 땅 준비 (폴더 만들기)
     2. 규칙 정하기 (CLAUDE.md)
     3. 설계도 먼저 보기 (구조 제안)
     4. 기초공사부터 시작 (하나씩 구현)
```

## 효과적인 첫 프롬프트

**나쁜 예시:**
```
"블로그 만들어줘. 메인, 글 목록, 글 쓰기, 댓글,
 좋아요, 공유, 태그, 검색, 다크모드 전부 구현해줘."

-> 너무 많은 걸 한꺼번에 요청
   중간에 기억을 잃어 뒤쪽 기능이 부실해짐

비유: 이사 당일에 "가구 조립, 벽지, 청소, 인터넷 설치 다 해주세요"
     한꺼번에 시키면 다 부실해짐
```

**좋은 예시:**
```
"개인 블로그를 만들려고 해.
 주요 기능은 글 쓰기와 글 목록이야.
 어떤 구조가 좋을지 먼저 제안해줘."

-> 구조를 먼저 합의한 뒤 하나씩 구현
   매 단계마다 확인하면서 진행

비유: "먼저 큰 가구부터 배치하고, 다음에 소품을 정리하자"
     순서대로 진행하면 결과가 깔끔
```

## 프로젝트 시작 패턴

### 패턴 1: 설계 먼저, 구현 나중에
```
1차: "이 프로젝트의 폴더 구조를 제안해줘"
     -> 구조 확인

2차: "좋아, 이 구조로 시작해줘"
     -> 기본 틀 생성

3차: "메인 화면 레이아웃을 만들어줘"
     -> 하나씩 채워나가기

비유: 옷장 정리할 때
     먼저 칸 나누기 -> 큰 옷부터 -> 작은 옷 -> 소품
```

### 패턴 2: 참고 사이트가 있을 때
```
"이 프로젝트를 시작할 건데,
 이 사이트(참고 URL)랑 비슷한 구조로 만들고 싶어.
 먼저 구조를 분석해줘."

-> 레퍼런스가 있으면 방향 설정이 훨씬 빠름

비유: 미용실에서 "알아서 해주세요" vs "이 사진처럼 해주세요"
     참고 이미지가 있으면 결과가 정확
```

### 패턴 3: 기존 프로젝트에 합류했을 때
```
"이 프로젝트의 전체 구조를 파악하고,
 어떤 기술을 사용하고 있는지 정리해줘."

-> 기존 프로젝트를 빠르게 파악

비유: 새 직장에서 첫 날 "이 팀이 뭘 하는 팀인지"
     먼저 파악하는 것
```
