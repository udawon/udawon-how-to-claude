## 체크포인트란?

Claude Code는 **모든 파일 변경 전에 자동으로 상태를 저장**합니다. 덕분에 어떤 변경이든 이전 상태로 되돌릴 수 있습니다.

```
중요: 이것은 git과 별개입니다!
git commit 여부와 상관없이 Claude가 변경할 때마다 자동 저장됨

비유: 게임의 "자동 저장"
     세이브 포인트를 직접 만들지 않아도
     게임이 자동으로 중요한 순간마다 저장해줌
```

---

## Rewind 사용법

### 열기

```
방법 1: Esc 두 번 빠르게 누르기
방법 2: /rewind 명령어 입력

-> Rewind 메뉴가 열림
```

### Rewind 메뉴 선택지

```
1. 대화만 되돌리기
   - 파일은 그대로, 대화 내용만 이전 시점으로
   - 언제: Claude가 잘못 이해한 것 같을 때

2. 코드만 되돌리기
   - 대화는 그대로, 파일 변경만 취소
   - 언제: 변경 내용은 마음에 안 들지만 대화 맥락은 유지하고 싶을 때

3. 대화 + 코드 모두 되돌리기
   - 완전히 해당 시점으로 복구
   - 언제: 완전히 다른 방향으로 다시 시작하고 싶을 때

4. 해당 시점부터 요약
   - 이 시점 이후의 대화를 요약해서 새로운 출발점 만들기
   - 언제: 컨텍스트가 너무 길어졌을 때 정리
```

## 실전 활용 예시

### 예시 1: 위험한 리팩토링 시도

```
상황: CSS 전체 구조를 바꿔야 하는데 잘 될지 모르겠음

일반적인 접근:
나: (변경하기 전에 git commit으로 백업을...)
나: (아, 귀찮은데...)
나: (그냥 해볼까... 망가지면 어떡하지...)

체크포인트 활용:
나: "Tailwind 클래스 방식을 CSS 모듈 방식으로 전환해줘"
Claude: (전환 시작)
나: (잠깐, 이거 훨씬 복잡해지는데?)

-> Esc + Esc -> "코드 + 대화 모두 되돌리기"
-> 변환 전 완벽한 상태로 복구!

나: "전환하기 전에 어떤 방식이 우리 프로젝트에 더 맞는지
     분석만 해줘 (수정하지 말고)"

비유: 헤어스타일 바꾸기 전에 "가상 피팅" 해보는 것
     마음에 안 들면 취소 가능
```

### 예시 2: 여러 접근법 비교

```
상황: 성능 최적화를 두 가지 방법으로 비교해보고 싶음

방법 A 시도:
나: "메모이제이션으로 최적화해줘"
Claude: (구현 완료)
나: (성능 측정 -> 30% 향상)

-> Esc + Esc -> "코드만 되돌리기" (A 방법의 코드만 취소)

방법 B 시도:
나: "이번엔 가상화(virtualization)로 최적화해줘"
Claude: (구현 완료)
나: (성능 측정 -> 50% 향상)

-> B 방법이 더 좋으니 유지!

비유: 요리할 때 두 가지 레시피를 차례로 시험해보기
     첫 번째가 맛없으면 되돌리고 두 번째 시도
```

### 예시 3: 컨텍스트 정리

```
상황: 작업이 길어져 대화가 너무 길어짐

-> Esc + Esc
-> "해당 시점부터 요약" 선택
-> 초기 시점 선택

결과:
초기 시점까지의 중요한 내용이 요약되어
깔끔한 새 출발점이 만들어짐

비유: 긴 회의록을 "핵심 결정사항" 으로 요약하는 것
     전체를 다시 읽지 않아도 핵심만 파악 가능
```

## 체크포인트 주의사항

```
가능한 것:
- Claude가 수정한 모든 파일을 되돌리기
- 세션을 닫아도 체크포인트는 유지됨
- 나중에 세션을 재개해도 rewind 가능

불가능한 것:
- Claude가 아닌 내가 직접 수정한 파일 (추적 안 됨)
- 외부 프로세스가 변경한 것 (npm install, git 작업 등)
- git commit/push는 별도로 해야 함

핵심 원칙:
"체크포인트는 Claude 작업의 안전망이지
 git을 대체하는 것이 아님"

권장 사용법:
- 실험적인 변경: 체크포인트로 되돌리기
- 확정된 변경: git commit으로 영구 저장
```

## Rewind를 활용한 개발 철학

```
체크포인트가 있으니:

(X) "이거 망가질 수도 있으니 신중하게 접근하자"
(O) "일단 해보고, 안 되면 되돌리자"

실험을 두려워하지 않는 개발 방식:
1. 아이디어 실행
2. 결과 확인
3. 좋으면 유지, 나쁘면 Rewind 후 다른 방법 시도

비유: 그림 그릴 때 연필로 스케치하는 것
     마음에 안 들면 지우개로 지우면 됨
     처음부터 완벽하게 그리려는 부담 없음
```

## /compact로 컨텍스트 정리

체크포인트와 함께 쓰면 유용한 컨텍스트 관리 방법입니다.

```
기본 사용:
/compact
-> Claude가 대화 내용을 자동으로 요약
   중요한 것만 남기고 불필요한 것 정리

지시 포함:
/compact 수정된 파일 목록과 테스트 명령어는 반드시 보존해줘
/compact OAuth 구현 관련 결정사항에 집중해서 요약해줘

CLAUDE.md에 영구 규칙 추가:
# CLAUDE.md
컨텍스트 압축 시 수정된 파일 목록, API 엔드포인트,
그리고 테스트 실행 명령어는 반드시 보존할 것

예시 케이스:
2시간째 작업 중... 대화가 너무 길어짐
-> /compact OAuth 관련 주요 결정사항 위주로 요약해줘
-> 깔끔해진 컨텍스트로 작업 계속
```
