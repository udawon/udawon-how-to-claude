## 먼저 비유로 이해하기

```
스킬(Skill) = 레시피 카드 한 장
  "김치찌개 만드는 법" 이라는 종이 한 장
  -> Claude에게 "이 방법대로 해줘"라고 알려주는 지침서

플러그인(Plugin) = 요리 키트 (밀키트)
  레시피 카드 + 양념 세트 + 특수 도구 + 조리법이 한 박스에 들어있음
  -> Claude에게 새로운 능력 자체를 설치해주는 패키지

정리하면:
  스킬 = 설명서 1장 (텍스트 지침)
  플러그인 = 설명서 + 도구 + 자동화 규칙이 묶인 패키지
```

## 스킬(Skill)이란?

```
SKILL.md라는 파일에 적힌 "이렇게 해줘" 지침

예시:
  "코드 리뷰할 때는 보안 -> 성능 -> 가독성 순서로 체크해줘"
  "커밋 메시지는 한국어로, 이 형식으로 써줘"

특징:
  - 텍스트 파일 하나 (SKILL.md)
  - Claude가 읽고 따르는 지침
  - CLAUDE.md의 확장판 같은 느낌
  - 직접 만들기 쉬움

비유: 신입사원에게 주는 업무 매뉴얼
     "고객 전화 받을 때는 이렇게 인사하고,
      이런 순서로 처리하세요"
     -> 행동 방식을 알려주는 것
```

## 플러그인(Plugin)이란?

```
여러 기능이 한 번에 묶인 설치형 패키지

플러그인 안에 들어있을 수 있는 것들:
  스킬 (SKILL.md) - 지침서
  MCP 서버 - 외부 도구 연결
  Hooks - 자동화 규칙
  에이전트 - 전문 AI 비서

예시: 'frontend-design' 플러그인
  -> 디자인 스킬 + 스크린샷 도구 + 코드 생성 규칙이 한 번에 설치됨

비유: 스마트폰 앱 설치
     카카오톡을 설치하면:
       메시지 기능 + 사진 전송 + 음성통화 + 송금
     이 모든 게 한 번에 들어옴

     플러그인도 마찬가지:
       설치 한 번이면 여러 기능이 한꺼번에 추가됨
```

## 핵심 차이 비교

| 구분 | 스킬 (Skill) | 플러그인 (Plugin) |
|------|-------------|-----------------|
| 정체 | 텍스트 지침 1개 | 기능 묶음 패키지 |
| 구성 | SKILL.md 파일 | 스킬+도구+자동화 |
| 설치 | 파일 만들기 | 마켓플레이스 설치 |
| 능력 | '이렇게 해줘' | '이 도구를 써' |
| 만들기 | 쉬움 (글쓰기) | 어려움 (개발 필요) |
| 외부 연결 | 불가능 | 가능 (API, 도구) |

```
비유:
  스킬 = 요리 레시피 (종이에 적힌 방법)
       -> "이 순서대로 만들어" 라고 알려줌
       -> 재료와 도구는 이미 있어야 함

  플러그인 = 밀키트 (레시피 + 재료 + 도구 세트)
           -> 레시피도 있고, 특수 소스도 들어있고,
              전용 조리 도구까지 포함
           -> 설치만 하면 바로 사용 가능
```

## 플러그인 마켓플레이스

```
마켓플레이스 = 앱스토어

스마트폰의 앱스토어처럼:
  Claude Code에도 플러그인 마켓플레이스가 있음
  필요한 기능을 검색하고 설치하면 끝

따라하기 (설치 방법):
  1. Claude Code를 실행한다 (터미널에서 claude 입력)
  2. Claude Code 대화창에서 /install 입력
     -> 마켓플레이스 목록이 표시됨
     -> 원하는 플러그인을 선택하면 설치 완료

  또는 설치할 플러그인을 이미 알고 있다면:
  1. Claude Code 대화창에서 직접 입력:
     /install @anthropic/frontend-design
     -> 바로 설치됨

비유: 아이폰 앱스토어
     앱스토어 열기(/install) -> 앱 검색 -> 설치 -> 바로 사용
```

## 인기 플러그인 종류

```
1. 코드 지능 플러그인
   코드 분석, 리뷰, 최적화 도구
   비유: 문서 맞춤법 검사기 (더 똑똑한 버전)

2. 외부 서비스 연결 플러그인
   GitHub, Slack, DB 등과 연결
   비유: 만능 리모컨 (여러 기기를 하나로 조작)

3. 개발 워크플로우 플러그인
   테스트 자동화, 배포, 모니터링
   비유: 공장 자동화 라인 (작업이 자동으로 흘러감)

4. 디자인 플러그인
   UI 생성, 스크린샷, 디자인 검증
   비유: 인테리어 시뮬레이터 (미리 보고 결정)
```

## 플러그인 관리

```
아래 명령어는 모두 Claude Code 대화창에서 입력합니다.
(claude를 실행한 후 나오는 화면)

설치:    /install @제작자/플러그인이름
목록:    /plugins
제거:    /uninstall 플러그인이름

따라하기 (설치된 플러그인 확인):
  1. 터미널에서 claude 입력하여 Claude Code 실행
  2. Claude Code 대화창에서 /plugins 입력
  3. 설치된 플러그인 목록이 표시됨

비유: 앱 관리
     설치 = 앱스토어에서 다운로드
     목록 = 설치된 앱 확인
     제거 = 앱 삭제
```

## 플러그인의 권한과 보안

```
플러그인도 내 허락 없이 마음대로 못 함

설치할 때:
  "이 플러그인이 파일을 읽어도 될까요?" 같은 확인 요청
  -> 동의해야 작동

비유: 앱 설치 시 권한 요청
     "카메라 접근을 허용하시겠습니까?"
     "위치 정보를 허용하시겠습니까?"
     -> 내가 허용한 것만 사용 가능
```

## 언제 뭘 쓰면 좋을까?

```
스킬을 쓸 때:
  Claude의 행동 방식을 바꾸고 싶을 때
  "이런 형식으로 해줘" 같은 규칙을 정할 때
  직접 만들어서 프로젝트에 넣고 싶을 때

  예: "코드 리뷰는 항상 보안 체크부터 해줘"
     "커밋 메시지는 이 형식으로 써줘"

플러그인을 쓸 때:
  새로운 도구/능력이 필요할 때
  외부 서비스와 연결해야 할 때
  복잡한 기능을 한 번에 추가하고 싶을 때

  예: "디자인 생성 기능이 필요해" -> frontend-design 플러그인
     "Slack과 연동하고 싶어" -> Slack 플러그인

비유 정리:
  스킬 = 직원에게 주는 업무 지침서
        (행동 규칙을 알려줌)

  플러그인 = 직원에게 주는 새 장비
            (새로운 능력을 부여함)
```
