## CLAUDE.md란?

Claude Code가 세션 시작할 때 자동으로 읽는 설정 파일입니다.
매번 말할 필요 없이, 한 번 적어두면 영원히 적용됩니다.

> 비유: 단골 카페에 "저는 항상 아메리카노 덜 달게요"를 등록해두는 것

## 파일 위치

| 위치 | 설명 | 비유 |
|------|------|------|
| `~/.claude/CLAUDE.md` | 글로벌 설정 (모든 프로젝트) | 회사 전체 규칙 |
| `프로젝트/.claude/CLAUDE.md` | 프로젝트별 설정 | 팀별 규칙 |

`~` 는 내 홈 폴더를 의미합니다:
- Windows: `C:\Users\내이름`
- Mac: `/Users/내이름`

예시:
- 글로벌: `C:\Users\내이름\.claude\CLAUDE.md`
- 프로젝트별: `C:\Users\내이름\my-project\.claude\CLAUDE.md`

## CLAUDE.md 만드는 방법

**방법 1:** Claude Code에게 시키기 (가장 쉬움)
1. 터미널에서 프로젝트 폴더로 이동
2. `claude` 입력하여 Claude Code 실행
3. Claude Code 대화창에서:
   "<span class="keyword-highlight">이 프로젝트의 CLAUDE.md를 만들어줘.</span>
    한국어로 응답하고, Tailwind CSS를 사용하는 규칙을 넣어줘."
→ Claude가 자동으로 .claude 폴더와 CLAUDE.md 파일을 생성

**방법 2:** `/init` 명령어 사용
1. Claude Code 대화창에서 `/init` 입력
→ Claude가 프로젝트를 분석하고 CLAUDE.md를 자동 생성

> **💡 `/init` 실행 타이밍 팁**: 프로젝트 시작 직후보다 **코드가 어느 정도 쌓인 후**에 `/init`을 실행하는 게 더 효과적입니다. 코드가 있어야 Claude가 프로젝트의 구조, 기술 스택, 아키텍처를 정확하게 파악해서 더 유용한 CLAUDE.md를 만들어줍니다. 초반에는 간단한 규칙만 직접 적어두고, 프로젝트가 어느 정도 진행된 후 `/init`으로 전체 맥락을 반영한 CLAUDE.md를 생성하세요.

**방법 3:** 직접 만들기
1. 프로젝트 폴더 안에 `.claude` 폴더를 만든다
2. 그 안에 `CLAUDE.md` 파일을 만든다
3. 메모장이나 편집기로 내용을 작성한다

비유: 카페 취향 등록
     방법1 = "제 취향 등록해주세요" (직원에게 맡기기)
     방법2 = 자동 설문조사 작성 (앱이 질문하고 등록)
     방법3 = 직접 수기로 적기

**예시 케이스:**

<div class="example-case">

글로벌 설정 (모든 프로젝트에 적용):
"기본 응답 언어: 한국어" 를 글로벌에 적어두면
→ 어떤 프로젝트를 열어도 항상 한국어로 답변

프로젝트별 설정:
쇼핑몰 프로젝트에만 "결제 관련 수정 시 반드시 확인받을 것"을 적어두면
→ 쇼핑몰에서만 적용. 다른 프로젝트엔 영향 없음

비유: 회사 전체 복장 규정 (글로벌) vs 마케팅팀만의 회의 규칙 (프로젝트별)
</div>

## 주요 설정 항목

### 언어 설정
```markdown
## 언어
- 기본 응답 언어: 한국어
- 코드 주석: 한국어
- 커밋 메시지: 한국어
```

**예시 케이스:**

<div class="example-case">

설정 없이:
Claude가 영어로 답변하거나, 코드 설명을 영어로 달 수 있음

설정 후:
항상 한국어로 답변, 한국어 주석, 한국어 저장 메시지

비유: 외국인 직원에게 "앞으로 보고서는 한국어로 써줘"라고
     한 번 말해두면 매번 안 말해도 됨
</div>

### 기술 스택 설정
```markdown
## 기술 스택
- CSS: Tailwind CSS
- UI: shadcn/ui
```

**예시 케이스:**

<div class="example-case">

설정 없이: "로그인 화면 만들어줘"
→ Claude가 일반 스타일로 만들 수 있음. 매번 "Tailwind 써줘"라고 말해야 함

설정 후: "로그인 화면 만들어줘"
→ 자동으로 Tailwind + shadcn/ui로 만들어줌

비유: 디자이너에게 "앞으로 모든 디자인은 이 스타일 가이드를 따라줘"
     한 번 정해두면 매번 안 말해도 됨
</div>

### 규칙 정의
```markdown
## 규칙
- any 타입 사용 금지
- 컴포넌트 분리 및 재사용
- 반응형 필수
```

**예시 케이스:**

<div class="example-case">

"반응형 필수" 규칙 설정 후:

Claude가 화면을 만들 때
→ 자동으로 PC/태블릿/모바일 모두에서 잘 보이게 만듦
   매번 "모바일에서도 잘 보이게 해줘"라고 안 말해도 됨

비유: 요리사에게 "앞으로 모든 음식은 저염으로"라고 정해두면
     매번 "싱겁게 해주세요" 안 말해도 됨
</div>

### 모델 전환 규칙
```markdown
## 모델 자동 전환 (IMPORTANT)
매 작업 시작 시 복잡도를 판단하여 적합한 모델을 안내한다.
전환이 필요하면 안내만 출력하고 멈춘다.
```

**예시 케이스:**

<div class="example-case">

이 규칙을 적어두면:

단순 질문을 했을 때:
→ Claude: "Sonnet 전환 → `/model` 후 resume"
   (비싼 모델 대신 저렴한 모델을 쓰라고 안내)

복잡한 설계를 요청했을 때:
→ Claude가 바로 작업 시작 (이미 적합한 모델)

비유: 비서가 "이건 택배로 보내면 되는데 퀵으로 보내실 건가요?"
     비용을 절약하도록 안내해주는 것
</div>

## @import로 파일 분리하기

CLAUDE.md가 길어지면 Claude가 뒤쪽 규칙을 무시합니다. `@파일경로` 문법으로 내용을 분리하세요.

```markdown
# CLAUDE.md (간결하게 유지)

프로젝트 개요: @README.md
사용 가능한 명령어: @package.json

# 추가 규칙
- Git 워크플로우: @docs/git-guide.md
- API 설계 규칙: @docs/api-conventions.md
- 개인 설정 (git 제외): @~/.claude/personal-overrides.md
```

왜 분리해야 하나?

(X) CLAUDE.md에 모든 내용을 때려넣는 경우:
200줄짜리 CLAUDE.md
→ Claude가 앞쪽 규칙만 보고 뒤쪽은 무시하는 현상 발생

(O) @import로 분리:
CLAUDE.md는 30줄 이내로 유지
세부 내용은 별도 파일로 분리
→ 모든 규칙이 안정적으로 적용됨

판단 기준:
"이 규칙이 없으면 Claude가 실수를 하나?"
→ 아니라면 삭제
→ 그렇다면 별도 파일로 분리

비유: 회사 규정집이 두꺼우면 아무도 안 읽음
     핵심 원칙 1페이지 + 세부 사항은 별첨으로

## CLAUDE.md 위치별 역할

| 경로 | 역할 |
|------|------|
| `~/.claude/CLAUDE.md` | 모든 프로젝트에 적용 (글로벌) |
| `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` | 현재 프로젝트 (git에 포함, 팀 공유) |
| `./.claude/CLAUDE.local.md` | 개인 설정 (.gitignore에 추가) |

### .claude/rules/ 디렉토리

대규모 프로젝트에서는 규칙을 여러 파일로 나눌 수 있습니다.

```
프로젝트/
├── .claude/
│   ├── CLAUDE.md           # 주요 프로젝트 지침
│   └── rules/
│       ├── code-style.md   # 코드 스타일 가이드
│       ├── testing.md      # 테스트 규칙
│       └── security.md     # 보안 요구사항
```

파일 맨 위에 `paths:` 설정을 넣으면 특정 파일에만 적용되는 규칙을 만들 수 있습니다:

```markdown
---
paths:
  - "src/api/**/*.ts"
---

# API 개발 규칙
- 모든 API 엔드포인트에 입력 검증을 포함해야 합니다
```

**실전 예시:**

글로벌(`~/.claude/CLAUDE.md`):
- 언어: 한국어
- 모델 전환 규칙

프로젝트(`./CLAUDE.md`):
- 기술 스택 (Tailwind, shadcn/ui)
- 팀 코딩 규칙
- 테스트 명령어

개인(`./.claude/CLAUDE.local.md`):
- 개인적인 선호사항
- 팀원과 공유하고 싶지 않은 설정

> **핵심**: CLAUDE.md에 적은 규칙은 Claude의 기본 동작보다 우선합니다.
> 한 번 잘 적어두면 매 세션마다 일관된 품질을 보장합니다.
> CLAUDE.md는 짧을수록 더 잘 지켜집니다.
