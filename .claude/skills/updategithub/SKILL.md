# GitHub Trending → 문서 자동 생성 스킬

## 실행 모드

스킬은 **두 가지 모드**로 동작한다. 인자(argument)에 따라 자동 분기:

| 인자 | 모드 | 동작 |
|------|------|------|
| 없음 | 트렌딩 모드 | GitHub trending Top 10 수집 → 분석 → 주간 문서 생성 |
| GitHub URL | 직접 지정 모드 | 해당 레포만 분석 → 단독 문서 생성 |

## 실행 방식

```
/updategithub                                      # 트렌딩 모드
/updategithub https://github.com/owner/repo        # 직접 지정 모드
```

---

## 실행 절차

### Step 1: 레포 소스 확보

#### 모드 A: 트렌딩 모드 (인자 없음)

`/browse` 스킬(gstack headless browser)을 사용하여 트렌딩 페이지를 열고 데이터를 수집한다.

#### 모드 B: 직접 지정 모드 (URL 제공)

사용자가 GitHub URL을 인자로 전달하면 트렌딩 수집을 건너뛰고 바로 Step 1.5로 진행한다.

---

### Step 1 (트렌딩 모드 전용): GitHub Trending 페이지 수집

`/browse` 스킬(gstack headless browser)을 사용하여 트렌딩 페이지를 열고 데이터를 수집한다.

```
URL: https://github.com/trending?since=weekly
```

각 레포지토리에서 추출할 정보:
- 레포 이름 (`owner/repo`)
- 설명 (description)
- 언어 (language)
- 이번 주 Stars 증가량
- 전체 Stars / Forks
- 레포 URL

상위 10개만 수집. 추출 후 `.tmp/github_trending.json`에 저장.

### Step 1.5: 중복 검사

이미 문서화된 레포인지 확인한다:

```bash
grep -r "repo_url:" content/github-update/ | grep "{REPO_NAME}"
```

중복 레포가 있으면 사용자에게 알리고:
- **트렌딩 모드**: 해당 레포를 목록에서 제외하고 나머지 계속 진행
- **직접 지정 모드**: 사용자에게 "이미 문서화된 레포입니다: {파일경로}" 알린 후, 계속 진행할지 선택 요청
  - (A) 기존 문서에 새 내용 추가/업데이트
  - (B) 새 문서로 다시 작성
  - (C) 취소

### Step 2: 레포별 상세 분석

#### 모드 A (트렌딩): Step 1에서 수집한 10개 레포를 분석. 5개 이상이면 서브에이전트로 병렬 처리.
#### 모드 B (직접 지정): 사용자가 제공한 URL의 레포 1개만 분석. 메인 에이전트가 직접 처리.

각 레포의 GitHub 페이지와 README를 방문하여 상세 정보를 수집한다.

**수집 항목:**
- 프로젝트 목적 (한 줄 설명)
- 핵심 기능 3~5가지
- 어떤 문제를 해결하는가
- 주요 사용 언어/기술 스택
- 라이선스
- 관련 태그/주제

**분석 리포트 형식:**

```markdown
## {owner/repo}

> **언어**: {언어} | **이번 주 Stars**: +{N} | **전체 Stars**: {N}
> **분야**: {AI/웹/데브옵스/보안/기타}

### 한 줄 요약
{이 프로젝트가 무엇인지 비개발자도 이해할 수 있는 설명}

### 핵심 기능
1. **{기능명}**: {설명}
2. **{기능명}**: {설명}
3. **{기능명}**: {설명}

### 왜 주목받는가
{이번 주에 특히 인기를 끈 이유 — 신기능 출시, 바이럴, 특정 문제 해결 등}

### 메타데이터
| 항목 | 값 |
|------|-----|
| repo_url | https://github.com/{owner/repo} |
| 분야 | {분야} |
| 문서화 가치 | {점수}/10 |
```

### Step 3: 분석 결과 보고 및 확인

#### 모드 A (트렌딩): 테이블로 보고 후 확인 요청

```
| 순위 | 레포 | 분야 | 이번 주 Stars | 한 줄 요약 |
|------|------|------|-------------|-----------|
| 1 | owner/repo | AI | +5,200 | ... |
```

**사용자 확인을 받은 후** Step 4 진행.

#### 모드 B (직접 지정): 분석 요약 보고 후 확인 요청

```
## {owner/repo} 분석 완료

- 분야: {분야}
- 전체 Stars: {N}
- 한 줄 요약: {설명}

문서를 생성할까요?
```

**사용자 확인을 받은 후** Step 4 진행.

### Step 4: 문서 생성

> `references/doc-template.md`를 읽어서 템플릿과 작성 규칙을 따른다.

**파일 저장 위치:** `content/github-update/`

#### 모드별 파일명 규칙

| 모드 | 파일명 |
|------|--------|
| 트렌딩 | `YYYY-MM-DD-github-trending.md` |
| 직접 지정 | `YYYY-MM-DD-{repo-name}.md` (예: `2026-04-09-anthropic-claude.md`) |

#### order 값 결정

```bash
grep -h "^order:" content/github-update/*.md 2>/dev/null | sort -t: -k2 -n | tail -1
```

없으면 `order: 1`부터 시작.

#### 문서 구조

- **트렌딩 모드**: 주간 트렌딩 전체를 **하나의 문서**로 생성 (Top 10 + 인사이트)
- **직접 지정 모드**: 해당 레포 **단독 심층 분석 문서** 생성

직접 지정 모드 문서에 포함할 내용:
1. 프로젝트 개요 (비개발자용 설명 + 비유)
2. 핵심 기능 상세 분석
3. 어떤 사람에게 유용한가
4. 실전 활용 예시
5. 주의할 점 / 한계
6. 관련 유사 프로젝트 (있으면)

### Step 4.5: 품질 체크

**필수 체크:**
1. **전문 용어 괄호 설명** — 영어 기술 용어 첫 등장 시 괄호 설명 포함
2. **빈 공백 없음** — 표나 예시 블록에 빈 칸 없음
3. **비유 존재** — "이게 뭔가요?" 섹션에 일상 비유 최소 1개
4. **예시 블록** — `<div class="example-case">` 블록 최소 2개

### Step 5: 결과 보고

#### 모드 A (트렌딩)
```
## 실행 결과

- 수집된 레포: 10개
- 중복 제외: N개
- 분석 완료: N개
- 생성된 문서: content/github-update/{YYYY-MM-DD}-github-trending.md

### 이번 주 트렌드 요약
- {분야 1}: N개 레포 진입
- {분야 2}: N개 레포 진입
- 급상승 언어: {언어}
```

#### 모드 B (직접 지정)
```
## 실행 결과

- 분석 레포: {owner/repo}
- 생성된 문서: content/github-update/{YYYY-MM-DD}-{repo-name}.md
```

보고 후: "문서를 커밋하려면 `/push`를 실행하세요."

## 에러 처리

| 상황 | 대응 |
|------|------|
| GitHub 페이지 로딩 실패 | `/browse` 재시도 1회. 실패 시 사용자 보고 |
| README 없는 레포 | GitHub description + 코드 파일명으로 추론 |
| 설명 없는 레포 | "설명 없음" 표기, 코드에서 목적 추론 시도 |
| `.tmp` 폴더 없음 | `mkdir -p .tmp` 자동 생성 후 진행 |

## .tmp 디렉토리

```
.tmp/
└── github_trending.json    # 수집된 트렌딩 데이터
```
