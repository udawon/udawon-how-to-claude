# 문서 템플릿 & 작성 규칙

> 이 파일은 /updateyoutube 스킬이 문서를 생성할 때 참조하는 템플릿과 규칙을 담고 있다.
> Step 4 (문서 생성) 시점에 읽는다. 서브에이전트에게도 이 파일 경로를 전달하여 직접 읽도록 한다.

## 문서 형식 (필수)

```markdown
---
date: "{YYYY-MM-DD 형식, /updateyoutube 실행한 당일 날짜}"
title: "문서 제목"
description: "한 줄 설명"
order: {기존 문서 최대 order + 1}
tags: ["관련", "태그", "목록"]
source_url: "https://youtube.com/watch?v={VIDEO_ID}"
---

## 이게 뭔가요?

{비개발자도 이해할 수 있는 쉬운 설명}
{일상 비유를 활용 — "카카오톡에서 메시지 보내듯이..." 같은 식}

## 왜 알아야 하나요?

{이걸 알면 뭐가 좋은지, 모르면 뭐가 불편한지}

## 어떻게 하나요?

### 방법 1: {구체적인 방법}

{단계별 설명}

<div class="example-case">
<strong>💬 예시</strong>

{실제로 따라할 수 있는 구체적 예시}

</div>

### 방법 2: {있다면 추가}

## 실전 예시

<div class="example-case">
<strong>📌 실전 케이스: {상황 설명}</strong>

{구체적인 시나리오와 해결 과정}

</div>

## 주의할 점

{흔한 실수, 주의사항}

{핵심 요약 3줄 이내}
```

## 문서 작성 규칙

1. **쉬운 말**: 전문 용어 쓸 때는 반드시 괄호 안에 쉬운 설명 추가. 예: "MCP(외부 도구 연결 기능)"
2. **비유 활용**: 모든 개념을 일상 비유로 한 번은 설명. 프로젝트 기존 스타일 참고:
   - "게임 세이브 포인트" (체크포인트)
   - "카카오톡 채팅창" (CLI 인터페이스)
   - "의사의 처방" (프롬프트)
3. **예시 필수**: `<div class="example-case">` 블록 최소 2개 이상
4. **파일명**: kebab-case (`youtube-tips.md`, `claude-workflow-pattern.md`)
5. **태그**: 영상 출처 태그 포함 (`"youtube"` 태그 필수)
6. **출처 표기**: 문서 하단에 참고 영상 링크 포함
7. **Mermaid 다이어그램**: 아래 기준에 따라 선택적으로 사용 (문서당 최대 2개)
8. **Mac/Windows 병기**: 키보드 단축키나 터미널 조작 설명 시 반드시 Mac과 Windows 둘 다 표기

## Mermaid 다이어그램 사용 기준

**사용하는 경우** — "텍스트로 읽으면 머릿속에서 그림을 그려야 하는 내용"에만 사용:

- **흐름/프로세스**: 스킬 체이닝 파이프라인, 자동화 워크플로우 등 순서가 있는 구조
- **계층 구조**: 레이어 구조, 시스템 아키텍처 등 위아래 관계
- **의사결정 분기**: "어떤 방법을 써야 할까?" 같은 선택 플로우차트
- **관계도**: A와 B가 어떻게 연결되는지 보여주는 구조

**사용하지 않는 경우:**

- 단순 나열 (기능 목록, 팁 모음) → 표나 불릿 포인트가 더 읽기 쉬움
- 단계별 가이드 (1→2→3) → 번호 매긴 텍스트가 더 명확
- 이미 표로 잘 정리된 비교 → 다이어그램으로 바꾸면 오히려 복잡
- 문서당 3개 이상 → 과하면 산만해짐

**Mermaid 코드 작성 규칙:**

```markdown
\`\`\`mermaid
graph TB
    A["한국어 라벨"] --> B["한국어 라벨"]
    style A fill:#c2410c,color:#fff,stroke:#9a3412,stroke-width:2px
\`\`\`
```

- 노드 라벨은 반드시 한국어
- `<br/>`로 줄바꿈 가능 (`A["첫 번째 줄<br/>두 번째 줄"]`)
- 색상은 사이트 브랜드 컬러 사용: `#c2410c`(빨강), `#d97706`(주황), `#2563eb`(파랑), `#16a34a`(초록)

## 태그 규칙 (섹션 분류용)

youtube-update 카테고리는 태그로 섹션이 나뉜다. 문서마다 아래 태그 중 하나 이상을 반드시 포함:

- `신기능` 또는 `업데이트` → "새로운 기능 & 업데이트" 섹션
- `활용법` 또는 `워크플로우` → "활용법 & 워크플로우" 섹션
- `설정` 또는 `연동` → "설정 & 연동" 섹션
- `비즈니스` 또는 `자동화` → "비즈니스 & 자동화" 섹션

모든 문서에 `youtube` 태그도 필수 포함.
