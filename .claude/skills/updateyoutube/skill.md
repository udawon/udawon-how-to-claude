# YouTube -> 문서 자동 생성 스킬

## 실행 모드

스킬은 **두 가지 모드**로 동작한다. 인자(argument)에 따라 자동 분기:

| 인자 | 모드 | 동작 |
|------|------|------|
| 없음 | 검색 모드 | YouTube에서 최근 영상 검색 -> 분석 -> 문서 생성 |
| URL | 직접 지정 모드 | 해당 영상만 추출 -> 분석 -> 문서 생성 |
| `--hours N` | 검색 모드 (시간 변경) | N시간 이내 영상 검색 |

## 실행 절차

### Step 1: 영상 소스 확보

#### 모드 A: 검색 모드 (인자 없음 또는 --hours)

```bash
python tools/youtube_search.py --hours 12 --output .tmp/youtube_search.json
```

- 결과 파일 `.tmp/youtube_search.json`을 읽어서 검색된 영상 목록 확인
- 영상이 0개면 "최근 12시간 내 신규 영상이 없습니다"라고 알려주고 종료
- `--hours` 값은 사용자가 다르게 요청하면 변경 가능

#### 모드 B: 직접 지정 모드 (URL 제공)

사용자가 YouTube URL을 인자로 전달하면 검색을 건너뛰고 바로 Step 2로 진행한다.

### Step 1.5: 중복 검사

자막 추출 전에, 이미 문서화된 영상인지 확인한다. 기존 문서의 frontmatter에서 `source_url` 필드를 검색:

```bash
grep -r "source_url:" content/youtube-update/ | grep "{VIDEO_ID}"
```

이미 문서화된 video_id가 발견되면:
- 해당 영상을 대상 목록에서 제외
- 사용자에게 "이미 문서화된 영상: {제목} ({파일경로})" 알림
- 모든 영상이 중복이면 "모두 이미 문서화되어 있습니다" 출력 후 종료

### Step 2: 영상 콘텐츠 추출

#### 검색 모드 (모드 A)
```bash
python tools/youtube_transcript.py .tmp/youtube_search.json --output .tmp/youtube_transcripts.json
```

#### 직접 지정 모드 (모드 B)
```bash
python tools/youtube_transcript.py "VIDEO_URL" --output .tmp/youtube_transcripts.json
```

직접 지정 모드에서는 자막 추출 후 **yt-dlp로 메타데이터**(제목, 채널명, 조회수, 길이, 업로드 날짜)를 별도로 가져온다. youtube_transcript.py가 메타데이터를 비워서 반환할 수 있으므로:

```bash
yt-dlp --dump-json --no-download "VIDEO_URL"
```

위 명령으로 title, channel, view_count, duration_string, upload_date를 보충한다.

- 자막 -> Whisper 음성인식 -> 메타데이터 순으로 자동 폴백
- 결과 파일에서 각 영상의 추출 방식(method)과 텍스트(text) 확인
- `method: "failed"`인 영상은 분석 대상에서 제외하되, 사용자에게 목록 보고

### Step 2.5: 1차 필터링 (토큰 절약)

본격적인 내용 분석 전에, 제목과 메타데이터만으로 빠르게 걸러낸다. 전체 자막을 분석하는 것보다 훨씬 저렴하기 때문이다.

**제외 기준:**
- Claude Code와 무관한 영상 (제목/설명에 관련 키워드 없음)
- 영상 길이 1분 미만 (Shorts 등 — 문서화 가치 낮음)
- 영상 길이 3시간 이상 (라이브 스트림 전체 녹화 등)

제외된 영상은 사용자에게 "1차 필터링에서 제외: {제목} (사유)" 한 줄씩 보고.

### Step 3: 내용 분석 및 기존 문서 비교

> **병렬 처리 규칙**: 분석 대상 영상이 5개 이상이면 서브에이전트를 활용하여 병렬 처리한다.
> 4개 이하면 메인 에이전트가 순차 처리한다.

#### 분석 리포트 형식

각 영상을 분석할 때 아래 형식으로 구조화된 리포트를 생성한다. 이 리포트는 사용자 보고와 문서 작성의 원본 자료로 모두 쓰인다.

```markdown
## {영상 제목}

> **채널**: {채널명} | **길이**: {영상 길이} | **조회수**: {조회수}
> **문서화 가치**: {점수}/10 | **판정**: {신규 생성/기존 보완/건너뛰기}

### 핵심 요약
{영상 전체를 3~5문장으로 압축. 이 영상이 뭘 말하는지 한눈에 파악 가능해야 함}

### 타임스탬프별 주요 내용
**섹션 1: {소제목}** [MM:SS]
1. **{핵심 키워드}**: {설명} [MM:SS]
2. **{핵심 키워드}**: {설명} [MM:SS]

### 실용적 팁 (바로 써먹기)
- {팁 1}
- {팁 2}

### 분석 메타데이터
| 항목 | 값 |
|------|-----|
| video_id | {id} |
| source_url | https://youtube.com/watch?v={id} |
| 태그 분류 | {아래 태그 규칙 참고} |
| 문서화 가치 | {score}/10 |
| 판정 | {verdict} |
| 판정 사유 | {reason} |
| 보완 대상 문서 | {경로 또는 "-"} |
```

**리포트 규칙:**
- 자막이 있는 영상은 `[MM:SS]` 타임스탬프 필수. 없으면 `[메타데이터 기반]` 표기
- 각 포인트는 `**볼드 키워드**: 설명` 패턴 유지
- 대주제(섹션) -> 소항목(번호) 2단 계층으로 구성

#### 분석 기준

| 항목 | 기준 |
|------|------|
| 핵심 주제 | Claude Code 관련 주제 |
| 실용적 팁 | 바로 써먹을 수 있는 구체적 팁/방법 |
| 태그 분류 | 아래 태그 규칙에 따라 분류 (복수 선택 가능) |
| 문서화 가치 | 10점 만점 — 7점 이상만 문서화 대상 |
| 기존 문서 비교 | `content/youtube-update/*.md`와 비교하여 신규 생성 / 기존 보완 / 건너뛰기 판정 |

#### 태그 분류 규칙

기존 문서와 일관성을 유지하기 위해, 아래 태그 조합 중에서 선택한다:

| 태그 | 해당하는 내용 | 사이트 섹션 |
|------|-------------|-----------|
| `신기능`, `업데이트` | 새로 출시된 기능, 버전 업데이트 | "새로운 기능 & 업데이트" |
| `활용법`, `워크플로우` | 사용법, 작업 흐름, 팁 | "활용법 & 워크플로우" |
| `설정`, `연동` | 설치, 설정, 외부 도구 연결 | "설정 & 연동" |
| `비즈니스`, `자동화` | 업무 활용, 자동화 사례 | "비즈니스 & 자동화" |

모든 문서에 `youtube` 태그는 필수. 복수 태그 선택 가능 (예: `["신기능", "업데이트", "활용법", "youtube"]`).

#### 병렬 처리 시 서브에이전트 배분

```
5~12개  -> 서브에이전트 2개 (절반씩)
13~24개 -> 서브에이전트 3개 (균등 분배)
25개+   -> 서브에이전트 4개 (균등 분배, 최대)
```

서브에이전트에게는 위 분석 리포트 형식 + 분석 기준 + 담당 영상 데이터를 전달한다. 코드를 작성하지 말고 분석만 수행하도록 지시한다.

#### 결과 취합

모든 분석이 완료되면 사용자에게 테이블로 보고:

```
| 영상 | 주제 | 점수 | 판정 | 사유 |
|------|------|------|------|------|
| 영상제목 | OO 활용법 | 9 | 신규 생성 | 관련 문서 없음 |
```

**사용자 확인을 받은 후** Step 4 진행.

### Step 4: 신규 문서 생성

> 문서 생성 시, `references/doc-template.md`를 읽어서 템플릿과 작성 규칙을 따른다.
> 서브에이전트에게도 이 파일을 직접 읽도록 경로를 전달한다.

#### order 값 결정

새 문서의 order 값을 정하려면 기존 문서의 최대값을 확인한다:

```bash
grep -h "^order:" content/youtube-update/*.md | sort -t: -k2 -n | tail -1
```

최대값 + 1부터 순차적으로 부여. 여러 문서를 동시에 생성할 때는 +1, +2, +3... 순서.

#### 병렬 처리

생성할 문서가 3개 이상이면 서브에이전트를 활용. 2개 이하면 메인이 직접 작성.

```
3~5개  -> 서브에이전트 2개
6개+   -> 서브에이전트 3개 (최대)

각 서브에이전트에게 전달:
  - 담당 영상의 분석 결과 + 자막 텍스트
  - references/doc-template.md 경로 (Read하여 템플릿 참조)
  - 저장할 파일 경로 + order 값
  - Write tool로 직접 파일 생성
```

**"기존 보완" 판정 문서는 메인 에이전트가 직접 처리한다** (기존 문서를 읽고 적절한 위치에 Edit으로 추가해야 하므로).

### Step 4.5: 품질 체크

문서 생성 후, 파일을 저장하기 전에 아래 체크리스트를 통과하는지 확인한다. 통과하지 못하면 수정 후 저장.

**필수 체크:**
1. **전문 용어 괄호 설명** — 문서에서 영어 기술 용어(Hook, YAML, API, CLI, SDK, MCP 등)가 처음 등장할 때 괄호 설명이 있는가?
2. **빈 공백 없음** — 표나 예시 블록 안에 빈 칸이 없는가? 데이터가 없으면 해당 섹션을 제거했는가?
3. **Mermaid 품질** — 노드 텍스트 15자 이내, 노드 8개 이하, 텍스트 색상 `#fff`, 배경 충분히 진한가?
4. **비유 존재** — "이게 뭔가요?" 섹션에 일상 비유가 최소 1개 포함되어 있는가?
5. **Mac/Windows 병기** — 단축키 설명이 있다면 두 OS 모두 표기했는가?

### Step 5: 결과 보고

최종 결과를 사용자에게 보고:

```
## 실행 결과

- 검색된 영상: N개
- 중복 제외: N개 (이미 문서화)
- 1차 필터링 제외: N개
- 분석 완료: N개 (자막: N, Whisper: N, 메타데이터: N, 실패: N)
- 신규 문서 생성: N개
- 기존 문서 보완: N개
- 건너뛴 영상: N개 (점수 미달/중복 주제)

### 생성된 문서
- content/youtube-update/새문서.md -- "문서 제목"

### 보완된 문서
- content/youtube-update/기존문서.md -- "OO 섹션 추가"
```

보고 후 사용자에게 제안: "생성된 문서를 커밋하려면 `/push`를 실행하세요."

## 에러 처리

| 에러 | 대응 |
|------|------|
| YouTube API 할당량 초과 | "일일 API 할당량을 초과했습니다. 내일 다시 시도하세요." 안내 |
| Whisper 실패 | 메타데이터 폴백으로 자동 전환, 실패 영상 목록 보고 |
| YouTube API 키 오류 | `yt-dlp --dump-json --no-download`로 메타데이터 폴백. yt-dlp 미설치 시 `pip install yt-dlp` 안내 |
| 패키지 미설치 | `pip install -r tools/requirements.txt` 안내 후 재실행 유도 |
| Whisper 미설치 | `pip install openai-whisper` 안내. GPU 없으면 `--model tiny` 권장 |
| .tmp 디렉토리 없음 | `mkdir -p .tmp` 자동 생성 후 진행 |

## .tmp 디렉토리

중간 결과물은 모두 `.tmp/`에 저장. 이 폴더는 일회용이므로 언제 삭제해도 무방.

```
.tmp/
├── youtube_search.json       # 검색 결과
└── youtube_transcripts.json  # 추출된 콘텐츠
```
