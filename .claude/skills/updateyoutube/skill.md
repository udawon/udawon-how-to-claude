# YouTube → 문서 자동 생성 스킬

## 트리거
- `/updateyoutube` 입력 시 실행

## 개요
YouTube에서 Claude 관련 최신 영상을 검색하고, 내용을 분석하여 how-to-claude 프로젝트에 신규 문서를 자동 생성한다.

## 실행 절차

### Step 1: YouTube 영상 검색

```bash
cd C:/Users/udawon/project/how-to-claude
python tools/youtube_search.py --hours 12 --output .tmp/youtube_search.json
```

- 결과 파일 `.tmp/youtube_search.json`을 읽어서 검색된 영상 목록 확인
- 영상이 0개면 "최근 12시간 내 신규 영상이 없습니다"라고 알려주고 종료
- `--hours` 값은 사용자가 다르게 요청하면 변경 가능

### Step 2: 영상 콘텐츠 추출

```bash
python tools/youtube_transcript.py .tmp/youtube_search.json --output .tmp/youtube_transcripts.json
```

- 자막 → Whisper 음성인식 → 메타데이터 순으로 자동 폴백
- 결과 파일에서 각 영상의 추출 방식(method)과 텍스트(text) 확인
- `method: "failed"`인 영상은 분석 대상에서 제외하되, 사용자에게 목록 보고

### Step 3: 내용 분석 (서브에이전트 병렬 처리)

> **병렬 처리 규칙**: 분석 대상 영상이 5개 이상이면 서브에이전트를 활용하여 병렬 처리한다.
> 4개 이하면 메인 에이전트가 순차 처리한다.

#### 분석 리포트 형식 (lilys.ai 자세한 리포트 스타일)

**리포트 작성 규칙:**

1. **타임스탬프 필수**: 자막이 있는 영상은 반드시 `[MM:SS]` 형식으로 시점 표기
2. **볼드 키워드 선행**: 각 포인트는 `**키워드**: 설명` 패턴 유지
3. **계층형 구조**: 대주제(섹션) → 소항목(번호) 2단 계층
4. **실용 팁 분리**: 핵심 내용과 별도로 "바로 써먹기" 섹션을 분리하여 실용성 강조
5. **자막 없는 영상**: 타임스탬프 대신 `[메타데이터 기반]` 표기, 섹션은 주제 기반으로 구성

#### 병렬 처리 방식

```
분석 대상 영상 수에 따라 서브에이전트 배분:

  5~12개  → 서브에이전트 2개 (절반씩)
  13~24개 → 서브에이전트 3개 (균등 분배)
  25개+   → 서브에이전트 4개 (균등 분배, 최대 4개)
```

### Step 5: 신규 문서 생성 (서브에이전트 병렬 처리)

> **병렬 처리 규칙**: 생성할 문서가 3개 이상이면 서브에이전트를 활용하여 병렬 작성한다.
> 2개 이하면 메인 에이전트가 직접 작성한다.

#### 병렬 처리 방식

```
생성할 문서 수에 따라 서브에이전트 배분:

  3~5개  → 서브에이전트 2개
  6개+   → 서브에이전트 3개 (최대 3개)

각 서브에이전트에게 전달:
  - 담당 영상의 분석 결과 + 자막 텍스트
  - 아래 "문서 형식"과 "문서 작성 규칙" 전문
  - 저장할 파일 경로
  - 서브에이전트가 Write tool로 직접 파일을 생성
```

**"기존 보완" 판정 문서는 메인 에이전트가 직접 처리한다** (기존 문서를 읽고 적절한 위치에 Edit으로 추가해야 하므로).

#### 문서 작성 규칙

1. **쉬운 말**: 전문 용어 쓸 때는 반드시 괄호 안에 쉬운 설명 추가. 예: "MCP(외부 도구 연결 기능)"
2. **비유 활용**: 모든 개념을 일상 비유로 한 번은 설명. 프로젝트 기존 스타일 참고:
   - "게임 세이브 포인트" (체크포인트)
   - "카카오톡 채팅창" (CLI 인터페이스)
   - "의사의 처방" (프롬프트)
3. **예시 필수**: `<div class="example-case">` 블록 최소 2개 이상
4. **파일명**: kebab-case (`youtube-tips.md`, `claude-workflow-pattern.md`)
5. **태그**: 영상 출처 태그 포함 (`"youtube"` 태그 필수)
6. **출처 표기**: 문서 하단에 참고 영상 링크 포함
7. **Mermaid 다이어그램**: 아래 기준에 따라 선택적으로 사용 (문서당 최대 2개)
8. **Mac/Windows 병기**: 키보드 단축키나 터미널 조작 설명 시 반드시 Mac과 Windows 둘 다 표기

#### Mermaid 다이어그램 사용 기준

**사용하는 경우** — "텍스트로 읽으면 머릿속에서 그림을 그려야 하는 내용"에만 사용:

- **흐름/프로세스**: 스킬 체이닝 파이프라인, 자동화 워크플로우 등 순서가 있는 구조
- **계층 구조**: 레이어 구조, 시스템 아키텍처 등 위아래 관계
- **의사결정 분기**: "어떤 방법을 써야 할까?" 같은 선택 플로우차트
- **관계도**: A와 B가 어떻게 연결되는지 보여주는 구조

**사용하지 않는 경우:**

- 단순 나열 (기능 목록, 팁 모음) → 표나 불릿 포인트가 더 읽기 쉬움
- 단계별 가이드 (1→2→3) → 번호 매긴 텍스트가 더 명확
- 이미 표로 잘 정리된 비교 → 다이어그램으로 바꾸면 오히려 복잡
- 문서당 3개 이상 → 과하면 산만해짐

**Mermaid 코드 작성 규칙:**

```markdown
\`\`\`mermaid
graph TB
    A["한국어 라벨"] --> B["한국어 라벨"]
    style A fill:#c2410c,color:#fff,stroke:#9a3412,stroke-width:2px
\`\`\`
```

- 노드 라벨은 반드시 한국어
- `<br/>`로 줄바꿈 가능 (`A["첫 번째 줄<br/>두 번째 줄"]`)
- 색상은 사이트 브랜드 컬러 사용: `#c2410c`(빨강), `#d97706`(주황), `#2563eb`(파랑), `#16a34a`(초록)

#### 태그 규칙 (섹션 분류용)

youtube-update 카테고리는 태그로 섹션이 나뉜다. 문서마다 아래 태그 중 하나 이상을 반드시 포함:

- `신기능` 또는 `업데이트` → "새로운 기능 & 업데이트" 섹션
- `활용법` 또는 `워크플로우` → "활용법 & 워크플로우" 섹션
- `설정` 또는 `연동` → "설정 & 연동" 섹션
- `비즈니스` 또는 `자동화` → "비즈니스 & 자동화" 섹션

모든 문서에 `youtube` 태그도 필수 포함.
